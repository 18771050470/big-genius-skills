# 常见陷阱

## 陷阱 1：忽略边界条件导致算法在边缘输入时崩溃

### ❌ 反例
设计查找数组中最大值的函数，代码：
```python
def find_max(arr):
    max_val = arr[0]  # 假设数组非空
    for x in arr:
        if x > max_val:
            max_val = x
    return max_val
```
当 `arr = []` 时，`arr[0]` 抛出 IndexError。

### ✅ 正例
```python
def find_max(arr):
    if not arr:
        return None  # 或抛出特定异常
    max_val = arr[0]
    for x in arr:
        if x > max_val:
            max_val = x
    return max_val
```
先处理边界，再执行主逻辑。

### 原因分析
算法思维要求“输入域”必须被完整覆盖。未显式处理空数组意味着步骤序列隐含了“数组至少有一个元素”的前提，破坏了算法的通用性。

## 陷阱 2：步骤依赖隐含状态导致结果不可重复

### ❌ 反例
伪代码：  
```
1. 获取当前用户
2. 如果用户已登录，执行操作 A
3. 否则执行操作 B
```
问题在于没有定义“获取当前用户”的具体机制（从 session？token？用户输入？），导致不同环境执行结果可能不同。

### ✅ 正例
```
1. 接收参数 current_user（字符串，从调用处传入）
2. 如果 current_user 非空且存在于用户表，执行操作 A
3. 否则执行操作 B
```
明确输入来自函数参数，不依赖外部上下文。

### 原因分析
算法思维的关键是确定性。如果步骤依赖隐式全局状态（如环境变量、数据库连接状态），则同一算法在不同时间或机器上可能产生不同输出，违反了“有限步骤有序序列”的基本要求。必须将依赖项显式列为输入。
