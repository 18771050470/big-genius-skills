# 分析示例

## 示例 1：心理抗拒——产品功能变更

**输入**：我们强制所有用户使用新版界面，旧版直接下线，用户必须适应。

```
Step 1: 识别抗拒信号 → 产出：发现"强制...必须适应"表述，剥夺用户选择权触发心理抗拒
Step 2: 评估抗拒程度 → 产出：强制变更使用户失去选择自由，可能触发强烈抗拒反应
Step 3: 分析抗拒机制 → 产出：Brehm(1966)心理抗拒理论：自由被威胁时产生抗拒；反向行为：用户可能故意抵制
Step 4: 校正抗拒触发 → 产出：建议保留选择权，渐进过渡，降低抗拒
```

**输出**：
```
## 分析结论
### 核心发现
1. 强制变更剥夺用户选择权，触发心理抗拒
2. 抗拒表现：负面评价、抵制使用、流失
3. 历史数据：强制变更使用户流失率增加30%

### 建议（按优先级）
1. [高优先级] 保留旧版过渡期（3个月）
2. [高优先级] 提供选择权：用户可自主选择新版/旧版
3. [中优先级] 收集用户反馈，逐步优化

### 关键假设
- 用户重视选择权
- 渐进过渡降低抗拒

### 风险
- 强制变更可能触发用户流失
- 需要平衡产品演进和用户体验
```

## 示例 2：心理抗拒——管理政策

**输入**：公司规定所有员工必须9点到岗，迟到扣工资，没有例外。

```
Step 1: 识别抗拒信号 → 产出：发现"必须...没有例外"表述，刚性规定剥夺自主权
Step 2: 评估抗拒程度 → 产出：员工失去工作时间自主权，可能触发消极抵抗
Step 3: 分析抗拒机制 → 产出：自由威胁：自主权被剥夺；反向行为：迟到、低效、离职
Step 4: 校正抗拒触发 → 产出：建议弹性工作制，保留自主权
```

**输出**：
```
## 分析结论
### 核心发现
1. 刚性规定剥夺员工自主权，触发心理抗拒
2. 抗拒表现：消极抵抗、效率下降、离职率上升
3. 研究数据：弹性工作制使满意度提升25%，离职率降低15%

### 建议（按优先级）
1. [高优先级] 弹性工作制：核心时间10-15点，其余弹性
2. [高优先级] 结果导向：关注产出而非考勤
3. [中优先级] 试点弹性政策，评估效果

### 关键假设
- 员工可自我管理
- 弹性不影响协作

### 风险
- 刚性政策可能降低员工满意度
- 需要平衡管理和自主
```

## 反例：不该触发此 Skill 的场景

**输入**：这个安全操作必须严格遵守，否则会有生命危险。

**原因**：这是安全关键场景，自由限制是合理且必要的。心理抗拒适用于非必要的自由剥夺场景。
