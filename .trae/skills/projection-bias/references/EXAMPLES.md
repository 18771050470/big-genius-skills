# 分析示例

## 示例 1：投射偏差——用户需求假设

**输入**：我觉得这个功能很好用，用户肯定也喜欢，不需要做用户测试了。

```
Step 1: 识别投射信号 → 产出：发现"我觉得...用户肯定也喜欢"表述，将自己的偏好投射到他人
Step 2: 评估投射程度 → 产出：开发者偏好≠用户偏好，用户群体更多样化
Step 3: 分析投射机制 → 产出：自我中心偏差：自己是认知锚点；虚假共识：高估他人与自己的一致性
Step 4: 校正投射偏差 → 产出：建议用用户研究数据，而非自我假设
```

**输出**：
```
## 分析结论
### 核心发现
1. "我觉得好用"是开发者视角，不代表用户视角
2. 用户群体多样性：技术水平、使用场景、需求优先级差异大
3. 历史数据：开发者认为好的功能只有40%被用户广泛使用

### 建议（按优先级）
1. [高优先级] 进行用户测试，收集真实用户反馈
2. [高优先级] 区分开发者需求和用户需求
3. [中优先级] 建立用户画像，理解不同用户群体

### 关键假设
- 用户群体比开发者更多样化
- 历史数据反映普遍趋势

### 风险
- 投射偏差可能导致功能不符合用户需求
- 需要验证而非假设
```

## 示例 2：投射偏差——团队管理

**输入**：我喜欢独立工作，所以团队成员应该也喜欢被赋予自主权，不需要太多管理。

```
Step 1: 识别投射信号 → 产出：发现"我喜欢...团队成员应该也喜欢"表述，将自己的工作偏好投射到团队
Step 2: 评估投射程度 → 产出：团队成员工作偏好多样：有人喜欢独立，有人需要指导
Step 3: 分析投射机制 → 产出：自我中心偏差：自己的工作偏好被普遍化；虚假共识：高估偏好一致性
Step 4: 校正投射偏差 → 产出：建议了解团队成员个体差异，个性化管理
```

**输出**：
```
## 分析结论
### 核心发现
1. 管理者偏好(独立工作)≠团队偏好
2. 团队成员工作风格：独立型40%、协作型35%、指导型25%
3. 统一管理方式可能不适合所有人

### 建议（按优先级）
1. [高优先级] 了解每个团队成员的工作偏好
2. [高优先级] 个性化管理：独立型给空间，指导型给支持
3. [中优先级] 定期收集团队反馈，调整管理方式

### 关键假设
- 团队成员偏好多样
- 个性化管理可提高满意度

### 风险
- 投射偏差可能导致管理方式不适合部分成员
- 需要平衡统一和个性
```

## 反例：不该触发此 Skill 的场景

**输入**：根据用户调研数据，80%用户喜欢这个功能。

**原因**：这是基于用户调研数据的结论，不是自我投射。投射偏差指用自己的偏好替代他人偏好。
