# 分析示例

## 示例 1：峰终定律——用户体验评估

**输入**：这个产品整体体验一般，但最后一步支付流程很流畅，我对它印象还不错。

```
Step 1: 识别峰终信号 → 产出：发现"整体一般...最后流畅...印象不错"表述，最终体验主导整体评价
Step 2: 评估体验分布 → 产出：搜索(中)→浏览(差)→加购(中)→支付(好)，峰值和终值影响记忆
Step 3: 分析峰终机制 → 产出：Kahneman(1993)实验：痛苦体验的记忆由峰值和终值决定，而非总痛苦
Step 4: 校正峰终定律 → 产出：建议评估完整体验旅程，而非仅关注峰值和终值
```

**输出**：
```
## 分析结论
### 核心发现
1. 整体体验一般，但支付终值好使整体印象被提升
2. 峰终定律：记忆由峰值(最好/最坏)和终值(最后)决定，忽略持续时间
3. 浏览环节体验差但被终值掩盖，可能影响复购

### 建议（按优先级）
1. [高优先级] 优化完整体验旅程，而非仅关注终值
2. [高优先级] 识别并修复浏览环节的痛点
3. [中优先级] 保持支付流程优势

### 关键假设
- 用户体验各环节可测量
- 复购受完整体验影响

### 风险
- 峰终定律可能掩盖系统性问题
- 好终值可能暂时提升印象但长期流失
```

## 示例 2：峰终定律——面试体验

**输入**：面试过程大部分时间很无聊，但最后面试官很友好，整体感觉还行。

```
Step 1: 识别峰终信号 → 产出：发现"大部分无聊...最后友好...感觉还行"表述，终值覆盖整体体验
Step 2: 评估体验分布 → 产出：前40分钟(无聊)→最后10分钟(友好)，终值主导记忆
Step 3: 分析峰终机制 → 产出：持续时间忽视：60分钟体验被10分钟终值主导；峰值效应：友好时刻成为记忆锚点
Step 4: 校正峰终定律 → 产出：建议评估完整面试流程，而非仅终值
```

**输出**：
```
## 分析结论
### 核心发现
1. 前40分钟无聊但被最后10分钟友好覆盖
2. 峰终定律使终值不成比例地影响整体评价
3. 无聊的前40分钟可能反映面试设计问题

### 建议（按优先级）
1. [高优先级] 优化前40分钟面试内容
2. [高优先级] 设计结构化面试流程
3. [中优先级] 保持友好的结束氛围

### 关键假设
- 面试各环节都可优化
- 完整体验影响候选人判断

### 风险
- 峰终定律可能掩盖面试设计问题
- 好终值可能使候选人忽视公司问题
```

## 反例：不该触发此 Skill 的场景

**输入**：这个产品每个环节体验都很好，整体印象非常好。

**原因**：这是各环节一致好的情况，不存在峰终偏差。峰终定律指峰值和终值不成比例地影响记忆。
