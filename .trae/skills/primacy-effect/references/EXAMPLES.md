# 分析示例

## 示例 1：首因效应——供应商评估

**输入**：第一次合作这个供应商就出了点问题，后面虽然改善了，但我还是不太信任他们。

```
Step 1: 识别首因信号 → 产出：发现"第一次...还是不太信任"表述，最早信息主导整体评价
Step 2: 评估信息序列 → 产出：第一次问题(负面)→后续10次合作零问题(正面)，但首次印象仍主导
Step 3: 分析首因机制 → 产出：注意力峰值：第一次合作获得最多注意力；解释框架：首次问题成为后续解释过滤器
Step 4: 校正首因效应 → 产出：建议按完整合作记录评估，而非首次印象
```

**输出**：
```
## 分析结论
### 核心发现
1. 首次合作问题是负面首因，但后续10次合作零问题
2. 整体合作记录：11次合作，1次问题(9%)，91%准时交付
3. 首次印象使91%准时率被低估

### 建议（按优先级）
1. [高优先级] 基于完整合作记录评估：91%准时率优秀
2. [高优先级] 识别首次问题的根本原因，确认已解决
3. [中优先级] 设置持续监控，而非基于首次印象决策

### 关键假设
- 后续10次合作数据可靠
- 首次问题根因已解决

### 风险
- 首因效应可能导致错失优质供应商
- 需要区分首次问题和系统性问题
```

## 示例 2：首因效应——面试评价

**输入**：面试前5分钟他有点紧张，后面表现很好，但第一印象不太好。

```
Step 1: 识别首因信号 → 产出：发现"第一印象不太好"表述，最早信息主导整体评价
Step 2: 评估信息序列 → 产出：前5分钟紧张(负面)→后续55分钟优秀(正面)，但首印象仍影响评价
Step 3: 分析首因机制 → 产出：一致性偏好：维持初始印象；确认偏误：寻找支持首印象的证据
Step 4: 校正首因效应 → 产出：建议分阶段独立评分，加总评估
```

**输出**：
```
## 分析结论
### 核心发现
1. 前5分钟紧张是首因，但后续55分钟表现优秀
2. 分阶段评分：前5分钟6/10，后续55分钟9/10，加权平均8.6/10
3. 紧张是常见面试反应，不代表工作能力

### 建议（按优先级）
1. [高优先级] 基于加权平均(8.6/10)评估，而非首印象
2. [高优先级] 识别紧张原因：性格内向 vs 准备不足
3. [中优先级] 改进面试流程：前5分钟破冰环节

### 关键假设
- 后续表现更能反映真实能力
- 紧张与工作表现无强相关

### 风险
- 首因效应可能导致错失优秀候选人
- 需要区分紧张和能力不足
```

## 反例：不该触发此 Skill 的场景

**输入**：第一次合作就严重违约，造成重大损失，我们不考虑继续合作。

**原因**：这是基于严重违约事实的合理决策，不是首因效应。首因效应指最早信息过度影响评价，但严重违约是实质性风险信号。
