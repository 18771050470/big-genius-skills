# Gotchas

> 只写实验追踪容易犯错的**非显而易见的事实**。通用知识不要写。

## Gotcha 1：样本量不足导致统计功效低

**问题**：实验样本量太小，无法检测到真实存在的效应，导致"假阴性"（实际有效但实验显示无效）。

❌ **Before（错误做法）**
```markdown
# 实验设计
- 实验组：1000 用户
- 对照组：1000 用户
- 预期提升：5%
- 实验时长：1 周

# 结果
- 实验组转化率：12%
- 对照组转化率：11.5%
- P 值：0.15
- 结论：无显著差异，功能无效
```

✅ **After（正确做法）**
```markdown
# 实验设计（样本量计算）
- 基线转化率：11.5%
- 预期提升：5%（相对提升，即 11.5% → 12.075%）
- 显著性水平 α：0.05
- 统计功效 1-β：0.8
- **计算所需样本量**：每组 38,000 用户（使用功率分析）

# 实际执行
- 实验组：40,000 用户
- 对照组：40,000 用户
- 实验时长：4 周（达到样本量）

# 结果
- 实验组转化率：12.1%
- 对照组转化率：11.5%
- P 值：0.03
- 结论：显著有效，建议全量发布
```

**原因**：样本量不足是 A/B 测试最常见的错误。使用功率分析（power analysis）计算所需样本量，确保有足够能力检测到预期效应。

## Gotcha 2：偷看数据导致假阳性

**问题**：实验进行中频繁查看数据，看到"显著"就停止，大大增加假阳性率。

❌ **Before（错误做法）**
```markdown
# 实验执行
- Day 1：P = 0.3，继续
- Day 2：P = 0.15，继续
- Day 3：P = 0.04，显著！停止实验，发布功能
```

✅ **After（正确做法）**
```markdown
# 实验设计（预先确定停止规则）
- 实验时长：预先计算需 2 周达到样本量
- 停止规则：
  - 正常停止：2 周后分析结果
  - 提前停止（仅当严重负面）：核心指标下降 > 20%

# 实验执行
- Week 1：不查看数据（避免偷看）
- Week 2：不查看数据
- Week 2 结束：正式分析
  - P = 0.04，置信区间 [2%, 8%]
  - 结论：显著有效，建议发布

# 如果必须监控
- 只看护栏指标（如错误率、退款率）
- 核心指标在实验结束前不分析
- 使用序贯检验（sequential testing）方法
```

**原因**：多次检验会放大假阳性率（5% 的 α 检验 10 次，假阳性率约 40%）。必须预先确定实验时长，避免偷看数据。

## Gotcha 3：网络效应污染实验结果

**问题**：实验组和对照组用户相互影响（如社交功能、共享资源），导致实验结果不可信。

❌ **Before（错误做法）**
```markdown
# 实验设计：新社交功能
- 分组：按用户 ID 哈希 50/50
- 实验组：可以使用新功能
- 对照组：不能使用

# 问题
- 实验组用户邀请对照组用户使用新功能
- 对照组用户通过实验组用户间接体验功能
- 两组不再独立，结果不可解释
```

✅ **After（正确做法）**
```markdown
# 实验设计：新社交功能（处理网络效应）

## 方案 A：按集群分组
- 按"城市"分组：北京用户 vs 上海用户
- 同一城市内用户相互影响，但城市间影响小
- 缺点：城市间可能有系统性差异

## 方案 B：Ego-network 分组
- 识别社交网络中的"簇"（clusters）
- 按簇随机分组，确保同簇用户在同一组
- 需要社交网络数据支持

## 方案 C：网络效应建模
- 假设网络效应可量化
- 使用网络效应模型（如 SUTVA 违反模型）
- 估计直接效应和间接效应

# 推荐
- 如果网络效应强（如通讯功能），使用方案 A 或 B
- 如果网络效应弱，使用方案 C 进行敏感性分析
```

**原因**：网络效应违反 SUTVA（Stable Unit Treatment Value Assumption），使得传统 A/B 测试结论不可信。必须根据网络效应强度选择合适的实验设计。

## Gotcha 4：多重比较未校正导致假阳性

**问题**：同时检验多个指标或多个子组，不校正假阳性率，导致"显著"结果实际是巧合。

❌ **Before（错误做法）**
```markdown
# 实验结果
| 指标 | P 值 | 显著？ |
|------|------|--------|
| 转化率 | 0.04 | ✅ |
| 留存率 | 0.03 | ✅ |
| 客单价 | 0.02 | ✅ |
| 新用户转化 | 0.05 | ✅ |
| 老用户转化 | 0.04 | ✅ |
| iOS 转化 | 0.03 | ✅ |
| Android 转化 | 0.08 | ❌ |

结论：6/7 指标显著，功能非常有效！
```

✅ **After（正确做法）**
```markdown
# 实验结果（Bonferroni 校正）

## 整体指标（α = 0.05）
| 指标 | P 值 | 校正后 | 显著？ |
|------|------|--------|--------|
| 转化率（OEC） | 0.04 | 0.04 | ✅ |
| 留存率 | 0.03 | 0.06 | ❌ |
| 客单价 | 0.02 | 0.04 | ✅ |

## 子组分析（α = 0.05/4 = 0.0125）
| 子组 | P 值 | 显著？ |
|------|------|--------|
| 新用户 | 0.05 | ❌ |
| 老用户 | 0.04 | ❌ |
| iOS | 0.03 | ❌ |
| Android | 0.08 | ❌ |

## 结论
- 核心指标（转化率、客单价）显著
- 子组分析无显著差异（可能是假阳性）
- 建议：全量发布，继续监控留存率
```

**原因**：同时检验 N 个指标，至少一个假阳性的概率 = 1 - (1-α)^N。检验 7 个指标时，假阳性率约 30%。必须使用 Bonferroni 或 FDR 方法校正。
