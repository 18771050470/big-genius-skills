# 常见陷阱

## 陷阱 1：用结果评价决策质量

**问题**：因坏结果而否定好决策过程。

❌ **Before（错误做法）**
```markdown
## 分析结论
诊断错误，延误治疗，医生水平不行。
建议：追责。
```
*问题：结果偏差使坏结果否定好决策过程*

✅ **After（正确做法）**
```markdown
## 分析结论
### 决策过程评估
| 维度 | 评估 |
|------|------|
| 症状分析 | 正确(发热、咳嗽→感冒) |
| 概率判断 | 正确(常见病>罕见病) |
| 流程遵循 | 正确 |
| 结果 | 坏(罕见病) |

### 建议
1. 决策过程正确，结果受运气影响
2. 不应因坏结果追责好决策
```
*优势：过程评估替代结果评价*

**原因**：Baron & Hershey(1988)实验：相同决策过程，好结果时评分7.2/10，坏结果时评分4.5/10。结果影响过程评价。

## 陷阱 2：后见之明偏差

**问题**：知道结果后认为应该预见。

❌ **Before（错误做法）**
```markdown
## 分析结论
现在看症状很明显，当时怎么没发现？
建议：医生应该更仔细。
```
*问题：后见之明使结果看起来可预见*

✅ **After（正确做法）**
```markdown
## 分析结论
### 时间视角对比
| 时间点 | 信息 | 可预见性 |
|--------|------|---------|
| 决策时 | 发热、咳嗽 | 低(罕见病<0.1%) |
| 知道结果后 | 回顾症状 | 高(后见之明) |

### 建议
1. 后见之明使结果看起来可预见
2. 应基于决策时信息评估
```
*优势：区分决策时和知道结果后的信息*

**原因**：Fischhoff(1975)后见之明偏差：知道结果后，人们系统性高估结果的可预见性。

## 陷阱 3：混淆技能和运气

**问题**：将运气导致的结果归因于技能。

❌ **Before（错误做法）**
```markdown
## 分析结论
他买股票赚了10倍，投资能力超强。
建议：跟他学习。
```
*问题：结果偏差将运气归因于技能*

✅ **After（正确做法）**
```markdown
## 分析结论
### 技能vs运气分析
| 因素 | 评估 |
|------|------|
| 决策过程 | 随机选择(无分析) |
| 结果 | 10倍收益 |
| 归因 | 运气(非技能) |

### 建议
1. 随机选择+好结果=运气
2. 不应从运气结果学习"经验"
```
*优势：区分技能和运气*

**原因**：Mauboussin(2012)：结果=技能+运气。短期结果运气主导，长期结果技能主导。
