# 技术交付物模板

> 提供可直接使用的 Markdown/代码模板。

## 交付物模板 1：标注项目计划书

```markdown
# 标注项目计划: [项目名称]

## 项目概述
- **项目目标**: ...
- **数据规模**: [N] 条
- **标注类型**: [分类/实体/关系/...]
- **交付日期**: YYYY-MM-DD

## 团队配置

| 角色 | 人数 | 职责 |
|------|------|------|
| 项目经理 | 1 | 进度管理、质量控制 |
| 标注员 | 10 | 数据标注 |
| 质检员 | 2 | 标注审核、一致性检查 |
| 领域专家 | 1 | 指南制定、争议仲裁 |

## 时间计划

| 阶段 | 工期 | 开始 | 结束 | 交付物 |
|------|------|------|------|--------|
| 指南制定 | 1周 | ... | ... | 标注指南 v1.0 |
| 试点标注 | 3天 | ... | ... | 一致性报告 |
| 批量标注 | 2周 | ... | ... | 标注数据 |
| 质量审核 | 1周 | ... | ... | 质检报告 |
| 验收交付 | 2天 | ... | ... | 最终数据集 |

## 质量标准
- 标注者间一致性 (Cohen's Kappa) > 0.8
- 质检通过率 > 95%
- 黄金样本准确率 > 98%

## 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 标注质量不达标 | 中 | 高 | 加强培训、增加质检轮次 |
| 进度延迟 | 低 | 中 | 预留缓冲时间 |
| 人员流失 | 中 | 中 | 建立备份人员池 |
```

## 交付物模板 4：数据集交付清单

```markdown
# 数据集交付清单: [数据集名称]

## 数据集信息
- **名称**: ...
- **版本**: v[X.Y.Z]
- **数据量**: [N] 条
- **标注类型**: ...
- **创建日期**: YYYY-MM-DD

## 文件清单

| 文件名 | 说明 | 格式 | 大小 | 记录数 |
|--------|------|------|------|--------|
| data.jsonl | 标注数据 | JSONL | 100MB | 10000 |
| metadata.csv | 元数据 | CSV | 1MB | 10000 |
| guide.md | 标注指南 | Markdown | 50KB | - |
| report.pdf | 质量报告 | PDF | 2MB | - |

## 数据格式

```json
{
  "id": "uuid",
  "text": "原始文本",
  "annotations": [
    {
      "label": "类别",
      "start": 0,
      "end": 10,
      "annotator": "A001",
      "timestamp": "2024-01-01T00:00:00Z"
    }
  ],
  "metadata": {
    "source": "...",
    "language": "zh",
    "domain": "..."
  }
}
```

## 质量统计

| 指标 | 数值 |
|------|------|
| 总样本数 | 10000 |
| 有效标注数 | 9800 |
| 弃用样本数 | 200 |
| 一致性 (Kappa) | 0.82 |

## 使用许可
- [ ] 内部使用
- [ ] 商用许可
- [ ] 需署名

## 已知问题
- ...
- ...
```
