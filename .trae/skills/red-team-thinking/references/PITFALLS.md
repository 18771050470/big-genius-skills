# 常见陷阱

## 陷阱 1：基于"正常用户"假设设计防御

**反例**：
```
"用户不会尝试上传.exe文件"（攻击者不是正常用户）
```

**正例**：
```
"攻击者会尝试所有文件类型，包括绕过扩展名检查的变种"
```

**原因**：攻击者不遵守规则。

---

## 陷阱 2：只防已知攻击，不防未知攻击

**反例**：
```
"我们防御了OWASP Top 10，所以安全了"
```

**正例**：
```
"OWASP Top 10是基线，还需要防御组合攻击和0day"
```

**原因**：攻击者会使用你没想到的路径。

---

## 陷阱 3：纵深防御不独立

**反例**：
```
"我们有WAF+IDS+防火墙"（但都依赖同一个规则库）
```

**正例**：
```
"WAF(签名检测)+IDS(行为分析)+防火墙(网络层)相互独立"
```

**原因**：共享弱点的多层防御等于单层。

---

## 陷阱 4：忽略内部威胁

**反例**：
```
"内部人员可信，不需要防护"
```

**正例**：
```
"内部人员有最高权限，一旦泄露影响最大，需要最小权限+审计"
```

**原因**：内部威胁概率低但影响最大。
