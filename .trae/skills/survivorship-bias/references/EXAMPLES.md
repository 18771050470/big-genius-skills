# 分析示例

## 示例 1：幸存者偏差——创业经验学习

**输入**：这些成功企业家都说要坚持梦想、永不放弃，我们也应该这样，一定能成功。

```
Step 1: 识别幸存者信号 → 产出：发现"成功企业家说...一定能成功"表述，只看到成功者，忽视失败者
Step 2: 评估样本偏差 → 产出：成功企业家是幸存者，失败者声音不可见；失败者可能也"坚持梦想"但失败了
Step 3: 分析幸存者机制 → 产出：可见性偏差：成功者被媒体放大；沉默证据：失败者不被关注
Step 4: 校正幸存者偏差 → 产出：建议研究失败案例，获取完整样本
```

**输出**：
```
## 分析结论
### 核心发现
1. 只听到成功者的建议，失败者声音不可见
2. 失败者可能也"坚持梦想"，但结果不同
3. 创业成功率<10%，成功者建议不代表普遍规律

### 建议（按优先级）
1. [高优先级] 研究失败案例：失败者做了什么相同的事？
2. [高优先级] 理解成功=能力+运气，非仅坚持
3. [中优先级] 用统计数据评估建议可靠性

### 关键假设
- 失败者样本不可见
- 成功建议可能不适用于所有人

### 风险
- 幸存者偏差可能导致盲目模仿
- 需要完整样本学习
```

## 示例 2：幸存者偏差——产品功能设计

**输入**：用户反馈说需要这个功能，我们应该加上，加了肯定受欢迎。

```
Step 1: 识别幸存者信号 → 产出：发现"用户反馈说...肯定受欢迎"表述，只听到活跃用户声音
Step 2: 评估样本偏差 → 产出：反馈用户是活跃用户(幸存者)，已流失用户声音不可见
Step 3: 分析幸存者机制 → 产出：自我选择偏差：只有留存用户会反馈；流失用户可能因缺少此功能而离开
Step 4: 校正幸存者偏差 → 产出：建议调研流失用户，获取完整反馈
```

**输出**：
```
## 分析结论
### 核心发现
1. 反馈来自活跃用户，流失用户声音不可见
2. 流失用户可能因缺少此功能而离开
3. 只优化活跃用户需求可能加剧流失

### 建议（按优先级）
1. [高优先级] 调研流失用户：为什么离开？
2. [高优先级] 分析完整用户生命周期
3. [中优先级] 平衡活跃用户和潜在用户需求

### 关键假设
- 流失用户可联系
- 流失原因可识别

### 风险
- 幸存者偏差可能导致产品方向偏离
- 需要完整用户视角
```

## 反例：不该触发此 Skill 的场景

**输入**：我们调研了1000个用户，包括活跃和流失用户，80%需要这个功能。

**原因**：这已包含完整样本（活跃+流失用户），不是幸存者偏差。幸存者偏差指只看到成功/存活样本。
