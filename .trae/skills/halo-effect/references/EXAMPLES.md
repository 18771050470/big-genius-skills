# 分析示例

## 示例 1：面试评价中的晕轮效应

**输入**：这个候选人是清华毕业的，沟通能力也不错，肯定是个优秀的人才。

```
Step 1: 识别晕轮信号 → 产出：发现"清华毕业的"、"肯定优秀"表述，单一特征扩散到整体评价
Step 2: 评估晕轮影响 → 产出：名校光环使其他能力被高估，忽视实际技能评估
Step 3: 分析晕轮机制 → 产出：认知捷径：名校=优秀；扩散效应：正面特征扩散到无关特征
Step 4: 校正晕轮效应 → 产出：建议分维度独立评估，避免特征扩散
```

**输出**：
```
## 分析结论
### 核心发现
1. "清华毕业"是单一正面特征，不应扩散到整体评价
2. 岗位核心需求：编程能力、项目经验、团队协作
3. 候选人实际评估：编程测试70分（中等）、项目经验2年（一般）、团队协作待验证

### 建议（按优先级）
1. [高优先级] 分维度独立评估：学历、编程、经验、协作
2. [高优先级] 编程测试70分需关注，是否满足岗位要求
3. [中优先级] 增加实际项目测试，验证真实能力

### 关键假设
- 学历≠工作能力
- 编程测试可反映真实水平

### 风险
- 晕轮效应可能导致高估候选人
- 需要区分学历光环和实际能力
```

## 示例 2：产品评价中的晕轮效应

**输入**：Apple设计的产品肯定好用，这个新功能体验一定很棒。

```
Step 1: 识别晕轮信号 → 产出：发现"Apple设计的肯定好用"表述，品牌光环扩散到功能评价
Step 2: 评估晕轮影响 → 产出：品牌正面印象使新功能被预设为好，忽视实际功能评估
Step 3: 分析晕轮机制 → 产出：品牌光环：Apple=好设计；扩散效应：品牌好感扩散到具体功能
Step 4: 校正晕轮效应 → 产出：建议独立评估功能本身，而非基于品牌预设
```

**输出**：
```
## 分析结论
### 核心发现
1. "Apple设计"是品牌光环，不应作为功能质量依据
2. 该功能实际评估：用户测试显示学习曲线陡峭，30%用户首次使用失败
3. Apple历史产品整体优秀，但非每个功能都成功

### 建议（按优先级）
1. [高优先级] 基于用户测试数据评估功能，而非品牌预设
2. [高优先级] 识别功能具体问题（学习曲线、首次失败率）
3. [中优先级] 改进功能设计，降低使用门槛

### 关键假设
- 用户测试数据可靠
- 品牌历史表现≠当前功能质量

### 风险
- 品牌光环可能掩盖功能缺陷
- 需要客观评估每个功能
```

## 反例：不该触发此 Skill 的场景

**输入**：这个候选人学历好、编程测试好、项目经验好、面试表现好，综合评价优秀。

**原因**：这是多维度独立评估后的一致结论，不是单一特征扩散。晕轮效应指单一特征影响整体评价。
