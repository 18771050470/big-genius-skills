# 常见陷阱

## 陷阱 1：单一特征扩散到整体评价

**问题**：因一个正面特征而认为整体都优秀。

❌ **Before（错误做法）**
```markdown
## 分析结论
候选人是清华毕业的，肯定是个优秀人才。
建议：录用。
```
*问题：晕轮效应使名校光环扩散到整体评价，忽视实际能力评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 分维度评估
| 维度 | 评分 | 依据 |
|------|------|------|
| 学历 | 9/10 | 清华毕业 |
| 编程能力 | 7/10 | 测试70分 |
| 项目经验 | 6/10 | 2年经验，一般 |
| 团队协作 | ?/10 | 待验证 |

### 建议
1. 学历优秀但编程能力中等，需关注
2. 岗位核心需求是编程，学历是加分项非决定项
```
*优势：分维度独立评估，防止特征扩散*

**原因**：Thorndike(1920)军官评估实验证明：外貌好的军官被评价为智力更高、领导力更强。单一正面特征的光环扩散到无关特征的评价。

## 陷阱 2：品牌光环替代产品评估

**问题**：因品牌好而认为产品一定好。

❌ **Before（错误做法）**
```markdown
## 分析结论
Apple设计的功能肯定好用。
建议：不需要用户测试，直接上线。
```
*问题：晕轮效应使品牌光环替代功能评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 功能独立评估
| 评估维度 | 结果 |
|---------|------|
| 用户测试 | 30%首次使用失败 |
| 学习曲线 | 陡峭，需3次以上使用 |
| 品牌历史 | 整体优秀，但非每个功能成功 |

### 建议
1. 基于用户测试数据评估，而非品牌预设
2. 改进功能设计，降低使用门槛
```
*优势：独立评估功能质量，不受品牌影响*

**原因**：晕轮效应使品牌好感扩散到具体产品。但每个产品/功能需独立评估。

## 陷阱 3：第一印象锁定后续评价

**问题**：因第一印象好而后续评价都偏向正面。

❌ **Before（错误做法）**
```markdown
## 分析结论
面试开始10分钟感觉很好，后面应该也不错。
建议：给高分。
```
*问题：晕轮效应+首因效应使第一印象锁定后续评价*

✅ **After（正确做法）**
```markdown
## 分析结论
### 分阶段评估
| 阶段 | 评估 | 说明 |
|------|------|------|
| 前10分钟 | 8/10 | 沟通流畅、自信 |
| 技术问答 | 6/10 | 基础知识扎实，深度不足 |
| 案例分析 | 5/10 | 思路清晰但缺乏实战经验 |

### 建议
1. 每阶段独立评分，避免第一印象影响
2. 技术深度和实战经验是岗位核心需求
```
*优势：分阶段独立评估，防止第一印象锁定*

**原因**：晕轮效应使第一印象成为后续评价的过滤器。分阶段评估打破这种锁定。
