# 因果思维常见陷阱 (PITFALLS)

## 陷阱1：忽略混淆变量导致虚假因果

### 问题描述
当两个变量X和Y高度相关时，错误地认为X是Y的原因，而实际上是由第三个变量Z同时影响两者导致的虚假相关。这是因果推断中最经典的陷阱。

**❌ 反例**
某电商发现“用户点击主页横幅广告的次数”与“最终购买率”相关系数 r=0.68。产品经理立即建议：增加广告横幅展示次数，预计购买率提升30%。

分析：忽略混淆变量“用户购买意图”。事实上，对商品已有购买意图的用户更可能主动点击广告（而非广告导致他们购买）。广告展示次数和购买率都是购买意图的结果，而非因果关系。

**✅ 正例**
正确的做法是先识别可能的混淆变量Z（如用户搜索历史、驻足时间），然后使用倾向得分匹配或工具变量法控制Z。对本案例：
- 构造反事实：将点击广告的用户分为“随机看到广告”（如通过A/B测试）与“已搜过商品”两组，对比购买率。
- 正确结论：广告曝光本身对购买率影响很小（仅增加品牌回忆），主要驱动是用户已有购买意图。因此不应盲目增加横幅，而应优化搜索结果的推荐策略。

**原因分析**：混淆变量是因果推断中最隐蔽的威胁。人脑对相关性的直观感受倾向于立即赋予因果解释（因果直觉），而混淆变量往往不显眼。如果没有系统性怀疑并列出潜在的Z，就会落入“相关→因果”的陷阱。

## 陷阱2：后此谬误（Post Hoc Ergo Propter Hoc）

### 问题描述
因为事件B发生在事件A之后（时间顺序），就轻易断定A是B的原因。在时间序列数据中尤其常见。

**❌ 反例**
某SaaS公司在5月推出了一项新功能“团队看板”，随后6月份公司的月活跃用户（MAU）环比增长了15%。CEO在新功能发布会上声称：“是团队看板带来了月活增长”。

分析：忽略其他同时发生的变化（如5月底营销活动“邀请同事送免费名额”也刚好启动），以及季节性因素。6月份是教育行业采购高峰，这部分用户并不一定使用新功能。将增长归因于新功能是典型的后此谬误。

**✅ 正例**
正确的分析：
1. 不要只看时间先后，要构建因果链：新功能 → 用户使用率 → 留存率 → MAU。
2. 收集数据：新功能上线后，实际使用了团队看板的用户占比仅15%，且这些用户之前的留存率已经很高。增加功能后，这些用户的留存率并未显著变化。
3. 控制其他变量：使用双重差分法（DID），比较“使用新功能”与“未使用新功能”用户的增长率变化，同时考虑是否在营销活动前后。
4. 结论：新功能对MAU增长的贡献微乎其微（＜1%），主要增长来自营销活动带来的新用户，而这些新用户并未留到6月底。

**原因分析**：人们天生倾向于寻找模式（因果归因），而时间顺序是模式中最直观的元素。避免后此谬误的关键在于：
- 永远追问“除了时间先后，还有其他证据吗？”
- 寻找反例（有没有B发生但A没发生的案例？）
- 使用实验或准实验方法来分离因果效应。

## 陷阱3：忽略因果链中的中介变量与反馈回路

### 问题描述
在构建因果模型时，仅仅关注X→Y的直接路径，忽略了可能存在的中间变量M（中介）或反馈回路（Y影响X），导致因果解释不完整或错误。

**❌ 反例**
某公司发现“加班时长（X）”与“员工离职率（Y）”正相关，于是管理层得出结论：加班导致离职，因此严格控制加班。

分析：忽略了中介变量M=“员工疲劳指数”。事实上，加班通过增加疲劳导致离职，但更重要的中介可能是M2=“工作成就感缺失”。同时可能存在反馈：离职率上升（Y）→ 团队剩余人员工作负荷更大 → 加班更多（X），形成恶性循环。仅归因于“加班→离职”会忽略这个反馈机制。

**✅ 正例**
正确的因果分析：
1. 画出因果链图（DAG）：X → M1（疲劳）→ Y；X → M2（成就感下降）→ Y；同时Y → X（反馈）。
2. 收集疲劳度、成就感等数据，使用中介分析检验M1、M2的效应大小。
3. 发现：M2（成就感缺失）的间接效应远大于M1（疲劳）。同时存在Y→X的强反馈（离职→加班增加→更多离职）。
4. 建议：与其直接控制加班时长，不如改善任务分配和团队支持，打破反馈回路。

**原因分析**：人脑倾向于用“直接因果关系”简化世界，忘记中间环节和系统动态。使用DAG强制要求列出所有变量和箭头方向，可以有效避免此类陷阱。不画出因果图就进行直接归因，是最大的风险。
