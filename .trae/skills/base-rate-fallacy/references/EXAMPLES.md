# 分析示例

## 示例 1：项目风险评估中的基率忽视

**输入**：我们团队技术很强，这个AI项目虽然行业失败率90%，但我们肯定能成功。

```
Step 1: 识别基率信号 → 产出：发现"行业失败率90%"被忽视，"我们肯定能成功"替代基率
Step 2: 评估基率影响 → 产出：行业基率90%失败是重要参考，团队能力需在此基率上调整
Step 3: 分析基率机制 → 产出：内部视角：过度关注自身优势；外部视角：忽视行业统计
Step 4: 校正基率谬误 → 产出：建议结合基率和团队特质，计算调整后成功率
```

**输出**：
```
## 分析结论
### 核心发现
1. 行业基率：AI项目失败率90%，这是重要参考点
2. 团队优势：技术强，但AI项目成功还需数据、场景、商业化能力
3. 调整后估计：基于团队优势，成功率从10%调整到25%，仍偏低

### 建议（按优先级）
1. [高优先级] 识别降低失败率的具体措施（数据优势、场景壁垒）
2. [高优先级] 分阶段投入，每阶段设置继续/终止决策点
3. [中优先级] 寻找已成功案例，学习其成功要素

### 关键假设
- 行业基率适用于本项目类型
- 团队技术优势可转化为项目优势

### 风险
- 即使调整后成功率仍只有25%
- 需要控制单次投入规模
```

## 示例 2：招聘决策中的基率忽视

**输入**：这个候选人面试表现很好，虽然名校毕业生在我们公司留存率只有30%，但他看起来不一样。

```
Step 1: 识别基率信号 → 产出：发现"留存率30%"基率被忽视，"看起来不一样"替代统计
Step 2: 评估基率影响 → 产出：30%留存率是重要参考，个人表现需在此基率上评估
Step 3: 分析基率机制 → 产出：代表性启发：面试表现好被误认为留存率高；个案突出：个人印象覆盖统计
Step 4: 校正基率谬误 → 产出：建议结合基率和个体信号，计算留存概率
```

**输出**：
```
## 分析结论
### 核心发现
1. 基率：名校毕业生在公司留存率30%
2. 个体信号：面试表现好，但面试表现与留存相关性仅0.3
3. 调整后估计：基于面试表现，留存概率从30%调整到40%，仍不确定

### 建议（按优先级）
1. [高优先级] 增加留存预测因子：职业规划匹配度、通勤距离
2. [高优先级] 设置3个月试用期，验证实际适配度
3. [中优先级] 改进入职流程，提升整体留存率

### 关键假设
- 30%基率反映系统性问题，非个人问题
- 面试表现与留存相关性低

### 风险
- 即使调整后留存概率仍只有40%
- 需要系统性提升留存率，而非依赖个人筛选
```
