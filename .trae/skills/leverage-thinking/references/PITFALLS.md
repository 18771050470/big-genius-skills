# 常见陷阱

## 陷阱 1：把紧急当高杠杆

**反例**：
```
"这个bug最紧急，先修"（但影响只有1个用户）
```

**正例**：
```
"这个bug影响30%用户，杠杆率高于那个只影响1个用户的紧急bug"
```

**原因**：紧急不等于高杠杆。

---

## 陷阱 2：忽略行动间的依赖

**反例**：
```
"先做B（杠杆率高）"但B依赖A的完成
```

**正例**：
```
"A是B的前提，虽然A杠杆率中等，但必须先做A"
```

**原因**：依赖关系可能改变执行顺序。

---

## 陷阱 3：杠杆率不随时间更新

**反例**：
```
"这个行动杠杆率最高，永远先做它"
```

**正例**：
```
"当前这个行动杠杆率最高，但3个月后数据增长，性能优化的杠杆率会超过它"
```

**原因**：杠杆率随系统状态变化。

---

## 陷阱 4：忽略负杠杆

**反例**：
```
只计算正杠杆行动
```

**正例**：
```
"这个重构看似好，但引入新bug的概率30%，期望值为负，不做"
```

**原因**：错误方向的努力比不努力更糟。
