# 分析示例

## 示例 1：技术债务处理优先级

**输入**：有50个技术债务项，资源有限，先处理哪些？

```
Step 1: 列出所有可行行动
→ 50个技术债务项，按类型分类：代码重复、缺少测试、性能瓶颈、安全漏洞、文档缺失

Step 2: 评估影响幅度
→ 安全漏洞：影响所有用户（影响幅度100%）
→ 性能瓶颈：影响30%慢查询用户
→ 缺少测试：影响未来开发效率（间接影响）
→ 代码重复：影响维护成本
→ 文档缺失：影响新成员上手速度

Step 3: 评估影响持久度
→ 安全漏洞修复：持久（除非新漏洞出现）
→ 性能优化：半衰期6个月（数据增长会抵消）
→ 测试覆盖：持久（持续保护）
→ 代码重构：半衰期12个月（新功能会引入新重复）

Step 4: 计算杠杆率
→ 安全漏洞：(100% × 持久) / 2天 = 极高
→ 核心路径性能：(30% × 6个月) / 1周 = 高
→ 关键模块测试：(间接 × 持久) / 3天 = 中高
→ 文档：(间接 × 12个月) / 1天 = 中
→ 产出：按杠杆率排序的行动清单
```

**输出**：
```
## 分析结论

### 核心发现
1. 2个安全漏洞杠杆率最高，影响所有用户且修复成本低
2. 核心路径性能优化杠杆率高，但需定期重新优化
3. 测试覆盖的长期杠杆率最高，但短期效果不明显

### 建议（按优先级）
1. [高] 修复2个安全漏洞，2天
2. [高] 优化核心查询路径性能，1周
3. [中] 为核心业务逻辑添加测试覆盖，2周
4. [低] 补充架构文档，1天
```
