# 费曼学习法思维（Feynman Technique Thinking）

## 核心定义

通过用简单语言向一个外行（或假想的孩子）解释概念，来暴露自己理解中的模糊地带和知识盲区——如果你不能简单地解释它，说明你还没有真正理解它。

---

## 思考流程控制

### 中断处理

- **信息不足**：用 AskUserQuestion 向用户确认目标受众的知识水平（完全外行/有一定基础/专业人士）
- **发现矛盾**：停止思考，用 AskUserQuestion 向用户报告矛盾点（如"简化导致关键信息丢失"），等待决策
- **提前终止**：若概念本身已足够简单（如"变量是存储数据的容器"），可提前终止

---

## 思考流程

### Step 1: 选择目标概念

- 明确要学习的概念：一个理论、一个算法、一个设计模式、一个业务规则
- 限定范围：不要试图一次性理解整个领域，聚焦一个具体概念
- 准备白纸：在顶部写下概念名称，下方留白用于书写解释
- 陷阱：选择范围过大（如"理解机器学习"而非"理解决策树"），导致无法深入

### Step 2: 用简单语言解释

- 假装教一个12岁孩子：使用日常语言，避免专业术语和 jargon
- 使用类比：将抽象概念映射到具体生活场景（如"递归就像俄罗斯套娃"）
- 可视化：用图表、流程图或示意图辅助解释
- 写下方能想到的关于这个概念的一切：不要查资料，仅凭当前理解
- 陷阱：使用专业术语伪装理解（如"这是通过反向传播优化损失函数"——孩子听不懂）

### Step 3: 识别知识盲区

- 标记卡壳点：在解释过程中，哪些地方说不下去了？
- 识别模糊地带：哪些表述感觉"大概对但不精确"？
- 发现逻辑断层：哪些步骤之间的因果关系不清楚？
- 记录术语依赖：是否不自觉地使用了未解释的专业术语？
- 陷阱：跳过卡壳点（"这部分比较复杂，先跳过"），导致盲区持续存在

### Step 4: 回归原始资料

- 针对盲区重新学习：回到教材、论文、源码，专门攻克卡壳点
- 多角度理解：寻找不同的解释方式（数学公式、代码实现、可视化、历史背景）
- 追根溯源：理解概念的来源和演化（谁提出的？解决什么问题？）
- 陷阱：只记忆了新的表述，但没有真正理解（换了一种说法复述）

### Step 5: 简化和类比

- 压缩解释：将解释压缩到最短，去除所有非必要细节
- 创造精准类比：找到一个既简单又准确的类比（如"区块链像一本无法篡改的公共账本"）
- 使用故事：将概念嵌入一个叙事中（"想象一下，你是一个邮递员..."）
- 测试类比边界：类比在哪些地方失效？（如"管道像队列"——但管道是字节流，队列是消息）
- 陷阱：为了简化而牺牲准确性，导致"美丽的错误"（如"电子围绕原子核像行星围绕太阳"）

### Step 6: 教学与验证

- 向真实的人解释：找到听众（同事、朋友、论坛），实际讲解
- 回答问题：听众的提问会暴露你未考虑到的角度
- 迭代改进：根据反馈修正解释，填补新发现的盲区
- 应用检验：尝试用所学解决实际问题，验证理解深度
- 陷阱：只向自己解释（或假想听众），缺乏真实反馈

### Step 7: 总结输出

- 整合前面各步的分析结果，形成最终简化解释
- 不遗漏核心概念、关键类比和边界条件
- 明确下一步应该调用哪个 Skill（如需要深入数学推导则调用 first-principles-thinking）
- 输出应包含：概念定义 + 简化解释 + 核心类比 + 盲区记录 + 应用场景 + 下一步

---

## 关键原则

1. **简单是理解的试金石**：能用简单语言解释才是真理解，专业术语往往掩盖理解不足。原因：术语是压缩包，如果你不能解压，说明你并不拥有里面的内容。
2. **教学是最好的学习**：教别人时暴露的盲区，是自己学习时难以发现的。原因：输出倒逼输入，教学要求逻辑完整性和一致性。
3. **类比是理解的桥梁**：好的类比连接新旧知识，但要清楚其边界。原因：类比帮助快速建立直觉，但所有类比都有局限性。
4. **迭代优于一次性**：理解是螺旋上升的，每次迭代都会发现新的盲区。原因：知识有层次，表层理解后才会发现深层问题。

## 自检清单

**执行规则**：
- **执行方**：Agent 自检，在最终输出前逐项确认
- **未通过处理**：任一检查项未通过，必须修正后重新自检，禁止跳过
- **记录方式**：自检结果写入 `self_check_result.md`（临时文件），包含：通过项、未通过项、修正记录
- **强制停止项**：以下检查项未通过时，必须停止输出并修正：
  - [ ] 已按思考流程完成全部步骤，无跳过（或已标注提前终止原因）
  - [ ] 输出具体可执行，不含模糊建议
  - [ ] 标注了关键假设和不确定性

**完整检查项**：

- [ ] 已按思考流程完成全部步骤，无跳过（或已标注提前终止原因）
- [ ] 每步思考要点已覆盖，无遗漏
- [ ] 每步结论标注了置信度和反证可能性
- [ ] 考虑了二阶/三阶效应（如果适用）
- [ ] 输出具体可执行，不含模糊建议（如"优化一下"）
- [ ] 标注了关键假设和不确定性
- [ ] 识别了最大风险点和缓解措施
- [ ] 建议按优先级排序，不是平铺罗列
- [ ] 明确了下一步应该调用哪个 Skill（如果需要）

**费曼学习法特有关键检查项**：

- [ ] **费曼特供**：是否限定了概念范围？目标概念是否足够具体（非整个领域）？是否写在白纸顶部？
- [ ] **费曼特供**：是否使用了简单语言？解释是否避免了专业术语？12岁孩子是否能听懂？
- [ ] **费曼特供**：是否识别了知识盲区？卡壳点、模糊地带、逻辑断层、术语依赖是否被记录？
- [ ] **费曼特供**：是否回归了原始资料？盲区是否通过教材、论文、源码重新学习？是否多角度理解？
- [ ] **费曼特供**：是否创造了精准类比？类比是否既简单又准确？类比的边界（失效点）是否被识别？
- [ ] **费曼特供**：是否进行了教学验证？是否向真实听众解释？听众的提问是否被回答？是否应用检验？
