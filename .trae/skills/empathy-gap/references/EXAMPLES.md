# 分析示例

## 示例 1：共情差距——产品可用性测试

**输入**：这个产品界面我觉得很简单，用户应该能轻松上手，不需要做可用性测试。

```
Step 1: 识别共情信号 → 产出：发现"我觉得简单...用户应该能轻松"表述，用当前知识状态预测他人体验
Step 2: 评估共情差距 → 产出：开发者熟悉产品逻辑，用户是首次接触，认知负荷差异大
Step 3: 分析共情机制 → 产出：知识诅咒：知道后无法想象不知道的状态；冷状态：当前平静状态无法预测用户困惑时的感受
Step 4: 校正共情差距 → 产出：建议用真实用户测试，而非自我评估
```

**输出**：
```
## 分析结论
### 核心发现
1. 开发者熟悉产品逻辑，用户首次接触的认知负荷完全不同
2. 历史数据：开发者认为简单的功能，用户平均需要3次尝试才能掌握
3. 共情差距使无法准确预测用户的困惑和挫折感

### 建议（按优先级）
1. [高优先级] 进行可用性测试，观察真实用户操作
2. [高优先级] 记录用户首次使用的困惑点和错误
3. [中优先级] 基于用户反馈优化界面，而非开发者直觉

### 关键假设
- 用户群体技术水平多样
- 首次使用体验影响留存

### 风险
- 共情差距可能导致产品难用
- 需要真实用户数据验证
```

## 示例 2：共情差距——情绪状态决策

**输入**：我现在很冷静，觉得这个投资很理性，明天就执行。

```
Step 1: 识别共情信号 → 产出：发现"我很冷静...很理性"表述，用当前冷静状态预测未来决策
Step 2: 评估共情差距 → 产出：当前冷静状态≠市场波动时的情绪状态，情绪会影响决策
Step 3: 分析共情机制 → 产出：冷热共情差距：冷静时无法准确预测情绪状态下的行为；状态依赖：不同情绪状态有不同偏好
Step 4: 校正共情差距 → 产出：建议预设决策规则，避免情绪状态影响
```

**输出**：
```
## 分析结论
### 核心发现
1. 当前冷静状态下的决策，可能在市场波动时被情绪推翻
2. Loewenstein(1996)证明：冷静时无法准确预测饥饿、愤怒、恐惧时的行为
3. 市场下跌10%时的恐慌情绪可能导致非理性卖出

### 建议（按优先级）
1. [高优先级] 预设投资决策规则（如止损线、买入条件）
2. [高优先级] 书面记录当前决策逻辑，情绪波动时回顾
3. [中优先级] 设置冷静期，情绪激动时不做大决策

### 关键假设
- 市场波动会触发情绪反应
- 预设规则可降低情绪影响

### 风险
- 共情差距可能导致情绪化决策
- 需要机制约束而非依赖意志力
```
