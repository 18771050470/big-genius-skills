# 动效/交互设计师常见陷阱

## 1. 动画无目的

### 反例：装饰动画
- **错误**：添加没有用户价值的纯装饰动画
- **案例**：页面加载时所有元素都弹跳进入，分散用户注意力
- **原因**：为了动画而动画，忽视了用户体验

### 正例： purposeful 动画
- **正确**：每个动画都有明确的用户价值
- **案例**：按钮点击时轻微缩放反馈，让用户知道操作被接收
- **要点**：动画是工具，不是装饰

## 2. 时长不当

### 反例：过快或过慢
- **错误**：动画时长不合适，要么太快用户感知不到，要么太慢让用户等待
- **案例**：微交互用 1 秒，用户感觉拖沓；页面转场用 100ms，用户感觉突兀
- **原因**：没有遵循动画时长的最佳实践

### 正例：合适的时长
- **正确**：微交互 200-400ms，页面转场 300-500ms，复杂动画 < 1s
- **案例**：按钮反馈 250ms，页面切换 400ms，加载动画循环 2s
- **要点**：时长影响用户对速度和流畅度的感知

## 3. 性能问题

### 反例：卡顿动画
- **错误**：动画导致页面卡顿，影响用户体验
- **案例**：使用 height/width 动画触发重排，导致帧率下降到 20fps
- **原因**：没有考虑动画的性能影响

### 正例：性能优化
- **正确**：使用 transform 和 opacity，避免触发重排
- **案例**：用 transform: translateX() 替代 left/right，用 opacity 替代 display
- **要点**：60fps 是动画流畅的底线

## 4. 忽视可访问性

### 反例：强制动画
- **错误**：不考虑用户的 motion sensitivity，强制播放动画
- **案例**：用户设置了减少动画偏好，但产品仍播放大量动画
- **原因**：忽视了 prefers-reduced-motion 设置

### 正例：尊重用户偏好
- **正确**：支持 prefers-reduced-motion，提供静态替代
- **案例**：检测到用户偏好减少动画时，禁用非必要动画，保持功能完整
- **要点**：动画不应引起不适或眩晕

## 5. 过度复杂

### 反例：炫技动画
- **错误**：动画过于复杂，反而干扰用户操作
- **案例**：页面切换使用 3D 翻转效果，用户等待且迷失方向
- **原因**：追求视觉效果，忽视了简洁性

### 正例：简洁有效
- **正确**：动画简洁明了，不干扰用户任务
- **案例**：页面切换用简单的淡入淡出，用户关注点始终在内容上
- **要点**：最好的动画是用户几乎注意不到的动画
