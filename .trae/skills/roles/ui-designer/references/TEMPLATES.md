# 技术交付物模板

> 提供可直接使用的 Markdown/代码模板。

## 交付物模板 1：设计规范文档

```markdown
# 设计规范: [产品/系统名称]

## 1. 设计原则

- **简洁**：每页只呈现一个核心任务
- **一致**：相同元素在不同页面表现一致
- **反馈**：每个操作都有即时响应
- **容错**：允许撤销，防止误操作

## 2. 色彩系统

### 主色
| 名称 | 色值 | 用途 |
|------|------|------|
| Primary | #1890FF | 主按钮、链接、重点 |
| Primary Hover | #40A9FF | 悬停状态 |
| Primary Active | #096DD9 | 点击状态 |

### 功能色
| 名称 | 色值 | 用途 |
|------|------|------|
| Success | #52C41A | 成功状态、通过 |
| Warning | #FAAD14 | 警告、提示 |
| Error | #F5222D | 错误、失败、删除 |
| Info | #1890FF | 信息提示 |

### 中性色
| 名称 | 色值 | 用途 |
|------|------|------|
| Text Primary | #262626 | 主文本 |
| Text Secondary | #595959 | 次要文本 |
| Text Disabled | #BFBFBF | 禁用文本 |
| Border | #D9D9D9 | 边框、分割线 |
| Background | #F5F5F5 | 页面背景 |

## 3. 字体规范

| 层级 | 大小 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 24px | Bold | 1.4 | 页面标题 |
| H2 | 20px | Bold | 1.4 | 模块标题 |
| H3 | 16px | Medium | 1.5 | 卡片标题 |
| Body | 14px | Regular | 1.6 | 正文 |
| Caption | 12px | Regular | 1.5 | 辅助说明 |

## 4. 间距系统

基于 8px 网格：
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- xxl: 48px

## 5. 圆角规范

| 元素 | 圆角 |
|------|------|
| 按钮 | 4px |
| 输入框 | 4px |
| 卡片 | 8px |
| 弹窗 | 8px |
| 标签 | 2px |

## 6. 阴影规范

| 层级 | 阴影值 | 用途 |
|------|--------|------|
| Shadow 1 | 0 1px 2px rgba(0,0,0,0.05) | 卡片默认 |
| Shadow 2 | 0 4px 12px rgba(0,0,0,0.1) | 悬停、下拉 |
| Shadow 3 | 0 8px 24px rgba(0,0,0,0.12) | 弹窗、浮层 |
```

## 交付物模板 3：页面设计说明

```markdown
# 页面设计: [页面名称]

## 页面信息
- **页面名称**：[中文名]
- **页面路径**：[URL 路径]
- **页面类型**：列表页 / 详情页 / 表单页 / 仪表盘
- **权限要求**：[公开 / 登录 / 特定角色]

## 布局结构

```
[Header: 高度 64px，固定顶部]
  ├── Logo
  ├── 导航菜单
  └── 用户头像

[Main Content: 最大宽度 1200px，居中]
  ├── [模块 A: 占 2/3 宽度]
  │   ├── [子模块 A1]
  │   └── [子模块 A2]
  └── [模块 B: 占 1/3 宽度]
      ├── [子模块 B1]
      └── [子模块 B2]

[Footer: 高度 48px]
```

## 模块详解

### 模块 A: [模块名称]
- **位置**：[布局位置]
- **尺寸**：[宽 × 高]
- **内容**：[描述]
- **交互**：[描述用户操作和系统反馈]
- **数据**：[描述数据来源和格式]

### 模块 B: [模块名称]
...

## 交互说明

| 操作 | 触发 | 反馈 | 跳转 |
|------|------|------|------|
| 点击按钮 X | 鼠标点击 | 按钮加载态 | 页面 Y |
| 输入搜索 | 键盘输入 | 实时搜索建议 | 无 |

## 空状态
- **场景**：[描述何时出现空状态]
- **展示**：[空状态的视觉表现]
- **引导**：[如何引导用户离开空状态]

## 异常状态
- **加载失败**：错误提示 + 重试按钮
- **权限不足**：提示信息 + 引导升级
```

## 交付物模板 4：设计评审清单

```markdown
# 设计评审: [设计稿/页面名称]

## 总体印象
[🟢 通过 / 🟡 有条件通过 / 🔴 需要修改]

## 视觉设计
- [ ] 色彩使用符合规范
- [ ] 字体层级清晰
- [ ] 间距统一（8px 网格）
- [ ] 圆角使用一致
- [ ] 阴影层级合理

## 交互设计
- [ ] 操作流程完整（主流程 + 异常流程）
- [ ] 每个操作都有反馈
- [ ] 加载状态已设计
- [ ] 空状态已设计
- [ ] 错误状态已设计

## 可访问性
- [ ] 颜色对比度 ≥ 4.5:1
- [ ] 触控目标 ≥ 44×44px
- [ ] 焦点状态可见
- [ ] 支持键盘导航

## 一致性
- [ ] 与现有设计系统一致
- [ ] 组件使用符合规范
- [ ] 图标语义明确

## 关键问题（🔴 Blocker）
- [ ] **问题1**：[位置] — [问题描述] — [建议修改]

## 改进建议（🟡 Suggestion）
- [ ] **建议1**：[位置] — [描述] — [理由]

## 表扬（💭 Nit）
- ✅ [位置] — [好的实践] — [为什么好]
```
