# 数据分析师常见陷阱

## 1. 相关当因果

### 反例：因果混淆
- **错误**：将相关性误认为因果性，做出错误决策
- **案例**：发现"冰淇淋销量"和"溺水事故"高度相关，就建议限制冰淇淋销售
- **原因**：忽视了第三方变量（气温）的影响

### 正例：因果推断
- **正确**：使用实验或因果推断方法验证因果关系
- **案例**：通过 A/B 测试验证新功能对留存的影响，控制其他变量
- **要点**：相关不等于因果，需要更严格的证据

## 2. 采样偏差

### 反例：样本不具代表性
- **错误**：基于有偏样本得出结论，推广到整体
- **案例**：仅分析付费用户的反馈，就推断所有用户的需求
- **原因**：忽视了免费用户和流失用户的声音

### 正例：随机采样
- **正确**：确保样本随机且具代表性
- **案例**：从所有用户中随机抽取样本进行分析，确保各群体比例一致
- **要点**：样本质量决定结论质量

## 3. 过度拟合

### 反例：模型复杂度过高
- **错误**：为了拟合历史数据而构建过度复杂的模型
- **案例**：用 20 个变量的模型拟合 30 个数据点，预测效果极差
- **原因**：忽视了模型的泛化能力

### 正例：简洁模型
- **正确**：使用简洁的模型，优先可解释性
- **案例**：用 3-5 个关键变量的线性模型，虽然拟合度略低但预测稳定
- **要点**：模型是为了预测，不是为了拟合历史

## 4. 可视化误导

### 反例：图表陷阱
- **错误**：使用误导性的图表呈现数据
- **案例**：Y 轴不从 0 开始，使 5% 的增长看起来像 50%
- **原因**：有意或无意地扭曲了数据呈现

### 正例：诚实可视化
- **正确**：使用恰当的图表，准确呈现数据
- **案例**：Y 轴从 0 开始，标注数据来源，使用合适的图表类型
- **要点**：可视化是为了清晰传达，不是为了夸大

## 5. 忽视数据质量

### 反例：脏数据直接分析
- **错误**：不对数据进行清洗和验证，直接进行分析
- **案例**：分析中发现异常值，但没有调查原因，导致结论严重偏差
- **原因**：忽视了数据质量对分析结果的影响

### 正例：数据质量优先
- **正确**：分析前先进行数据质量检查
- **案例**：建立数据质量检查清单，包括完整性、一致性、准确性、时效性
- **要点**：垃圾进，垃圾出，数据质量是分析的基础
