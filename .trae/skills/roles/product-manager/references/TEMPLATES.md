# 技术交付物模板

> 提供可直接使用的 Markdown/代码模板。

## 交付物模板 1：产品需求文档（PRD）

```markdown
# PRD: [功能/产品名称]

## 1. 背景与目标

### 1.1 背景
[描述做这个功能的背景原因，用户痛点或业务机会]

### 1.2 目标
- **业务目标**：[如：提升转化率 10%]
- **用户目标**：[如：让用户在 30 秒内完成下单]
- **成功指标**：
  - 指标 1：[具体数值]
  - 指标 2：[具体数值]

## 2. 用户故事

### 2.1 目标用户
- **用户角色**：[如：电商平台买家]
- **用户画像**：[年龄/职业/使用场景]

### 2.2 用户故事列表

#### 故事 1: [故事标题]
**作为** [用户角色]，
**我想要** [功能/能力]，
**以便** [达成的目标]。

**验收标准**：
- [ ] Given [前置条件], When [操作], Then [结果]
- [ ] Given [前置条件], When [操作], Then [结果]

**优先级**：P0/P1/P2
**故事点**：[X 点]

## 3. 功能需求

### 3.1 功能清单

| 功能模块 | 功能点 | 优先级 | 故事点 | 备注 |
|---------|--------|--------|--------|------|
| 模块 A | 功能 1 | P0 | 3 | |
| 模块 A | 功能 2 | P1 | 5 | |

### 3.2 功能流程

```
[步骤 1] → [步骤 2] → [步骤 3]
   ↓
[异常分支 1] → [错误处理]
```

## 4. 非功能需求

- **性能**：页面加载 < 2s，API 响应 < 200ms
- **兼容性**：支持 Chrome/Firefox/Safari 最新 2 个版本
- **安全**：敏感操作需二次确认
- **可访问性**：符合 WCAG 2.1 AA 标准

## 5. 数据需求

| 数据项 | 来源 | 展示方式 | 更新频率 |
|--------|------|---------|---------|
| 数据 1 | 数据库 | 列表 | 实时 |
| 数据 2 | 第三方 API | 卡片 | 每小时 |

## 6. 发布计划

- **MVP 范围**：[功能清单]
- **发布时间**：[日期]
- **灰度策略**：[如：先开放 10% 用户]

## 7. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 风险 1 | 高/中/低 | 高/中/低 | [措施] |

## 8. 附录

- [原型链接]
- [竞品分析]
- [数据支撑]
```

## 交付物模板 2：用户故事卡片

```markdown
# 用户故事: [故事标题]

## 基本信息
- **故事 ID**: US-XXX
- **优先级**: P0/P1/P2
- **故事点**: X
- **迭代**: Sprint X

## 用户故事
**作为** [用户角色]，
**我想要** [功能/能力]，
**以便** [达成的目标]。

## 验收标准（AC）

### AC1: [场景描述]
**Given** [前置条件]
**When** [用户操作]
**Then** [系统响应]

### AC2: [场景描述]
**Given** [前置条件]
**When** [用户操作]
**Then** [系统响应]

## 边界情况
- [ ] 空状态：[描述]
- [ ] 异常状态：[描述]
- [ ] 极限状态：[描述]

## 依赖
- [依赖故事 ID]: [说明]

## 设计稿
- [链接或描述]

## 备注
- [其他需要说明的信息]
```

## 交付物模板 3：产品路线图

```markdown
# 产品路线图: [产品名称]

## 愿景
[一句话描述产品的长期愿景]

## 时间轴

### Q1 202X（当前季度）
**主题**：[如：夯实基础]
**目标**：[1-3 个可衡量的目标]

| 功能 | 优先级 | 状态 | 负责人 | 发布日期 |
|------|--------|------|--------|---------|
| 功能 A | P0 | 开发中 | [姓名] | 1月15日 |
| 功能 B | P1 | 设计中 | [姓名] | 2月1日 |

**关键指标**：
- 指标 1：从 X 提升到 Y
- 指标 2：达到 Z

### Q2 202X
**主题**：[如：增长实验]
**目标**：...

### Q3 202X
**主题**：[如：体验优化]
**目标**：...

## 里程碑

```
[MVP 发布] → [V1.0 发布] → [V2.0 发布]
   ↓
[用户量 1K]   [用户量 10K]   [用户量 100K]
```

## 资源分配
- **研发团队**：X 人
- **设计团队**：X 人
- **预算**：X 万元

## 风险与应对
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| [风险] | 高/中/低 | [措施] |
```

## 交付物模板 4：竞品分析报告

```markdown
# 竞品分析: [产品/功能领域]

## 分析目标
[说明这次竞品分析的目的，如：寻找差异化机会、评估市场进入策略]

## 竞品选择

| 竞品 | 类型 | 定位 | 选择理由 |
|------|------|------|---------|
| 竞品 A | 直接竞品 | [定位] | [理由] |
| 竞品 B | 间接竞品 | [定位] | [理由] |
| 竞品 C | 参考对象 | [定位] | [理由] |

## 功能对比

| 功能维度 | 我方产品 | 竞品 A | 竞品 B | 竞品 C |
|---------|---------|--------|--------|--------|
| 功能 1 | ✓/✗/△ | ✓/✗/△ | ✓/✗/△ | ✓/✗/△ |
| 功能 2 | ✓/✗/△ | ✓/✗/△ | ✓/✗/△ | ✓/✗/△ |

✓ 支持 ✗ 不支持 △ 部分支持

## 优劣势分析

### 我方优势
1. [优势 1]
2. [优势 2]

### 我方劣势
1. [劣势 1]
2. [劣势 2]

## 机会与威胁

### 机会（Opportunities）
- [机会 1：竞品未满足的需求]
- [机会 2：市场趋势]

### 威胁（Threats）
- [威胁 1：竞品即将推出的功能]
- [威胁 2：市场变化]

## 差异化策略
[基于分析结果，提出我方的差异化定位和策略]

## 行动建议
1. [短期行动]
2. [中期行动]
3. [长期行动]
```
