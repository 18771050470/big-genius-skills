# Gotchas

> 只写 Agent 容易犯错的**非显而易见的事实**。通用知识不要写。
> 每个 Gotcha 必须包含：问题描述 + ❌ Before（错误做法） + ✅ After（正确做法） + 原因解释。
> **筛选标准**：只有满足以下条件的坑才值得写：
> 1. 非显而易见（不是"要接退耦电容"这种常识）
> 2. 有实际危害（会导致硬件损坏、性能问题或生产失败）
> 3. 有明确规避方法（不能只是"小心点"）

## Gotcha 1：只关注芯片单价，忽略外围电路成本

**问题**：选型时只看MCU单价便宜，结果需要复杂外围电路（额外电源、电平转换、晶振），总成本反而更高。

❌ **Before（错误做法）**
```markdown
# 选型对比
| 器件 | 单价 | 选择 |
|------|------|------|
| MCU A | $1.5 | ✅ 便宜 |
| MCU B | $2.5 | ❌ 贵 |

# 结果
- MCU A 需要外部晶振 ($0.3)、外部Flash ($0.5)、电平转换 ($0.4)
- 总成本：$1.5 + $0.3 + $0.5 + $0.4 = $2.7
```

✅ **After（正确做法）**
```markdown
# 选型对比（总拥有成本）
| 器件 | 芯片单价 | 外围成本 | 总成本 | 选择 |
|------|---------|---------|--------|------|
| MCU A | $1.5 | $1.2 | $2.7 | ❌ |
| MCU B | $2.5 | $0.2 | $2.7 | ✅ 内置晶振/Flash |

# 评估维度
- 芯片单价、外围器件数量、PCB面积、功耗、开发时间
```

**原因**：硬件设计的总拥有成本（TCO）包括芯片、外围电路、PCB面积、功耗、开发时间。便宜芯片可能需要更多外围，反而增加总成本和复杂度。

## Gotcha 2：电源芯片额定电流刚好等于负载电流

**问题**：电源芯片长期满负荷运行，导致发热严重、效率下降、寿命缩短，甚至触发过温保护。

❌ **Before（错误做法）**
```markdown
# 电源设计
- 负载电流：500mA
- 选择LDO：额定500mA
- 结果：LDO温度升高40°C，效率仅60%，夏天触发过温保护
```

✅ **After（正确做法）**
```markdown
# 电源设计
- 负载电流：500mA
- 选择LDO：额定700mA（余量40%）
- 或选择DCDC：额定1A，效率90%
- 结果：温升 < 20°C，稳定运行

# 余量规则
- 持续负载：额定电流 > 实际电流 × 1.3
- 峰值负载：额定电流 > 峰值电流 × 1.2
- 高温环境：额定电流 > 实际电流 × 1.5（考虑降额）
```

**原因**：电源芯片的额定电流是在理想条件下的最大值。实际应用中需考虑温度降额、老化、输入电压波动等因素，必须留有余量。

## Gotcha 3：忽略器件的生命周期状态

**问题**：选用了NRND（不推荐用于新设计）或EOL（停产）器件，量产时无法采购，被迫重新设计。

❌ **Before（错误做法）**
```markdown
# 选型
- 选择某款经典MCU，已使用10年
- 未查生命周期状态
- 量产时发现已宣布EOL，最后采购日期仅剩3个月
```

✅ **After（正确做法）**
```markdown
# 选型
- 查询器件生命周期：Active ✅ / NRND ⚠️ / EOL ❌
- 优先选择Active状态，且有至少5年供货保障的器件
- 对NRND器件，要求供应商提供书面供货承诺
- 建立器件生命周期监控机制，每季度检查一次

# 生命周期检查清单
- [ ] 器件状态：Active
- [ ] 供货保障：≥ 5年
- [ ] 替代料：已确认且可采购
- [ ] 供应商健康度：无破产/并购风险
```

**原因**：电子元器件生命周期短，特别是消费级芯片。忽略生命周期状态会导致供应链断裂，重新设计的成本远超器件本身。

## Gotcha 4：高速信号不做阻抗控制

**问题**：USB、HDMI、Ethernet等高速信号未做阻抗控制，导致信号反射、眼图闭合、通信失败。

❌ **Before（错误做法）**
```markdown
# USB 2.0 布线
- D+/D- 线宽：0.2mm（随意设置）
- 差分对间距：0.3mm（未优化）
- 结果：USB识别不稳定，大文件传输报错
```

✅ **After（正确做法）**
```markdown
# USB 2.0 布线
- D+/D- 差分阻抗：90Ω ± 10%
- 线宽/间距：根据层叠结构计算（如0.15mm/0.15mm）
- 等长控制：D+与D-长度差 < 50mil
- 过孔：最少化，必要时对称放置
- 结果：USB稳定运行，通过眼图测试

# 常见高速信号阻抗
| 信号 | 阻抗 | 公差 |
|------|------|------|
| USB 2.0 | 90Ω 差分 | ±10% |
| USB 3.0 | 90Ω 差分 | ±7% |
| HDMI | 100Ω 差分 | ±10% |
| Ethernet | 100Ω 差分 | ±10% |
| PCIe | 85Ω 差分 | ±5% |
```

**原因**：高速信号的波长与PCB走线长度可比拟，阻抗不连续会导致信号反射。阻抗控制是高速信号稳定传输的基础，必须在PCB设计阶段严格计算和验证。

## Gotcha 5：3.3V MCU直接连接5V传感器

**问题**：将5V传感器的输出直接接到3.3V MCU的GPIO，导致MCU引脚过压损坏。

❌ **Before（错误做法）**
```markdown
# 连接
5V传感器输出 ──────→ 3.3V MCU GPIO
                    （无电平转换）

# 结果
- MCU GPIO钳位二极管导通
- 长期过压导致GPIO损坏
- 可能波及整个MCU
```

✅ **After（正确做法）**
```markdown
# 方案1：电平转换芯片
5V传感器输出 ──→ TXB0108 ──→ 3.3V MCU GPIO

# 方案2：分压电阻（单向）
5V传感器输出 ──→ 10KΩ ──┬──→ 3.3V MCU GPIO
              20KΩ ──┘

# 方案3：开漏输出+上拉
5V传感器（开漏）──→ 3.3V上拉 ──→ 3.3V MCU GPIO

# 选择依据
- 双向通信：用电平转换芯片
- 单向输入（5V→3.3V）：用分压电阻
- 单向输出（3.3V→5V）：用开漏+上拉或电平转换
```

**原因**：MCU的GPIO有绝对最大电压限制（通常为VDD+0.3V）。5V信号直接输入3.3V MCU会超过限制，导致钳位二极管过流、氧化层击穿，最终损坏引脚或整个芯片。
