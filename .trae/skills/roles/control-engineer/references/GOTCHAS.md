# Gotchas

> 只写 Agent 容易犯错的**非显而易见的事实**。通用知识不要写。
> 每个 Gotcha 必须包含：问题描述 + ❌ Before（错误做法） + ✅ After（正确做法） + 原因解释。

## Gotcha 1：PID参数过大导致系统振荡

**问题**：PID参数（尤其是Kp和Ki）设置过大，导致系统振荡、电机发热、机械损坏。

❌ **Before（错误做法）**
```markdown
# PID参数整定
- 直接设置大参数：Kp=100, Ki=50, Kd=10
- 无仿真验证，直接上机
- 结果：系统剧烈振荡，电机过热，机械结构损坏
```

✅ **After（正确做法）**
```markdown
# PID参数整定（逐步法）
1. 先设Ki=0, Kd=0，从小Kp开始（如Kp=1）
2. 逐步增大Kp，直到系统出现轻微振荡
3. 记录临界增益Ku和振荡周期Tu
4. 用Ziegler-Nichols公式计算初始参数：
   - Kp = 0.6 × Ku
   - Ki = 2 × Kp / Tu
   - Kd = Kp × Tu / 8
5. 仿真验证稳定性（相位裕度 > 30°）
6. 上机微调

# 安全保护
- 输出限幅：±100%（防止执行器饱和）
- 积分限幅：防止积分饱和
- 急停按钮：紧急情况下立即停止
```

**原因**：PID参数过大导致闭环极点进入右半平面，系统不稳定。必须先仿真验证稳定性，再逐步增大参数。

## Gotcha 2：忽略电机反电动势

**问题**：设计电机驱动时忽略反电动势（Back-EMF），导致高速时驱动电压不足，电流下降，转矩降低。

❌ **Before（错误做法）**
```markdown
# 电机驱动设计
- 电源电压：12V
- 电机额定电压：12V
- 忽略反电动势
- 结果：电机转速达到50%额定转速时，反电动势6V，驱动电压仅6V，电流下降50%，转矩不足
```

✅ **After（正确做法）**
```markdown
# 电机驱动设计（考虑反电动势）
- 电机额定电压：12V
- 电机反电动势常数：Ke = 0.05 V/rpm
- 额定转速：2000 rpm
- 额定转速时反电动势：2000 × 0.05 = 100V（注意：这是假设值，实际应根据电机参数计算）
- 修正：实际应根据电机参数重新计算

# 正确做法
- 计算最大反电动势：Ke × 最大转速
- 驱动电压 > 反电动势 + I×R（电枢电阻压降）+ 裕量
- 例如：反电动势10V，I×R=2V，裕量5V → 驱动电压 ≥ 17V
- 选择24V驱动电源
```

**原因**：电机旋转时产生反电动势，与转速成正比。高速时反电动势接近电源电压，导致驱动电流不足，转矩下降。必须确保电源电压有足够裕量克服反电动势。

## Gotcha 3：采样周期过大导致离散化误差

**问题**：数字控制系统采样周期选择过大，导致离散化误差大、控制性能差、甚至不稳定。

❌ **Before（错误做法）**
```markdown
# 控制系统设计
- 被控对象时间常数：100ms
- 采样周期：50ms
- 结果：采样周期 = 时间常数/2，离散化误差大，控制性能差
```

✅ **After（正确做法）**
```markdown
# 控制系统设计
- 被控对象时间常数：100ms
- 采样周期：10ms（时间常数/10）
- 结果：离散化误差小，控制性能接近连续系统

# 采样周期选择规则
- 快速系统（时间常数 < 10ms）：采样周期 0.1-1ms
- 中速系统（时间常数 10-100ms）：采样周期 1-10ms
- 慢速系统（时间常数 > 100ms）：采样周期 10-100ms
- 一般规则：采样周期 ≤ 时间常数 / 10
```

**原因**：采样定理要求采样频率 > 2倍信号带宽。工程上通常取10倍以上，以确保离散化误差小、控制性能接近连续系统。

## Gotcha 4：无位置限位保护

**问题**：位置控制系统无软限位和硬限位保护，导致机械超程、设备损坏。

❌ **Before（错误做法）**
```markdown
# 位置控制
- 目标位置：由上位机任意指定
- 无限位保护
- 结果：目标位置超出机械范围，电机继续驱动，机械结构损坏
```

✅ **After（正确做法）**
```markdown
# 位置控制（带限位保护）
- 软限位（软件）：
  - 最小位置：-100mm
  - 最大位置：+100mm
  - 到达限位前10mm开始减速
  - 超出限位时停止运动并报警

- 硬限位（硬件）：
  - 限位开关：-110mm和+110mm
  - 触发时立即切断电机电源
  - 需手动复位后才能恢复

- 急停：
  - 急停按钮：任意时刻立即停止
  - 故障检测：过流、过温、编码器异常时自动停止
```

**原因**：位置控制系统必须有双重保护。软限位在软件层面预防超程，硬限位在硬件层面提供最终保护。无限位保护会导致机械损坏和安全事故。

## Gotcha 5：忽略积分饱和

**问题**：PID控制器在输出饱和时继续积分，导致系统退出饱和时产生巨大超调。

❌ **Before（错误做法）**
```markdown
# PID实现
- 标准PID公式：u = Kp×e + Ki×∫e + Kd×de/dt
- 输出限幅：±100%
- 问题：当误差持续存在时，积分项不断累积
- 退出饱和时：积分项巨大，导致严重超调
```

✅ **After（正确做法）**
```markdown
# PID实现（抗积分饱和）
- 条件积分：输出饱和时停止积分
  ```c
  if (output > MAX) {
      output = MAX;
      if (error > 0) error = 0;  // 正饱和时，正误差不积分
  } else if (output < MIN) {
      output = MIN;
      if (error < 0) error = 0;  // 负饱和时，负误差不积分
  }
  ```
- 积分限幅：限制积分项的最大值
  ```c
  integral = constrain(integral, -INTEGRAL_MAX, INTEGRAL_MAX);
  ```
- 反计算法：根据实际输出反算积分项
  ```c
  integral = (actual_output - Kp*error - Kd*derivative) / Ki;
  ```

# 效果
- 无抗饱和：超调50%，恢复时间5s
- 有抗饱和：超调10%，恢复时间1s
```

**原因**：积分饱和是PID控制中最常见的问题。当执行器饱和时，积分项继续累积，导致系统退出饱和时产生巨大超调。抗积分饱和是PID实现的必备功能。
