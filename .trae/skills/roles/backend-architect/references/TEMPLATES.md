# 技术交付物模板

> 提供可直接使用的 Markdown/代码模板。

## 交付物模板 1：API 接口规范

```markdown
# API 规范: [模块名称]

## 基础信息
- **协议**：HTTP/HTTPS
- **格式**：JSON
- **编码**：UTF-8
- **版本**：v1（通过 URL 路径 /api/v1/...）

## 认证方式
- **类型**：Bearer Token / API Key / OAuth2
- **位置**：Header: `Authorization: Bearer <token>`

## 通用响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1699999999,
  "request_id": "uuid-string"
}
```

## 错误码定义

| 错误码 | 说明 | 处理建议 |
|--------|------|---------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数格式和必填项 |
| 401 | 未认证 | 检查 Token 是否有效 |
| 403 | 无权限 | 检查用户权限配置 |
| 404 | 资源不存在 | 检查资源 ID 是否正确 |
| 429 | 请求过于频繁 | 降低请求频率，实现退避策略 |
| 500 | 服务器内部错误 | 联系后端开发人员 |

## 接口列表

### 1. [接口名称]

- **URL**：`/api/v1/[path]`
- **Method**：GET/POST/PUT/DELETE
- **描述**：一句话描述接口功能

**请求参数**：

| 参数 | 类型 | 位置 | 必填 | 说明 |
|------|------|------|------|------|
| param1 | string | query/body | 是 | 参数说明 |
| param2 | integer | body | 否 | 参数说明 |

**响应示例**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "example"
  }
}
```
```

## 交付物模板 2：数据库设计文档

```markdown
# 数据库设计: [系统名称]

## 命名规范
- **表名**：小写 + 下划线，复数形式（users, order_items）
- **字段名**：小写 + 下划线
- **索引名**：idx_表名_字段名
- **外键名**：fk_表名_关联表名

## 表结构

### 表名: [表名]

| 字段 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| id | BIGINT | - | 是 | 自增 | 主键 |
| created_at | TIMESTAMP | - | 是 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | - | 是 | CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | TIMESTAMP | - | 否 | NULL | 软删除标记 |

**索引**：
- PRIMARY KEY (`id`)
- INDEX `idx_created_at` (`created_at`)

**外键**：
- FOREIGN KEY (`user_id`) REFERENCES `users` (`id`)

## ER 关系

```
[表A] 1:N [表B]
[表C] N:M [表D] (通过关联表)
```

## 分库分表策略
- **分片键**：[字段名]
- **分片算法**：[hash/range]
- **分片数量**：[数量]
```
