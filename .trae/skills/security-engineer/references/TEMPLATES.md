# 技术交付物模板

## 交付物模板 1：安全评估报告

```markdown
# 安全评估报告：[系统名称]

## 评估信息
- **评估时间**：[起止日期]
- **评估范围**：[范围描述]
- **评估方法**：自动化扫描 + 手动渗透测试 + 代码审计
- **评估人员**：[姓名]

## 风险评级标准
| 等级 | CVSS 分数 | 说明 |
|------|----------|------|
| 🔴 严重 | 9.0-10.0 | 可远程获取系统完全控制权 |
| 🟡 高危 | 7.0-8.9 | 可获取敏感数据或部分控制权 |
| 💭 中等 | 4.0-6.9 | 可利用但影响有限 |
| 低危 | 0.1-3.9 | 风险较低 |

## 漏洞汇总
| 等级 | 数量 | 已修复 | 待修复 |
|------|------|--------|--------|
| 🔴 严重 | [N] | [N] | [N] |
| 🟡 高危 | [N] | [N] | [N] |
| 💭 中等 | [N] | [N] | [N] |
| 低危 | [N] | [N] | [N] |

## 详细漏洞

### VUL-001：[漏洞标题]
- **等级**：[等级]
- **CVSS**：[分数]
- **位置**：[URL/文件/组件]
- **描述**：[漏洞描述]
- **复现步骤**：
  1. [步骤 1]
  2. [步骤 2]
- **影响**：[影响描述]
- **修复建议**：[具体修复方案]
- **参考**：[CVE/CWE 链接]

## 整体评估
- **安全等级**：[优秀/良好/一般/差]
- **主要风险**：[描述]
- **改进建议**：[描述]
```

## 交付物模板 2：渗透测试授权书

```markdown
- **目标系统**：[IP/域名/应用名称]

- **测试方法**：[黑盒/白盒/灰盒]
- **允许操作**：[列出允许的操作]

## 签字
- **授权方**：___________ 日期：___________
- **被授权方**：___________ 日期：___________
```

## 交付物模板 3：威胁建模文档（STRIDE）

```markdown
# 威胁建模：[系统名称]

## 系统架构
[架构图/描述]

## 数据流图
[DFD 图]

## 威胁分析

### 组件：[组件名称]

| 威胁类型 | 威胁描述 | 风险等级 | 缓解措施 |
|---------|---------|---------|---------|
| 欺骗 (Spoofing) | 攻击者伪造身份访问 API | 高 | JWT + MFA |
| 篡改 (Tampering) | 修改传输中的订单数据 | 高 | TLS + 签名 |
| 否认 (Repudiation) | 用户否认提交过订单 | 中 | 审计日志 |
| 信息泄露 (Information Disclosure) | 数据库备份泄露 | 高 | 加密 + 访问控制 |
| 拒绝服务 (Denial of Service) | API 被刷导致服务不可用 | 中 | 限流 + CDN |
| 权限提升 (Elevation of Privilege) | 普通用户获取管理员权限 | 高 | RBAC + 校验 |

## 缓解措施跟踪
| 威胁 | 缓解措施 | 状态 | 负责人 |
|------|---------|------|--------|
| [描述] | [措施] | 已实施/进行中/待实施 | [姓名] |
```

## 交付物模板 4：安全事件响应预案

```markdown
# 安全事件响应预案

## 事件分级
| 级别 | 定义 | 响应时间 | 升级路径 |
|------|------|---------|---------|
| P0 | 生产环境被入侵 | 15 分钟 | 立即通知 CISO |
| P1 | 数据泄露确认 | 30 分钟 | 1 小时内通知管理层 |
| P2 | 高危漏洞被利用 | 1 小时 | 安全团队处理 |
| P3 | 安全告警待确认 | 4 小时 | 值班人员处理 |

## 响应流程
1. **发现**：SIEM 告警、用户报告、外部通报
2. **遏制**：隔离受影响系统、阻断攻击路径
3. **根除**：清除恶意软件、修复漏洞、重置凭据
4. **恢复**：系统恢复、数据还原、服务上线
5. **复盘**：时间线重建、根因分析、改进措施

## 联系人
| 角色 | 姓名 | 电话 | 邮箱 |
|------|------|------|------|
| 安全负责人 | [姓名] | [电话] | [邮箱] |
| 技术负责人 | [姓名] | [电话] | [邮箱] |
| 法务 | [姓名] | [电话] | [邮箱] |
```
