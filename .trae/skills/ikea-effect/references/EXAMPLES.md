# 分析示例

## 示例 1：IKEA效应——内部工具开发

**输入**：我们自己开发的这个工具虽然功能简单，但比外面买的成熟产品更好用，因为是我们自己做的。

```
Step 1: 识别IKEA信号 → 产出：发现"我们自己做的...更好用"表述，因投入劳动而高估成果价值
Step 2: 评估价值偏差 → 产出：内部工具功能简单，外部成熟产品功能完善，但劳动投入使内部工具被高估
Step 3: 分析IKEA机制 → 产出：努力正当化：投入劳动使成果被赋予额外价值；自我一致性：承认自己做的不好会认知失调
Step 4: 校正IKEA效应 → 产出：建议客观评估功能和质量，而非基于投入劳动
```

**输出**：
```
## 分析结论
### 核心发现
1. 内部工具功能简单，外部产品功能完善
2. Norton等(2012)实验：自己组装的宜家盒子估值比预组装的高63%
3. 劳动投入使内部工具被高估，但实际功能差距明显

### 建议（按优先级）
1. [高优先级] 客观对比功能清单：内部vs外部
2. [高优先级] 评估维护成本：内部工具需持续投入人力
3. [中优先级] 考虑混合方案：外部产品+自定义配置

### 关键假设
- 功能对比数据客观
- 维护成本可量化

### 风险
- IKEA效应可能导致坚持低效方案
- 需要区分情感价值和实际价值
```

## 示例 2：IKEA效应——方案选择

**输入**：这个方案我花了2周时间设计，虽然有其他更好的方案，但我还是倾向用自己的。

```
Step 1: 识别IKEA信号 → 产出：发现"我花了2周...倾向用自己的"表述，因投入时间而偏好自己方案
Step 2: 评估价值偏差 → 产出：自己方案投入2周劳动，但其他方案可能更优
Step 3: 分析IKEA机制 → 产出：努力正当化：2周投入使方案被高估；沉没成本：已投入时间影响未来决策
Step 4: 校正IKEA效应 → 产出：建议基于方案质量选择，而非投入时间
```

**输出**：
```
## 分析结论
### 核心发现
1. 自己方案投入2周，但其他方案可能更优
2. IKEA效应使劳动投入被误认为价值指标
3. 方案质量应基于客观标准评估，而非投入时间

### 建议（按优先级）
1. [高优先级] 用客观标准评估各方案：性能、成本、可维护性
2. [高优先级] 邀请第三方评估，避免自我偏见
3. [中优先级] 理解2周投入是沉没成本，不应影响决策

### 关键假设
- 客观评估标准可定义
- 第三方可提供 unbiased 评估

### 风险
- IKEA效应可能导致选择次优方案
- 需要区分投入和质量
```

## 反例：不该触发此 Skill 的场景

**输入**：自己开发的工具经过用户测试，满意度90%，优于外部产品。

**原因**：这是基于用户测试数据的结论，不是劳动投入偏差。IKEA效应指仅因"自己做的"而高估。
