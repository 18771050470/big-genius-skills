# 常见陷阱

## 陷阱 1：将信心等同于可实现性

**问题**：因团队有信心而认为计划可实现，忽视历史数据。

❌ **Before（错误做法）**
```markdown
## 分析结论
技术团队很有信心，6个月能完成。
建议：按6个月计划。
```
*问题：乐观偏差使信心替代数据，忽视历史延期率*

✅ **After（正确做法）**
```markdown
## 分析结论
### 信心vs数据对比
| 指标 | 值 | 说明 |
|------|-----|------|
| 团队信心 | 高 | 主观感受 |
| 历史准时率 | 40% | 过去10个项目只有4个准时 |
| 平均延期率 | 40% | 平均比计划多40%时间 |

### 建议
1. 信心是主观感受，历史数据是客观事实
2. 基于历史数据调整：6个月×1.4=8.4个月
```
*优势：用历史数据校正主观信心*

**原因**：Sharot(2011)证明：大脑对正面信息更新更积极，对负面信息更新更消极。这导致系统性乐观偏差。

## 陷阱 2：独特优越感

**问题**：认为自己比统计更幸运，风险不会发生在自己身上。

❌ **Before（错误做法）**
```markdown
## 分析结论
别人会延期，但我们团队不一样，这次能准时。
建议：按原计划执行。
```
*问题：乐观偏差使认为自己是例外，忽视统计规律*

✅ **After（正确做法）**
```markdown
## 分析结论
### "我们是例外"验证
| 差异因素 | 是否存在 | 影响 |
|---------|---------|------|
| 团队规模 | 相同 | 无差异 |
| 技术栈 | 相同 | 无差异 |
| 项目复杂度 | 更高 | 更可能延期 |
| 新增因素 | 无 | 无改善 |

### 建议
1. 无证据表明这次是例外
2. 项目复杂度更高，延期风险更大
```
*优势：验证"例外"假设，揭示无依据乐观*

**原因**：Weinstein(1980)证明：人们系统性低估自己遭遇负面事件的概率，高估遭遇正面事件的概率。

## 陷阱 3：规划谬误

**问题**：基于理想情况做计划，不考虑可能出现的问题。

❌ **Before（错误做法）**
```markdown
## 分析结论
开发2个月、测试1个月、上线1个月，共4个月。
建议：按此计划执行。
```
*问题：规划谬误使基于最佳情况估计，忽视常见问题*

✅ **After（正确做法）**
```markdown
## 分析结论
### 参考类预测
| 阶段 | 最佳情况 | 历史平均 | 建议估算 |
|------|---------|---------|---------|
| 开发 | 2个月 | 3个月 | 3个月 |
| 测试 | 1个月 | 1.5个月 | 1.5个月 |
| 上线 | 1个月 | 1.5个月 | 1.5个月 |
| 总计 | 4个月 | 6个月 | 6个月 |

### 建议
1. 使用历史平均而非最佳情况
2. 添加15%应急缓冲：6.9个月
```
*优势：参考类预测基于历史数据，而非理想情况*

**原因**：Kahneman & Tversky(1979)规划谬误：人们基于最佳情况估计，忽视历史分布。参考类预测是校正方法。
