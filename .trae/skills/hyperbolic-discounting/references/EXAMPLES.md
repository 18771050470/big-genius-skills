# 分析示例

## 示例 1：双曲贴现——项目优先级选择

**输入**：这个月先做那个能快速上线的小功能，长期架构重构等下个月再说。

```
Step 1: 识别贴现信号 → 产出：发现"先做快速上线...下个月再说"表述，偏好即时收益而非延迟大收益
Step 2: 评估时间偏好 → 产出：小功能1周上线(即时满足)，架构重构3个月见效(延迟满足)，但长期收益差距大
Step 3: 分析贴现机制 → 产出：现时偏好：现在收益被过度加权；时间不一致偏好：未来的"以后做"不等于现在的"以后做"
Step 4: 校正双曲贴现 → 产出：建议用NPV评估长期收益，设置承诺机制
```

**输出**：
```
## 分析结论
### 核心发现
1. 小功能1周上线，收益10万；架构重构3个月，年收益100万
2. 双曲贴现使即时收益(10万)被过度加权，延迟收益(100万/年)被低估
3. 3个月后可能继续推迟重构，时间不一致偏好

### 建议（按优先级）
1. [高优先级] 计算NPV：架构重构NPV远高于小功能
2. [高优先级] 设置承诺机制：本月启动重构，不可推迟
3. [中优先级] 平衡短期和长期：70%重构+30%小功能

### 关键假设
- 架构重构收益可量化
- 承诺机制有效

### 风险
- 双曲贴现可能导致持续推迟重要但不紧急的任务
- 需要外部约束
```

## 示例 2：双曲贴现——技术债务

**输入**：先快速实现功能，代码质量后面再优化，现在赶进度最重要。

```
Step 1: 识别贴现信号 → 产出：发现"先快速实现...后面再优化"表述，偏好即时交付而非长期质量
Step 2: 评估时间偏好 → 产出：快速实现节省2周(即时)，但技术债务导致未来维护成本增加50%(延迟成本)
Step 3: 分析贴现机制 → 产出：现时偏好：当前进度压力被过度加权；未来折扣：未来成本被大幅折扣
Step 4: 校正双曲贴现 → 产出：建议量化技术债务成本，设置质量门禁
```

**输出**：
```
## 分析结论
### 核心发现
1. 快速实现节省2周，但技术债务使未来维护成本增加50%
2. 技术债务复利效应：每月额外维护时间递增
3. "后面再优化"往往不会发生，时间不一致偏好

### 建议（按优先级）
1. [高优先级] 量化技术债务成本：每月额外维护时间×人力成本
2. [高优先级] 设置代码质量门禁，不可妥协
3. [中优先级] 分阶段交付：核心功能高质量+边缘功能快速

### 关键假设
- 技术债务成本可量化
- 质量门禁可执行

### 风险
- 双曲贴现可能导致技术债务持续积累
- 需要制度约束而非依赖自律
```

## 反例：不该触发此 Skill 的场景

**输入**：这个紧急bug需要立即修复，长期优化可以排期到下周。

**原因**：这是紧急问题处理，bug修复不可延迟。双曲贴现指可延迟的长期收益被不合理折扣。
