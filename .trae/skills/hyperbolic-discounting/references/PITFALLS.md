# 常见陷阱

## 陷阱 1：过度偏好即时收益

**问题**：因即时满足而放弃更大的延迟收益。

❌ **Before（错误做法）**
```markdown
## 分析结论
先做快速上线的小功能，架构重构下个月再说。
建议：优先小功能。
```
*问题：双曲贴现使即时收益被过度加权*

✅ **After（正确做法）**
```markdown
## 分析结论
### 收益时间对比
| 选项 | 时间 | 收益 | NPV(10%折现) |
|------|------|------|-------------|
| 小功能 | 1周 | 10万 | 10万 |
| 架构重构 | 3月 | 100万/年 | 82万 |

### 建议
1. 架构重构NPV是小功能的8倍
2. 设置承诺机制，本月启动重构
```
*优势：NPV比较揭示真实价值差距*

**原因**：Laibson(1997)双曲贴现模型：人们对即时收益的贴现率远高于未来收益，导致时间不一致偏好。

## 陷阱 2：时间不一致偏好

**问题**：今天计划"以后做"，到以后时又不想做。

❌ **Before（错误做法）**
```markdown
## 分析结论
下个月一定做架构重构。
建议：本月先做小功能。
```
*问题：双曲贴现使未来的承诺不可靠*

✅ **After（正确做法）**
```markdown
## 分析结论
### 时间不一致分析
| 时间点 | 决策 | 原因 |
|--------|------|------|
| 今天 | 下月做重构 | 未来成本被折扣 |
| 下月 | 再下月做 | 现时偏好继续 |
| 3月后 | 还没做 | 持续推迟 |

### 建议
1. 时间不一致使"以后做"不可靠
2. 需要本月设置不可撤销的承诺
```
*优势：识别时间不一致模式，设置承诺机制*

**原因**：双曲贴现使偏好随时间变化。今天的"以后做"不等于以后的"做"。

## 陷阱 3：未来自我连续性断裂

**问题**：将未来自我视为陌生人，不关心未来成本。

❌ **Before（错误做法）**
```markdown
## 分析结论
先快速实现，后面再优化。
建议：赶进度优先。
```
*问题：双曲贴现使未来成本被大幅折扣*

✅ **After（正确做法）**
```markdown
## 分析结论
### 未来自我连续性
| 时间 | 成本承担者 | 决策者 |
|------|-----------|--------|
| 现在 | 无 | 你 |
| 3月后 | 维护成本+50% | 未来的你 |

### 建议
1. 未来的你也是你，成本真实存在
2. 量化技术债务成本，纳入当前决策
```
*优势：连接现在和未来自我*

**原因**：Hershfield等(2011)证明：人们将未来自我视为"陌生人"，大脑活动类似于思考他人。
