# 常见陷阱

## 陷阱 1：安全边际变成浪费

**反例**：
```
"预留10倍容量"（成本远超风险损失）
```

**正例**：
```
"预留2倍容量，成本$5K/月 vs 宕机损失$50K/小时，ROI合理"
```

**原因**：边际成本必须与风险损失期望值对比。

---

## 陷阱 2：边际设置后不再检查

**反例**：
```
"半年前设置了2倍边际，一直没问题"（但负载已增长到边际的90%）
```

**正例**：
```
"每月检查边际消耗情况，当前剩余30%，需在下月前扩容"
```

**原因**：边际会随时间消耗。

---

## 陷阱 3：用边际掩盖根本问题

**反例**：
```
"内存不够就加机器"（但根本原因是内存泄漏）
```

**正例**：
```
"先修复内存泄漏，再设置合理的内存安全边际"
```

**原因**：边际是临时保护，不是长期方案。

---

## 陷阱 4：边际系数拍脑袋

**反例**：
```
"设3倍吧，感觉差不多"
```

**正例**：
```
"基于历史波动率计算，95%分位需要2.3倍，取整为2.5倍"
```

**原因**：边际系数必须有数据支撑。
