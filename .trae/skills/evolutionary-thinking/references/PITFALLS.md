# 常见陷阱

## 陷阱 1：只有变异没有选择

**反例**：
```
"我们每两周都提很多新想法"但没有筛选机制，全部上线
```

**正例**：
```
"每两周提3-5个想法，淘汰率60%，只保留最符合选择标准的"
```

**原因**：没有选择压力的变异是资源浪费。

---

## 陷阱 2：选择标准与真实生存指标不对齐

**反例**：
```
"选择标准是代码质量"（但用户不关心代码质量）
```

**正例**：
```
"选择标准是用户留存影响和功能使用率"
```

**原因**：虚假的选择标准导致系统向错误方向进化。

---

## 陷阱 3：过早收敛

**反例**：
```
"这个方向效果好，所有资源都投进去"（放弃了其他可能性）
```

**正例**：
```
"70%资源投入已验证方向，30%探索新方向"
```

**原因**：环境变化可能使当前最优方向失效。

---

## 陷阱 4：每次从零开始

**反例**：
```
"上次迭代的知识没有传递到下次"
```

**正例**：
```
"每次迭代记录决策日志和效果数据，下次迭代前回顾"
```

**原因**：进化是累积过程，不是每次重新开始。

---

## 陷阱 5：探索和利用失衡

**反例**：
```
"100%资源用于优化现有功能"（陷入局部最优）
```

**正例**：
```
"70%利用现有成功模式，30%探索新方向"
```

**原因**：过度利用导致无法适应环境变化。
