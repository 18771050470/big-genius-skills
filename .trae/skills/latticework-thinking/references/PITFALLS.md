# 常见陷阱

## 陷阱 1：强行套用不相关学科

**反例**：
```
"用热力学第二定律解释代码腐化"（表面类比，无实质联系）
```

**正例**：
```
"用信息熵解释代码复杂度，有数学上的同构性"
```

**原因**：格栅的力量来自真实的学科概念，不是表面类比。

---

## 陷阱 2：只罗列不整合

**反例**：
```
"心理学说A，经济学说B，社会学说C"（然后没了）
```

**正例**：
```
"三个学科指向同一结论：冷启动是核心问题，解决方案需要三管齐下"
```

**原因**：格栅的价值在于交叉验证，不是简单罗列。

---

## 陷阱 3：模型前提假设不匹配

**反例**：
```
用"理性人假设"的经济学模型分析冲动消费行为
```

**正例**：
```
"行为经济学的前景理论更适合分析非理性消费决策"
```

**原因**：模型必须在前提假设匹配的场景下使用。

---

## 陷阱 4：分析瘫痪

**反例**：
```
引入10个学科视角，每个分析2小时，最终无法决策
```

**正例**：
```
引入3个最相关学科，快速交叉验证，得出可执行结论
```

**原因**：格栅是工具，不是目的。过度使用会消耗精力。
