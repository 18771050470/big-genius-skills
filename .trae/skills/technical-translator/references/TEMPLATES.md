# 技术交付物模板

## 术语表模板

```markdown
# [项目名] 术语表

> 语言对：[源语言] → [目标语言]
> 版本：[版本号]
> 日期：[日期]

## 术语列表

| 源术语 | 目标术语 | 词性 | 语境 | 不翻译 | 备注 |
|--------|---------|------|------|--------|------|
| [英文] | [中文] | [名/动/形] | [使用场景] | [是/否] | [选择依据] |
| cache | 缓存 | 名/动 | 通用 | 否 | 社区通用译法 |
| bug | - | 名 | 通用 | 是 | 中文技术圈习惯保留英文 |

## 不翻译词汇
以下词汇保留英文原文：
- 品牌名：[列表]
- API标识符：[列表]
- 技术错误码：[列表]
- 社区习惯保留词：[列表]

## 风格指南
- **语气**：[正式/非正式]
- **人称**：[你/您]
- **语态**：主动语态优先
- **格式**：代码不翻译，注释翻译
```

## 翻译质量检查清单

```markdown
# 翻译质量检查清单

## 基础检查
- [ ] 所有代码片段保持原样，未被翻译
- [ ] 所有变量名、函数名、API路径保持原样
- [ ] 术语使用与术语表一致，无多种译法
- [ ] 链接和锚点保持有效
- [ ] 图片说明和替代文本已翻译

## 格式检查
- [ ] Markdown结构完整（标题、列表、表格、代码块）
- [ ] 原始格式标记（粗体、斜体、代码）保持正确
- [ ] 表格列对齐，内容完整

## 本地化检查
- [ ] 日期格式符合目标语言习惯
- [ ] 货币符号和单位已转换
- [ ] 文化特定内容已适配
- [ ] 复数形式符合目标语言规则

## 一致性检查
- [ ] 同一术语全文统一
- [ ] 语气和人称前后一致
- [ ] 技术概念翻译准确
```

## 翻译风格指南模板

```markdown
# [项目名] 翻译风格指南

## 一般原则
1. **准确性优先**：技术准确性高于语言优美
2. **一致性**：同一术语、表达全文统一
3. **简洁性**：避免冗长，保留原文的简洁风格

## 语气与人称
- 使用[你/您]称呼用户
- 采用[主动/被动]语态
- 语气[正式/友好/技术]

## 特定规则
### 代码处理
- 代码块：保持原样，不翻译
- 注释：翻译为[目标语言]
- 变量名：不翻译

### 术语处理
- 首次出现：`英文（中文）`
- 后续出现：`中文`
- 不翻译词：保留英文，不加括号

### UI元素
- 按钮文字：简洁，考虑字符限制
- 错误信息：明确、可操作
- 提示文字：友好、有帮助
```
