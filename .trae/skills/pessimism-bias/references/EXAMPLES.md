# 分析示例

## 示例 1：悲观偏差——项目风险评估

**输入**：这个项目肯定会出问题，技术栈太新了，团队也没经验，迟早要延期。

```
Step 1: 识别悲观信号 → 产出：发现"肯定会出问题"、"迟早要延期"表述，高估坏结果概率
Step 2: 评估概率估计 → 产出：技术栈新≠必然失败，团队无经验≠无法学习
Step 3: 分析悲观机制 → 产出：恐惧思维：害怕失败影响概率估计；负面可得性：失败案例更容易回忆
Step 4: 校正悲观偏差 → 产出：建议用基准率数据和缓解措施评估真实风险
```

**输出**：
```
## 分析结论
### 核心发现
1. "肯定会出问题"是悲观信号，但基准率显示类似项目成功率65%
2. 技术栈新是风险因素，但有文档和社区支持可降低风险
3. 团队无经验可通过培训(2周)和外部顾问缓解

### 建议（按优先级）
1. [高优先级] 基于基准率评估：成功率65%，非"肯定失败"
2. [高优先级] 制定风险缓解计划：培训+顾问+里程碑检查
3. [中优先级] 设置风险触发条件，而非预设失败

### 关键假设
- 基准率数据适用于当前项目
- 风险缓解措施有效

### 风险
- 过度悲观可能导致资源投入不足
- 自我实现预言：预期失败导致放弃努力
```

## 示例 2：悲观偏差——市场进入决策

**输入**：现在进入这个市场肯定不行，竞争太激烈了，我们这种小公司没机会的。

```
Step 1: 识别悲观信号 → 产出：发现"肯定不行"、"没机会"表述，系统性低估好结果概率
Step 2: 评估概率估计 → 产出：竞争激烈≠无机会，小公司也有差异化优势
Step 3: 分析悲观机制 → 产出：灾难化：将竞争放大为不可逾越；控制缺失感：低估自己可影响的因素
Step 4: 校正悲观偏差 → 产出：建议分析差异化机会和细分市场
```

**输出**：
```
## 分析结论
### 核心发现
1. "没机会"是悲观偏差，但市场数据显示新进入者成功率15-20%
2. 小公司优势：灵活、专注细分市场、决策快
3. 差异化机会：现有巨头忽视的细分需求

### 建议（按优先级）
1. [高优先级] 聚焦细分市场，避免正面竞争
2. [高优先级] 利用小公司优势：快速迭代、个性化服务
3. [中优先级] MVP验证细分市场需求

### 关键假设
- 细分市场存在未满足需求
- 小公司优势可转化为竞争力

### 风险
- 过度悲观可能错过市场机会
- 需要平衡风险和机会
```

## 反例：不该触发此 Skill 的场景

**输入**：根据数据，这个项目失败率80%，我们需要认真评估是否继续。

**原因**：这是基于数据的理性风险评估(80%失败率)，不是悲观偏差。悲观偏差指无数据支撑的悲观估计。
