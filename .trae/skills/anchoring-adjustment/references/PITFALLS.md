# 常见陷阱

## 陷阱 1：将初始值作为估计依据

**问题**：用他人提供的初始值作为估计起点，导致估计偏向锚点。

❌ **Before（错误做法）**
```markdown
## 分析结论
候选人期望30k，我们预算20k，差距太大。
建议：要么接受30k，要么放弃。
```
*问题：锚定效应使30k成为唯一参考点，忽视市场数据*

✅ **After（正确做法）**
```markdown
## 分析结论
### 市场薪资数据
| 分位 | 薪资 | 说明 |
|------|------|------|
| P25 | 18k | 初级水平 |
| P50 | 22k | 市场中位数 |
| P75 | 25k | 经验丰富 |
| P90 | 28k | 顶级水平 |

### 建议
1. 候选人期望30k高于P90，需重新锚定到市场水平
2. 报价23k（略高于P50），提供市场数据支撑
```
*优势：用市场数据替代候选人报价作为参考，打破锚定*

**原因**：Tversky & Kahneman(1974)证明：随机数字也会影响估计值。锚点使估计值系统性偏向初始值，即使锚点与估计无关。

## 陷阱 2：调整不足

**问题**：从锚点开始调整，但调整幅度不够。

❌ **Before（错误做法）**
```markdown
## 分析结论
上次项目6个月，这次需求多30%，估算7.5个月。
建议：按7.5个月计划。
```
*问题：锚定调整不足：从6个月调整但幅度不够，自下而上估算显示需9个月*

✅ **After（正确做法）**
```markdown
## 分析结论
### 估算方法对比
| 方法 | 估算结果 | 依据 |
|------|---------|------|
| 锚定调整 | 7.5个月 | 从6个月+30%调整 |
| 自下而上 | 9个月 | 详细WBS累加 |
| 类基准 | 8-10个月 | 行业同类项目 |

### 建议
1. 放弃锚定调整法，采用自下而上估算
2. 计划9个月+15%缓冲=10.5个月
```
*优势：多种估算方法交叉验证，识别锚定调整不足*

**原因**：Epley & Gilovich(2006)证明：人们从锚点调整后会在"感觉合理"时停止，而非调整到准确值。

## 陷阱 3：忽视锚点的来源

**问题**：不区分锚点是随机值还是有依据的值，同等对待。

❌ **Before（错误做法）**
```markdown
## 分析结论
老板说这个项目最多3个月，我们就按3个月计划。
建议：压缩工期到3个月。
```
*问题：老板的3个月可能是随意说的锚点，非基于详细估算*

✅ **After（正确做法）**
```markdown
## 分析结论
### 锚点来源分析
| 来源 | 依据 | 可信度 |
|------|------|--------|
| 老板说3个月 | 无详细估算 | 低 |
| 自下而上估算 | 详细WBS | 高 |
| 类基准 | 历史数据 | 中 |

### 建议
1. 老板的3个月无依据，不应作为计划基础
2. 自下而上估算8个月更可靠
3. 向老板展示估算依据，管理预期
```
*优势：评估锚点来源可信度，拒绝无依据锚点*

**原因**：Strack & Mussweiler(1997)证明：即使知道锚点是随机的，锚定效应仍然存在。需要主动识别和拒绝无依据锚点。
