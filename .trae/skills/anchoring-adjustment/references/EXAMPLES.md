# 分析示例

## 示例 1：薪资谈判中的锚定

**输入**：候选人期望薪资是30k，我们预算只有20k，这个差距太大了没法谈。

```
Step 1: 识别锚定信号 → 产出：发现"30k"作为初始值，成为谈判锚点
Step 2: 评估锚定影响 → 产出：30k锚点使20k看起来过低，调整不足
Step 3: 分析锚定机制 → 产出：锚定使估计值偏向初始值；调整不充分：从锚点调整但幅度不够
Step 4: 校正锚定效应 → 产出：建议独立评估市场薪资中位数，忽略初始报价
```

**输出**：
```
## 分析结论
### 核心发现
1. 30k期望成为锚点，使20k预算看起来不合理
2. 该岗位市场中位数薪资为22k（P50），25k为P75
3. 候选人30k期望高于市场P90，可能基于前公司薪资或错误预期

### 建议（按优先级）
1. [高优先级] 提供市场薪资数据，重新锚定到市场水平
2. [高优先级] 报价23k（略高于中位数），留有谈判空间
3. [中优先级] 用非现金补偿补充（弹性工作、培训预算）

### 关键假设
- 市场薪资数据来自可靠来源
- 候选人可接受合理市场水平

### 风险
- 锚定效应使候选人难以从30k向下调整
- 可能需要多轮谈判
```

## 示例 2：项目估算中的锚定

**输入**：上次类似项目做了6个月，这次应该也差不多。

```
Step 1: 识别锚定信号 → 产出：发现"6个月"作为锚点，影响本次估算
Step 2: 评估锚定影响 → 产出：上次项目成为锚点，本次估算将围绕6个月调整
Step 3: 分析锚定机制 → 产出：调整不足：即使有差异，调整幅度也不够；可得性：上次项目记忆鲜明
Step 4: 校正锚定效应 → 产出：建议自下而上估算，而非基于上次调整
```

**输出**：
```
## 分析结论
### 核心发现
1. 上次6个月成为锚点，本次估算将受其影响
2. 本次项目差异：需求多30%、技术栈不同、团队新人多
3. 自下而上估算结果：9个月，远高于6个月锚点

### 建议（按优先级）
1. [高优先级] 使用自下而上估算，而非基于上次调整
2. [高优先级] 参考类基准数据（行业同类项目平均8-10个月）
3. [中优先级] 设置15%应急缓冲，计划10.5个月

### 关键假设
- 自下而上估算基于详细WBS
- 类基准数据可比

### 风险
- 锚定效应使团队倾向于接受6个月
- 需要管理层支持更现实的估算
```

## 反例：不该触发此 Skill 的场景

**输入**：根据合同条款，违约金是合同金额的20%。

**原因**：这是合同约定的固定值，不是认知估计偏差，无需识别锚定效应。
