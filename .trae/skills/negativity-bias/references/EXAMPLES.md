# 分析示例

## 示例 1：负面偏差——用户反馈处理

**输入**：这个产品上线后有100条评价，95条好评，5条差评，但老板只关注那5条差评。

```
Step 1: 识别负面信号 → 产出：发现"只关注5条差评"表述，负面信息获得不成比例的关注
Step 2: 评估负面权重 → 产出：5条差评(5%)获得50%+注意力，95条好评(95%)被忽视
Step 3: 分析负面机制 → 产出：进化适应：负面信息可能威胁生存；损失权重：损失影响是收益的2-2.5倍
Step 4: 校正负面偏差 → 产出：建议平衡评估正负面反馈，按权重分析
```

**输出**：
```
## 分析结论
### 核心发现
1. 5条差评(5%)获得不成比例的关注，95条好评(95%)被忽视
2. 差评内容分析：3条是操作问题(可优化文档)，2条是功能建议(已排期)
3. 好评关键词：易用(60%)、稳定(25%)、美观(15%)

### 建议（按优先级）
1. [高优先级] 平衡评估：好评率95%是健康指标
2. [高优先级] 分类处理差评：操作问题(优化文档)、功能建议(排期)
3. [中优先级] 提取好评中的核心优势，继续强化

### 关键假设
- 评价样本具有代表性
- 差评可分类处理

### 风险
- 过度关注差评可能导致产品方向偏离
- 忽视好评可能错过核心优势识别
```

## 示例 2：负面偏差——团队绩效评估

**输入**：团队今年做了20个项目，18个成功，2个有问题，年终总结应该重点反思那2个失败的。

```
Step 1: 识别负面信号 → 产出：发现"重点反思2个失败"表述，负面事件获得不成比例的关注
Step 2: 评估负面权重 → 产出：2个失败(10%)可能获得80%+的总结篇幅
Step 3: 分析负面机制 → 产出：负面事件更突出、更易回忆；失败比成功更具"教训价值"的假设
Step 4: 校正负面偏差 → 产出：建议平衡总结成功和失败，提取成功经验
```

**输出**：
```
## 分析结论
### 核心发现
1. 2个失败(10%)可能获得不成比例的总结关注
2. 18个成功项目的经验被忽视，这些经验可复制
3. 失败分析有价值，但成功经验同样重要

### 建议（按优先级）
1. [高优先级] 总结篇幅分配：成功70%、失败30%
2. [高优先级] 提取18个成功项目的共同成功因素
3. [中优先级] 分析2个失败项目的根本原因

### 关键假设
- 成功经验可复制
- 失败分析可避免重复

### 风险
- 过度关注失败可能打击团队士气
- 忽视成功经验可能错过最佳实践
```

## 反例：不该触发此 Skill 的场景

**原因**：安全漏洞是真实高风险事件，需要优先处理。负面偏差指对负面信息的过度关注，但安全漏洞确实需要紧急响应。
