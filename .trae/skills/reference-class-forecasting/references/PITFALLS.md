# 常见陷阱

## 陷阱 1：内部视角替代外部视角

**问题**：基于自己计划而非历史数据估计。

❌ **Before（错误做法）**
```markdown
## 分析结论
我们估计6个月完成，团队能力强。
建议：按6个月计划。
```
*问题：内部视角忽视历史数据*

✅ **After（正确做法）**
```markdown
## 分析结论
### 内部vs外部视角
| 视角 | 估计 | 依据 |
|------|------|------|
| 内部 | 6个月 | 当前计划 |
| 外部 | 9个月 | 10个类似项目 |
| 差距 | 33% | 规划谬误 |

### 建议
1. 外部视角更可靠
2. 基于参照类：9个月
```
*优势：外部视角校准内部乐观*

**原因**：Kahneman & Lovallo(1993)：内部视角关注当前计划细节，外部视角关注类似案例结果分布。外部视角更准确。

## 陷阱 2：选择不当参照类

**问题**：参照类与当前项目不可比。

❌ **Before（错误做法）**
```markdown
## 分析结论
参考网站开发项目，平均2个月，所以AI项目也2个月。
建议：按2个月计划。
```
*问题：参照类不当（网站≠AI项目）*

✅ **After（正确做法）**
```markdown
## 分析结论
### 参照类可比性
| 参照类 | 可比性 | 平均耗时 |
|--------|--------|---------|
| 网站开发 | 低 | 2个月 |
| AI项目 | 高 | 9个月 |
| 数据分析 | 中 | 5个月 |

### 建议
1. 选择高可比性参照类
2. AI项目参考AI项目数据
```
*优势：选择恰当参照类*

**原因**：参照类必须与当前项目在复杂度、技术栈、团队规模上可比。

## 陷阱 3：忽视分布范围

**问题**：只用平均值，忽视变化范围。

❌ **Before（错误做法）**
```markdown
## 分析结论
类似项目平均9个月，我们就按9个月。
建议：按平均估计。
```
*问题：忽视分布范围，平均可能不具代表性*

✅ **After（正确做法）**
```markdown
## 分析结论
### 分布分析
| 统计量 | 值 | 说明 |
|--------|-----|------|
| 最小值 | 6个月 | 最佳情况 |
| 中位数 | 8个月 | 典型情况 |
| 平均值 | 9个月 | 受极端值影响 |
| 最大值 | 14个月 | 最差情况 |

### 建议
1. 中位数8个月比平均9个月更典型
2. 规划应考虑范围：6-14个月
```
*优势：分布分析提供完整视角*

**原因**：平均值受极端值影响，中位数和分布范围提供更完整信息。
