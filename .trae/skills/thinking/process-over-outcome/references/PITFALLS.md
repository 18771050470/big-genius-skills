# 常见陷阱

## 陷阱 1：用结果评价决策质量

**问题**：因好结果而认为决策过程好。

❌ **Before（错误做法）**
```markdown
## 分析结论
股票涨了50%，说明投资眼光不错。
建议：继续这种投资方式。
```
*问题：结果偏差使好结果掩盖无分析过程*

✅ **After（正确做法）**
```markdown
## 分析结论
### 决策质量评估
| 维度 | 评估 | 说明 |
|------|------|------|
| 决策过程 | 差(随便买的) | 无分析框架 |
| 结果 | 好(涨50%) | 可能靠运气 |
| 可复制性 | 低 | 随机选择不可复制 |

### 建议
1. 好结果≠好决策
2. 建立投资分析框架
```
*优势：区分决策质量和结果*

**原因**：结果偏差使人们用结果评价决策质量。但好决策可能产生坏结果（运气差），坏决策可能产生好结果（运气好）。

## 陷阱 2：忽视过程的可复制性

**问题**：好结果靠运气但被归因于能力。

❌ **Before（错误做法）**
```markdown
## 分析结论
项目延期但质量好，是优秀项目。
建议：推广这种模式。
```
*问题：结果偏差使忽视过程问题，好结果不可复制*

✅ **After（正确做法）**
```markdown
## 分析结论
### 可复制性分析
| 因素 | 本次 | 可复制性 |
|------|------|---------|
| 团队加班 | 是 | 低(不可持续) |
| 需求变更 | 频繁 | 需改进 |
| 质量把控 | 好 | 可复制 |

### 建议
1. 好结果部分靠加班(不可持续)
2. 需改进需求管理过程
```
*优势：识别可复制和不可复制因素*

**原因**：过程决定可复制性。靠运气的结果无法复制，靠过程的结果可复制。

## 陷阱 3：结果合理化过程

**问题**：因好结果而为差过程找借口。

❌ **Before（错误做法）**
```markdown
## 分析结论
虽然过程有问题，但结果好就够了。
建议：不用改进过程。
```
*问题：结果偏差使好结果合理化差过程*

✅ **After（正确做法）**
```markdown
## 分析结论
### 过程vs结果矩阵
| | 好过程 | 差过程 |
|---|--------|--------|
| 好结果 | 优秀(可复制) | 幸运(不可复制) |
| 差结果 | 不幸(需坚持) | 糟糕(需改变) |

### 建议
1. 本次属于"幸运"：好结果+差过程
2. 需改进过程，使结果可复制
```
*优势：矩阵分类揭示结果与过程的关系*

**原因**：Mauboussin(2012)证明：结果=技能+运气。好结果可能来自好技能或好运气。
