# PITFALLS.md

## 陷阱1：过度校正——认为所有规避损失的行为都是非理性

### 问题描述
识别出损失厌恶后，容易走向另一个极端：盲目鼓励冒险，以为“只要对抗了损失厌恶，就是理性选择”。但损失厌恶在某些情境下是保护机制（如避免致命风险、维护核心资产），过度校正反而导致鲁莽决策。

### ❌ 反例
```
用户：我是否应该将全部积蓄投入这只高波动股票？客观数据它上涨概率60%，下跌40%。
分析者：你刚才犹豫是因为损失厌恶在作怪。理性期望值是正面的，所以你应该买入。
结果：用户全仓买入，遭遇下跌亏损50%，损失厌恶本应保护他却因过度校正而受害。
```
**错误原因**：期望值正面 ≠ 风险可接受。没考虑该损失（全部积蓄）的绝对规模巨大，且用户的风险承受能力有限。损失厌恶本应在此处拉响警报，却被错误地压下去了。

### ✅ 正例
```
用户：我是否应该将10%积蓄投入这只高波动股票？上涨概率60%，下跌40%。
分析者：你犹豫的感觉中确实有损失厌恶成分（担心亏损）。我们做两步：
1. 核查损失绝对值：10%积蓄的亏损意味着什么？会影响基本生活吗？（答案：不会）
2. 核算时间跨度：股票长期胜率更高吗？
最终结论：可投，但需明确告知用户这是略微激进的决策，且损失厌恶的担忧在此处被理性管理而非完全否定。
```
**为何正确**：没有机械地将损失厌恶归为“错误”，而是先评估损失的绝对危害，再决定是否允许冒险。保护了用户的可承受风险边界。

## 陷阱2：忽略群体损失厌恶——团队决策中放大偏差

### 问题描述
损失厌恶不只是个体现象，在团队或组织层面，由于成员互相强化“怕问责”、“怕背锅”的心理，损失厌恶会被放大数倍，导致决策瘫痪。分析时若只针对单个成员，会低估实际阻力。

### ❌ 反例
```
项目团队讨论是否升级老旧系统。收益：每年节省50万运维成本。
损失：升级期间可能中断服务1天，造成10万收入损失。
个体分析：净赚40万，决策应该通过。
实际团队反应：否决升级。因为每个部门都怕“自己负责的环节出问题”，怕担责。
分析者：你们这是损失厌恶，理性上应该升级！
结果：团队反弹更强烈，认为分析者不了解实际情况。
```
**错误原因**：只量化了数字损失，没量化“声誉损失”、“政治成本”等群体损失。每个人感知的损失不仅仅是10万收入，还有个人考核、团队面子等，加权后远超2倍。

### ✅ 正例
```
同上场景。分析者先承认：
“我明白每个人都担心项目失败的个人代价。我们来把整体决策拆开：
1. 组织层面净收益40万。
2. 每人感知的额外损失（声誉/考核等）假设等价于5万/人。
3. 团队6人，总感知损失=10万（服务收入）+ 6×5万=40万。
4. 现在感知损益：40万收益 vs 40万损失，刚好中性。所以团队犹豫是合理的。
解决方案：① 承诺一次性服务超时不用个人担责 ② 分阶段升级降低中断风险。
最终投赞成票。”
```
**为何正确**：将群体损失厌恶纳入量化，不指责团队“非理性”，而是通过解除个人损失负担来解锁决策，同时降低实际风险。

## 陷阱3：忽视上下文敏感性——同样的损失在不同财富水平下影响迥异

### 问题描述
损失厌恶的“2倍”比例是通用的，但**绝对规模**和**背景财富**会改变实际痛苦。对一个穷人和一个富人，同一笔损失的感受倍数可能从1.5到5不等。如果统一按2倍计算，会失准。

### ❌ 反例
```
决策：是否参与一个赌局——50%赢1000元，50%输800元。
用户A：月收入2000元（低财富）
用户B：月收入5万元（高财富）
分析者统一处理：损失权重2倍，则期望值=0.5×1000 - 2×0.5×800 = 500 - 800 = -300 → 建议不参与。
但用户A更痛苦，用户B相对无感。实际上对A而言损失权重可能是4倍，对B只有1.5倍。
```
**错误原因**：没有考虑损失金额相对于决策者总财富的比例。2倍是平均参数，在极端财富差异下会失效。

### ✅ 正例
```
同上赌局。分析者先采集背景信息：
用户A月收入2000，800元损失是他40%月收入 → 切实影响食品和租金 → 损失权重调整为4倍。
期望值=500 - 4×400 = 500 - 1600 = -1100 → 绝对不参与。
用户B月收入5万，800损失只占1.6% → 损失权重调整为1.5倍。
期望值=500 - 1.5×400 = 500 - 600 = -100 → 可以小玩，但期望仍为负，建议不玩但损失厌恶影响弱。

最终建议结合个人财富动态调整权重，比硬套2倍更精确。
```
**为何正确**：将损失厌恶的量化与决策者的经济背景挂钩，使分析结果更具个性化，避免一刀切偏差。
