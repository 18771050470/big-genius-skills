# 常见陷阱

## 陷阱 1：被诱饵引导选择

**问题**：因诱饵存在而选择原本不会选的选项。

❌ **Before（错误做法）**
```markdown
## 分析结论
B方案15万比C方案14万只多1万，但功能多很多，选B划算。
建议：选择B方案。
```
*问题：诱饵效应使B显得划算，但实际需求可能只需A*

✅ **After（正确做法）**
```markdown
## 分析结论
### 独立评估（移除诱饵C）
| 方案 | 价格 | 我们需要的功能 | 不需要的功能 |
|------|------|--------------|------------|
| A | 10万 | 100%覆盖 | 无 |
| B | 15万 | 100%覆盖 | 高级功能(用不上) |

### 建议
1. A方案满足需求，节省5万
2. B方案多花5万买不需要的功能
3. C是诱饵，不应影响决策
```
*优势：移除诱饵后独立评估，揭示真实需求匹配*

**原因**：Huber等(1982)证明：诱饵选项改变偏好分布，使被诱饵对比的选项选择率从40%升到60%。

## 陷阱 2：将"划算"等同于"需要"

**问题**：因性价比高而购买，忽视实际需求。

❌ **Before（错误做法）**
```markdown
## 分析结论
B方案性价比最高，应该选B。
建议：选择B方案。
```
*问题：诱饵效应使"划算"替代"需要"成为决策标准*

✅ **After（正确做法）**
```markdown
## 分析结论
### 需求vs性价比
| 评估标准 | A方案 | B方案 |
|---------|-------|-------|
| 需求满足度 | 100% | 100% |
| 多余功能 | 无 | 5个高级功能 |
| 价格 | 10万 | 15万 |
| 性价比 | 中 | 高(但含不需要的功能) |

### 建议
1. 需求满足度相同，选便宜的
2. "划算"是相对概念，绝对标准是"是否满足需求"
```
*优势：区分"划算"和"需要"，回归需求本质*

**原因**：诱饵效应使比较标准从"是否满足需求"变为"哪个更划算"。

## 陷阱 3：忽视诱饵的设计意图

**问题**：未识别选项是故意设计的诱饵。

❌ **Before（错误做法）**
```markdown
## 分析结论
3个方案都有各自特点，需要仔细比较。
建议：逐一比较后选择。
```
*问题：未识别C是诱饵，比较框架已被操纵*

✅ **After（正确做法）**
```markdown
## 分析结论
### 诱饵识别
1. C方案(14万)与B方案(15万)功能差距大但价格接近
2. C的存在使B显得性价比高，这是经典诱饵设计
3. 移除C后，决策简化为A vs B

### 建议
1. 识别并移除诱饵，简化决策
2. 独立评估剩余选项
```
*优势：识别诱饵设计意图，恢复原始决策框架*

**原因**：诱饵效应常被有意用于引导选择。识别诱饵是校正的第一步。
