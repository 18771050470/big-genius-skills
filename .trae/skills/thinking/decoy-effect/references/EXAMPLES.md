# 分析示例

## 示例 1：投资决策中的诱饵效应

**输入**：我们有3个方案：A方案10万基础功能、B方案15万高级功能、C方案14万基础功能+少量高级。

```
Step 1: 识别诱饵信号 → 产出：发现C方案(14万)与B方案(15万)对比，C是诱饵使B看起来更划算
Step 2: 评估诱饵影响 → 产出：C的存在使B显得性价比高，而非独立评估B的价值
Step 3: 分析诱饵机制 → 产出：不对称支配：C被B支配但接近B，使B成为折中选择
Step 4: 校正诱饵效应 → 产出：建议独立评估每个方案，移除诱饵后重新比较
```

**输出**：
```
## 分析结论
### 核心发现
1. C方案(14万基础+少量高级)是诱饵，使B方案(15万高级)显得性价比高
2. 移除C后，A(10万)和B(15万)差距明显，选择逻辑不同
3. 实际需求只需基础功能，B方案的高级功能用不上

### 建议（按优先级）
1. [高优先级] 独立评估：我们需要什么功能？基础还是高级？
2. [高优先级] 移除诱饵C后比较A和B
3. [中优先级] 如只需基础功能，A方案更合适

### 关键假设
- 高级功能确实不需要
- 基础功能满足核心需求

### 风险
- 诱饵使B方案看起来更划算，但可能过度购买
- 需要区分"划算"和"需要"
```

## 示例 2：招聘中的诱饵效应

**输入**：这3个候选人：A学历普通经验5年、B学历名校经验3年、C学历普通经验3年。

```
Step 1: 识别诱饵信号 → 产出：发现C(普通+3年)与B(名校+3年)对比，C是诱饵使B看起来更优秀
Step 2: 评估诱饵影响 → 产出：C的存在使B在经验相同情况下学历优势凸显
Step 3: 分析诱饵机制 → 产出：对比效应：C使B的学历优势被放大
Step 4: 校正诱饵效应 → 产出：建议独立评估每个候选人，关注岗位匹配度
```

**输出**：
```
## 分析结论
### 核心发现
1. C(普通+3年)可能是诱饵，使B(名校+3年)看起来更优秀
2. 岗位实际需求：经验>学历，A(5年经验)可能更匹配
3. 诱饵效应使B被高估，A被低估

### 建议（按优先级）
1. [高优先级] 独立评估：岗位最需要经验还是学历？
2. [高优先级] 用统一评分卡评估3人，避免两两对比
3. [中优先级] 安排技术面试，用实际表现评估

### 关键假设
- 岗位需求可明确定义
- 技术面试可客观评估能力

### 风险
- 诱饵使决策偏向B，但A可能更合适
- 学历≠能力，需实际验证
```

## 反例：不该触发此 Skill 的场景

**输入**：这3个方案功能完全不同，分别解决不同问题。

**原因**：方案解决不同问题，不存在可比性，诱饵效应只在可比选项间生效。
