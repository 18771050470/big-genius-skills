# 分析示例

## 示例 1：结果偏差——医疗诊断评估

**输入**：医生当时诊断是普通感冒，结果延误了治疗，这个医生水平不行。

```
Step 1: 识别结果信号 → 产出：发现"结果延误...水平不行"表述，用结果评价决策质量
Step 2: 评估决策过程 → 产出：基于当时症状(发热、咳嗽)诊断为感冒是合理的，罕见病概率低
Step 3: 分析结果机制 → 产出：结果偏差：坏结果使决策过程被否定；后见之明：知道结果后认为应该预见
Step 4: 校正结果偏差 → 产出：建议基于决策时的信息评估，而非结果
```

**输出**：
```
## 分析结论
### 核心发现
1. 基于当时症状诊断为感冒是合理决策（常见病概率高）
2. 罕见病概率<0.1%，不应作为常规排查
3. 结果偏差使坏结果否定好决策过程

### 建议（按优先级）
1. [高优先级] 基于决策时信息评估：症状指向感冒
2. [高优先级] 理解罕见病不可预见，非医生失误
3. [中优先级] 建立诊断流程，而非结果导向评价

### 关键假设
- 当时症状确实指向感冒
- 罕见病无早期特异性症状

### 风险
- 结果偏差可能导致防御性医疗
- 需要区分决策质量和结果
```

## 示例 2：结果偏差——投资决策

**输入**：他买这只股票亏了50%，投资决策太差了。

```
Step 1: 识别结果信号 → 产出：发现"亏了50%...决策太差"表述，用亏损结果评价决策质量
Step 2: 评估决策过程 → 产出：买入时有充分分析：财报好、行业增长、估值合理，决策过程正确
Step 3: 分析结果机制 → 产出：结果偏差：亏损使决策被否定；运气因素：黑天鹅事件不可预见
Step 4: 校正结果偏差 → 产出：建议评估决策过程，区分技能和运气
```

**输出**：
```
## 分析结论
### 核心发现
1. 买入决策过程正确：财报分析、行业研究、估值合理
2. 亏损原因：突发政策变化(黑天鹅)，不可预见
3. 好决策可能产生坏结果（运气差）

### 建议（按优先级）
1. [高优先级] 评估决策过程：分析框架正确
2. [高优先级] 区分技能(可控)和运气(不可控)
3. [中优先级] 从过程学习，而非结果

### 关键假设
- 决策过程可回顾
- 黑天鹅事件不可预见

### 风险
- 结果偏差可能否定正确决策过程
- 需要区分过程和结果
```

## 反例：不该触发此 Skill 的场景

**输入**：他没做任何分析就买了股票，结果亏了，决策过程有问题。

**原因**：这是决策过程确实有问题（无分析），结果只是验证了过程问题。结果偏差指好过程产生坏结果被否定。
