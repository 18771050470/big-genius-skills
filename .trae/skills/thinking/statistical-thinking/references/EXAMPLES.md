# 分析示例

## 示例 1：统计思维——A/B测试评估

**输入**：A版本转化率5%，B版本转化率6%，B版本更好，应该全量上线。

```
Step 1: 识别统计要素 → 产出：发现"5% vs 6%"表述，需评估统计显著性而非仅看点估计
Step 2: 评估显著性 → 产出：样本量各1000用户，p值=0.12>0.05，差异不显著
Step 3: 分析统计机制 → 产出：随机波动可能产生1%差异；样本量不足导致统计功效低
Step 4: 应用统计思维 → 产出：建议继续测试直到统计显著，或评估实际意义
```

**输出**：
```
## 分析结论
### 核心发现
1. B版本转化率6% vs A版本5%，但p值=0.12>0.05，差异不显著
2. 样本量各1000，统计功效60%<80%，可能漏检真实差异
3. 1%差异可能是随机波动，非真实效果

### 建议（按优先级）
1. [高优先级] 继续测试直到p<0.05且功效>80%
2. [高优先级] 计算需要样本量：约4000/组
3. [中优先级] 评估1%差异的实际业务价值

### 关键假设
- 随机分流正确
- 用户行为稳定

### 风险
- 过早结论可能基于随机波动
- 需要足够样本量
```

## 示例 2：统计思维——绩效评估

**输入**：小李本月业绩比上月增长20%，表现很好，应该奖励。

```
Step 1: 识别统计要素 → 产出：发现"增长20%"表述，需评估波动范围而非单月变化
Step 2: 评估波动 → 产出：历史月均波动±15%，20%增长在正常波动范围内
Step 3: 分析统计机制 → 产出：回归均值：上月低基数使本月增长看起来大；随机波动：月度业绩有自然波动
Step 4: 应用统计思维 → 产出：建议用多个月平均评估，而非单月变化
```

**输出**：
```
## 分析结论
### 核心发现
1. 20%增长在历史波动范围(±15%)内，可能只是正常波动
2. 上月业绩低基数(-10%)使本月增长看起来大
3. 单月数据噪声大，3个月移动平均更可靠

### 建议（按优先级）
1. [高优先级] 用3个月移动平均评估：实际增长5%
2. [高优先级] 理解回归均值：低基数后自然回升
3. [中优先级] 设置统计控制图，识别异常波动

### 关键假设
- 历史波动模式持续
- 无重大外部变化

### 风险
- 单月数据可能误导
- 需要区分信号和噪声
```

## 反例：不该触发此 Skill 的场景

**输入**：A版本转化率5%，B版本转化率15%，样本量各10000，p<0.001，B版本显著更好。

**原因**：这已经是统计显著的结论(p<0.001, 大样本)，不需要统计思维校正。统计思维适用于未考虑统计显著性的场景。
