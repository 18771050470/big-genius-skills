# 常见陷阱

## 陷阱 1：忽视统计显著性

**问题**：基于点估计做决策，不评估显著性。

❌ **Before（错误做法）**
```markdown
## 分析结论
B版本转化率6%比A版本5%好，全量上线。
建议：立即切换。
```
*问题：忽视统计显著性，1%差异可能是随机波动*

✅ **After（正确做法）**
```markdown
## 分析结论
### 显著性分析
| 指标 | 值 |
|------|-----|
| A版本 | 5% (n=1000) |
| B版本 | 6% (n=1000) |
| p值 | 0.12 |
| 显著性 | 不显著(p>0.05) |

### 建议
1. 差异不显著，可能是随机波动
2. 继续测试直到显著
```
*优势：显著性检验区分信号和噪声*

**原因**：小样本下，小差异很可能是随机波动。统计显著性检验帮助区分真实效果和随机噪声。

## 陷阱 2：混淆统计显著和实际意义

**问题**：统计显著不等于业务重要。

❌ **Before（错误做法）**
```markdown
## 分析结论
B版本显著更好(p<0.05)，应该上线。
建议：立即切换。
```
*问题：统计显著但实际差异可能很小*

✅ **After（正确做法）**
```markdown
## 分析结论
### 统计vs实际
| 维度 | 值 |
|------|-----|
| 统计显著 | 是(p=0.03) |
| 实际差异 | 0.1%(5.0% vs 5.1%) |
| 业务价值 | 低(年收入+1万) |
| 开发成本 | 5万 |

### 建议
1. 统计显著但实际价值低
2. 成本>收益，不建议上线
```
*优势：区分统计显著和实际意义*

**原因**：大样本下，微小差异也可能统计显著，但实际业务价值可能微不足道。

## 陷阱 3：单月数据过度解读

**问题**：将正常波动解读为趋势。

❌ **Before（错误做法）**
```markdown
## 分析结论
本月增长20%，表现很好。
建议：奖励。
```
*问题：单月数据在正常波动范围内*

✅ **After（正确做法）**
```markdown
## 分析结论
### 波动分析
| 指标 | 值 |
|------|-----|
| 本月增长 | 20% |
| 历史波动 | ±15% |
| 3月平均增长 | 5% |
| 是否在波动内 | 是 |

### 建议
1. 20%在正常波动范围内
2. 用3月平均评估更可靠
```
*优势：波动范围评估单月数据*

**原因**：Deming(1986)：大多数波动是系统固有变异，非特殊原因。统计控制图帮助区分常见原因和特殊原因变异。
