# 决策树常见陷阱

## 陷阱1：概率和收益不完整（忽略稀有但高影响的结果）

### 问题描述
构建决策树时，只考虑最常见的结果，忽略概率很低但影响巨大的“黑天鹅”事件，导致期望值计算严重失真。

### ❌ 反例
某投资决策树评估投资初创企业：
- 成功（概率80%）：收益100万  
- 失败（概率20%）：损失全部本金50万
期望值 = 0.8×100 + 0.2×(-50) = 80 - 10 = 70万 → 推荐投资。

但实际还存在“欺诈或法律风险”（概率5%）：损失500万（包括诉讼费）。未加入该分支，导致期望值虚高。

### ✅ 正例
正确构建分支：
- 极好（概率75%）：收益100万
- 一般（概率15%）：损失50万  
- 欺诈/法律（概率5%）：损失500万  
- 其他沉寂（概率5%）：损失50万  
期望值 = 0.75×100 + 0.15×(-50) + 0.05×(-500) + 0.05×(-50) = 75 - 7.5 - 25 - 2.5 = 40万。  
结论仍为投资，但风险显著更高，应设置止损点。

### 原因分析
人们倾向于聚焦大概率事件而忽视尾部风险。决策树要求“全部可能结果”穷举，需主动寻找极端但高影响的结果，并为之分配合理概率（即使很小）。缺少这一步会导致决策过于乐观。

## 陷阱2：概率值主观偏斜（过度自信或保守）

### 问题描述
在缺乏客观数据时，决策者常常主观赋予概率，且倾向于高估自己偏好的分支概率，低估不利分支概率，使期望值偏向某一选项。

### ❌ 反例
项目经理决定是否采用新技术，他偏爱该技术，于是：
- 顺利上线（概率90%）：节省成本200万  
- 困难延期（概率10%）：增加成本50万  
期望值 = 0.9×200 + 0.1×(-50) = 180 - 5 = 175万 → 推荐采用。

实际上，根据类似项目历史数据，顺利上线的概率只有60%，困难延期40%。正确期望值 = 0.6×200 + 0.4×(-50) = 120 - 20 = 100万，差异巨大。

### ✅ 正例
先收集历史数据（公司内部或行业报告）作为基准，再结合专家打分校正。例如：
- 顺利上线：基于以往3个项目中有2个顺利，概率≈67%
- 困难延期：33%
然后使用三角分布或贝叶斯调整。最终期望值更贴近实际，避免过度自信。

### 原因分析
认知偏见（如确认偏误、过度自信效应）直接影响概率分配。决策树本身不纠正概率质量，必须引入外部校准机制（如参考类似决策树的历史结局、德尔菲法、概率校准训练）。该陷阱的根源在于“量化”掩盖了主观性，让人误以为客观。
