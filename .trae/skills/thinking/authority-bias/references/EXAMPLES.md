# 分析示例

## 示例 1：技术选型中的权威偏差

**输入**：这个方案是Google首席工程师推荐的，肯定没问题，我们就用这个。

```
Step 1: 识别权威信号 → 产出：发现"Google首席工程师推荐的"表述，权威身份替代论点评估
Step 2: 评估权威影响 → 产出：权威身份使论点被不加批判接受，忽视适用性分析
Step 3: 分析权威机制 → 产出：服从倾向：权威使批判性思维关闭；光环扩散：权威身份扩散到方案质量
Step 4: 校正权威偏差 → 产出：建议独立评估方案优缺点，分离推荐人和方案本身
```

**输出**：
```
## 分析结论
### 核心发现
1. "Google首席工程师推荐"成为决策依据，而非方案本身质量
2. 该方案在Google场景有效，但我们的场景差异：数据量小100倍、团队规模不同
3. 方案缺点：学习曲线陡峭（3个月）、社区支持弱、与我们现有技术栈不兼容

### 建议（按优先级）
1. [高优先级] 独立评估方案优缺点，不看推荐人
2. [高优先级] 评估方案与我们场景的匹配度
3. [中优先级] 对比3个备选方案，做POC验证

### 关键假设
- 方案在Google场景有效≠在我们场景有效
- 团队技术能力可评估

### 风险
- 盲目跟随权威可能导致技术选型错误
- 学习成本可能被低估
```

## 示例 2：合规审查中的权威偏差

**输入**：这个合规方案是四大审计事务所出的，肯定符合法规要求。

```
Step 1: 识别权威信号 → 产出：发现"四大审计事务所出的"表述，机构权威替代合规验证
Step 2: 评估权威影响 → 产出：机构权威使方案被默认合规，忽视具体法规条款对照
Step 3: 分析权威机制 → 产出：信任转移：对机构的信任转移到方案质量；责任外包：依赖权威免除自身责任
Step 4: 校正权威偏差 → 产出：建议逐条对照法规验证，而非依赖机构声誉
```

**输出**：
```
## 分析结论
### 核心发现
1. "四大"声誉成为合规保证，但方案需逐条对照法规验证
2. 方案覆盖85%法规要求，但遗漏3项最新修订条款
3. 机构方案是通用模板，未针对我们行业特殊要求

### 建议（按优先级）
1. [高优先级] 逐条对照法规验证方案覆盖度
2. [高优先级] 补充遗漏的3项最新修订条款
3. [中优先级] 针对行业特殊要求定制方案

### 关键假设
- 法规条款可逐条验证
- 行业特殊要求已识别

### 风险
- 15%未覆盖条款可能导致合规风险
- 通用方案可能不满足行业监管
```

## 反例：不该触发此 Skill 的场景

**输入**：这位专家在机器学习领域发表了50篇论文，我们请他来做技术咨询。

**原因**：这是基于专业能力的合理选择，有具体能力依据（50篇论文），不是仅因权威身份而接受论点。
