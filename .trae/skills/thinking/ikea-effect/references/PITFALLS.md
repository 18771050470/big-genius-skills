# 常见陷阱

## 陷阱 1：因投入劳动而高估价值

**问题**：将劳动投入等同于成果价值。

❌ **Before（错误做法）**
```markdown
## 分析结论
我们自己做的工具更好用，因为是我们自己做的。
建议：继续使用内部工具。
```
*问题：IKEA效应使劳动投入替代质量评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 价值对比
| 维度 | 内部工具 | 外部产品 |
|------|---------|---------|
| 功能数量 | 5个 | 20个 |
| 用户满意度 | 60% | 85% |
| 维护成本 | 高(需人力) | 低(含支持) |
| 劳动投入 | 高 | 无 |

### 建议
1. 劳动投入≠价值
2. 外部产品在功能和满意度上更优
```
*优势：客观对比替代劳动投入偏见*

**原因**：Norton等(2012)实验：自己组装宜家盒子的人比拿到预组装盒子的人估值高63%。劳动投入创造情感价值，但不增加实际价值。

## 陷阱 2：努力正当化

**问题**：因付出努力而认为成果必须好。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个方案我花了2周，肯定比别人的好。
建议：用我的方案。
```
*问题：IKEA效应使努力被正当化为质量*

✅ **After（正确做法）**
```markdown
## 分析结论
### 方案评估
| 维度 | 自己方案 | 其他方案 |
|------|---------|---------|
| 投入时间 | 2周 | 1周 |
| 性能 | 中 | 高 |
| 成本 | 高 | 低 |
| 可维护性 | 中 | 高 |

### 建议
1. 投入时间≠方案质量
2. 其他方案在性能、成本、可维护性上更优
```
*优势：用客观标准替代努力正当化*

**原因**：努力正当化使人们倾向于认为付出努力产生的成果更有价值，即使客观质量不如他人。

## 陷阱 3：拒绝承认自己做的不好

**问题**：因认知失调而拒绝接受负面反馈。

❌ **Before（错误做法）**
```markdown
## 分析结论
用户说不好用是他们不会用，我们的工具没问题。
建议：不用改。
```
*问题：IKEA效应+认知失调使拒绝接受反馈*

✅ **After（正确做法）**
```markdown
## 分析结论
### 反馈分析
| 反馈来源 | 内容 | 可信度 |
|---------|------|--------|
| 用户测试 | 难用、功能少 | 高 |
| 开发者 | 好用 | 低(IKEA偏差) |

### 建议
1. 用户反馈比开发者自我评估更可靠
2. 接受负面反馈，改进产品
```
*优势：区分偏见反馈和客观反馈*

**原因**：认知失调使承认自己做的不好产生心理不适，导致拒绝接受负面反馈。
