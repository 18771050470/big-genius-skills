# 常见陷阱

## 陷阱 1：忽视隐性成本

**问题**：只计算显性成本，忽视机会成本和隐性成本。

❌ **Before（错误做法）**
```markdown
## 分析结论
开发成本30万，收益100万，值得做。
建议：立即启动。
```
*问题：忽视机会成本、运营成本、维护成本*

✅ **After（正确做法）**
```markdown
## 分析结论
### 完整成本清单
| 成本类型 | 金额 | 说明 |
|---------|------|------|
| 开发成本 | 30万 | 3个月人力 |
| 运营成本 | 5万 | 服务器+维护 |
| 机会成本 | 15万 | 放弃的其他项目 |
| 总成本 | 50万 | - |

### 建议
1. 完整成本50万，非仅开发成本30万
2. 收益/成本比=2.4，仍为正
```
*优势：完整成本清单避免低估成本*

**原因**：机会成本是放弃的最佳替代方案价值，常被忽视但真实存在。

## 陷阱 2：高估未来收益

**问题**：对未来收益估计过于乐观，未考虑折扣率。

❌ **Before（错误做法）**
```markdown
## 分析结论
2年收益50万，成本30万，值得做。
建议：启动项目。
```
*问题：未考虑时间价值，未来收益应折现*

✅ **After（正确做法）**
```markdown
## 分析结论
### 收益折现分析
| 年份 | 名义收益 | 折现率10% | 现值 |
|------|---------|----------|------|
| 第1年 | 25万 | 0.91 | 22.7万 |
| 第2年 | 25万 | 0.83 | 20.7万 |
| 总现值 | 50万 | - | 43.4万 |

### 建议
1. 折现后收益43.4万，非名义50万
2. 净现值=43.4-30=13.4万，仍为正
```
*优势：折现分析反映时间价值*

**原因**：货币有时间价值，未来1元不如现在1元值钱。

## 陷阱 3：忽视不确定性

**问题**：用点估计而非区间估计，忽视风险。

❌ **Before（错误做法）**
```markdown
## 分析结论
收益100万，成本50万，收益/成本=2。
建议：做。
```
*问题：未考虑收益的不确定性范围*

✅ **After（正确做法）**
```markdown
## 分析结论
### 敏感性分析
| 场景 | 收益 | 成本 | 收益/成本 |
|------|------|------|----------|
| 乐观 | 150万 | 45万 | 3.3 |
| 基准 | 100万 | 50万 | 2.0 |
| 悲观 | 50万 | 60万 | 0.8 |

### 建议
1. 悲观场景收益/成本<1，存在亏损风险
2. 设置止损条件，降低下行风险
```
*优势：敏感性分析揭示风险范围*

**原因**：点估计忽视不确定性，区间估计提供风险全景。
