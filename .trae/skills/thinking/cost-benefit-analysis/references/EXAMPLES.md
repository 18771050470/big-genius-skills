# 分析示例

## 示例 1：成本效益分析——技术债务偿还

**输入**：我们应该花2个月重构这个系统，虽然短期影响业务，但长期收益更大。

```
Step 1: 识别成本效益要素 → 产出：成本=2个月开发时间+业务延迟；收益=系统可维护性提升+bug减少
Step 2: 量化成本 → 产出：人力成本20万+业务延迟损失30万=50万
Step 3: 量化收益 → 产出：bug减少节省15万/年+开发效率提升20%=25万/年，2年回本
Step 4: 比较成本效益 → 产出：收益/成本比=1.0(2年回本)，需评估不确定性和替代方案
```

**输出**：
```
## 分析结论
### 核心发现
1. 总成本：50万（人力20万+业务延迟30万）
2. 年收益：25万/年（bug节省15万+效率提升10万）
3. 回本周期：2年，ROI=50%（2年）

### 建议（按优先级）
1. [高优先级] 分阶段重构，降低业务延迟影响
2. [高优先级] 优先重构bug最多的模块，快速见效
3. [中优先级] 设置重构效果度量指标

### 关键假设
- bug减少和效率提升可量化
- 2年内系统继续使用

### 风险
- 收益估计可能过于乐观
- 业务延迟可能影响客户满意度
```

## 示例 2：成本效益分析——新功能开发

**输入**：开发这个AI功能需要3个月，预计能带来100万收入，值得做吗？

```
Step 1: 识别成本效益要素 → 产出：成本=3个月开发+运营；收益=100万收入+用户增长
Step 2: 量化成本 → 产出：开发成本30万+运营成本5万+机会成本15万=50万
Step 3: 量化收益 → 产出：直接收入100万+间接收益(用户留存)20万=120万
Step 4: 比较成本效益 → 产出：收益/成本比=2.4，但需考虑成功概率和市场风险
```

**输出**：
```
## 分析结论
### 核心发现
1. 总成本：50万（开发30万+运营5万+机会成本15万）
2. 预期收益：120万（直接100万+间接20万）
3. 收益/成本比=2.4，但需考虑成功概率70%

### 建议（按优先级）
1. [高优先级] 调整后预期收益=120万×70%=84万，仍大于成本50万
2. [高优先级] 先用MVP验证市场需求，降低风险
3. [中优先级] 设置阶段性里程碑，及时止损

### 关键假设
- 70%成功概率基于类似功能历史数据
- 市场需求存在

### 风险
- 收入估计可能过于乐观
- 市场竞争可能影响收入
```

## 反例：不该触发此 Skill 的场景

**输入**：这个功能开发成本5万，收益只有3万，明显不划算。

**原因**：这是简单的成本效益比较，已经得出结论。成本效益分析适用于需要详细量化多因素的场景。
