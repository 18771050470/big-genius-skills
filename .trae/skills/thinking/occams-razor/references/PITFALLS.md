# 常见陷阱

## 陷阱 1：把"简单"等同于"好"

**反例**：
```
"这个方案代码最少，所以最好"（忽略了可维护性和扩展性）
```

**正例**：
```
"这个方案假设最少且解释力不弱于其他方案，所以选择它"
```

**原因**：剃刀比较的是假设数量，不是代码行数。

---

## 陷阱 2：解释力不等价时强行比简单

**反例**：
```
方案A能解释10个现象，方案B只能解释5个
→ "B更简单，选B"（错误：解释力不等价）
```

**正例**：
```
方案A和B都能解释全部10个现象
→ "B假设更少，选B"（正确：解释力等价）
```

**原因**：剃刀只在解释力等价时适用。

---

## 陷阱 3：虚假简单

**反例**：
```
"用全局变量最简单"（表面简单，但隐含了状态管理的复杂性）
```

**正例**：
```
"用状态管理库看似复杂，但消除了状态同步的隐含假设"
```

**原因**：表面简单可能隐藏了关键假设。
