# 分析示例

## 示例 1：API安全审查

**输入**：新上线的REST API需要安全审查

```
Step 1: 定义攻击面
→ 暴露面：公开API端点、认证接口、文件上传、管理后台
→ 信任边界：公网→API网关→内部服务→数据库
→ 高价值目标：用户数据、支付信息、管理权限
→ 产出：攻击面地图

Step 2: 构建威胁模型
→ 攻击者类型：脚本小子（低能力）、有组织犯罪（中高能力）、内部人员（高能力）
→ STRIDE分类：
  - Spoofing：JWT伪造
  - Tampering：请求体篡改
  - Repudiation：日志不完整
  - Information Disclosure：敏感数据泄露
  - DoS：API限流缺失
  - Elevation：越权访问
→ 产出：威胁清单

Step 3: 模拟攻击路径
→ 路径1：SQL注入→读取用户表→导出数据（难度低，检测概率中，影响高）
→ 路径2：JWT弱密钥→伪造管理员token→越权访问（难度中，检测概率低，影响高）
→ 路径3：文件上传→WebShell→内网渗透（难度高，检测概率低，影响极高）
→ 产出：攻击路径清单

Step 4: 评估防御有效性
→ SQL注入：有参数化查询，但部分动态查询未覆盖（防御部分有效）
→ JWT：密钥强度够，但未设置短过期时间（防御薄弱）
→ 文件上传：有类型检查，但无沙箱执行（防御不足）
→ 检测盲区：无API异常行为监控
→ 产出：防御评估
```

**输出**：
```
## 分析结论

### 核心发现
1. 3条高危攻击路径可直达核心数据
2. 现有防御覆盖60%威胁，但关键路径存在盲区
3. 缺乏API异常行为监控，攻击可能持续数周不被发现

### 建议（按优先级）
1. [高] 修复动态查询SQL注入，1天
2. [高] JWT设置15分钟过期+刷新token机制，2天
3. [高] 文件上传增加沙箱执行+病毒扫描，3天
4. [中] 部署API异常行为监控，1周

### 关键假设
- 攻击者会优先选择难度低、影响高的路径
- 内部人员威胁概率较低但影响最大
- 当前无已知漏洞被利用
```
