# 常见陷阱

## 陷阱 1：被单一框架引导决策

**问题**：因收益框架而接受，或因损失框架而拒绝，同一数据得出不同结论。

❌ **Before（错误做法）**
```markdown
## 分析结论
手术90%成功率，应该做。
建议：接受手术。
```
*问题：框架效应使收益框架下更倾向接受，但未考虑损失框架下的感受*

✅ **After（正确做法）**
```markdown
## 分析结论
### 双框架对比
| 框架 | 表述 | 选择率 | 我们的决策 |
|------|------|--------|-----------|
| 收益 | 90%成功率 | 84% | ？ |
| 损失 | 10%失败率 | 50% | ？ |

### 建议
1. 两种框架数据相同(90%成功=10%失败)
2. 决策应基于：10%失败风险是否可接受
3. 对比不手术的后果和替代方案
```
*优势：双框架对比揭示框架效应，基于绝对数据决策*

**原因**：Tversky & Kahneman(1981)亚洲疾病实验证明：收益框架下72%选择确定方案，损失框架下仅22%选择。同一数据因框架不同导致偏好反转。

## 陷阱 2：将"节省"等同于"收益"

**问题**：因节省框架而购买，忽视实际需求。

❌ **Before（错误做法）**
```markdown
## 分析结论
套餐比单独买节省300元，太划算了。
建议：购买套餐。
```
*问题：框架效应使"节省"被视为收益，诱导购买不需要的东西*

✅ **After（正确做法）**
```markdown
## 分析结论
### 需求评估
| 功能 | 是否需要 | 单独价格 |
|------|---------|---------|
| 功能A | 是 | 500元 |
| 功能B | 否 | 200元 |
| 功能C | 否 | 100元 |
| 套餐 | - | 700元 |

### 建议
1. 只需要功能A，单独买500元
2. 套餐700元但包含不需要的B和C
3. "节省300元"是相对于买全部功能，但只需要A
```
*优势：基于实际需求评估，而非节省感知*

**原因**：框架效应使节省被框架化为获得，触发购买冲动。

## 陷阱 3：忽视框架的操纵意图

**问题**：未识别表述是故意设计的框架。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个产品95%用户满意，质量很好。
建议：购买。
```
*问题：95%满意是收益框架，未考虑5%不满意的原因*

✅ **After（正确做法）**
```markdown
## 分析结论
### 框架转换
| 框架 | 表述 | 隐含信息 |
|------|------|---------|
| 收益 | 95%满意 | 正面 |
| 损失 | 5%不满意 | 需了解不满意原因 |

### 建议
1. 了解5%不满意的具体原因
2. 这些原因是否影响我们
3. 对比竞品满意度数据
```
*优势：转换框架揭示被隐藏的信息*

**原因**：框架常被有意用于引导决策。识别框架是校正的第一步。
