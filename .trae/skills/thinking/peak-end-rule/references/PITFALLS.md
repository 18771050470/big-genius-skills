# 常见陷阱

## 陷阱 1：终值覆盖整体体验

**问题**：因最后体验好而认为整体好。

❌ **Before（错误做法）**
```markdown
## 分析结论
最后支付很流畅，用户印象不错。
建议：保持支付流程即可。
```
*问题：峰终定律使终值掩盖其他环节问题*

✅ **After（正确做法）**
```markdown
## 分析结论
### 体验旅程分析
| 环节 | 体验评分 | 影响 |
|------|---------|------|
| 搜索 | 6/10 | 中 |
| 浏览 | 4/10 | 高(痛点) |
| 加购 | 6/10 | 中 |
| 支付 | 9/10 | 高(终值) |
| 整体记忆 | 7/10 | 被终值提升 |

### 建议
1. 浏览环节4/10是真实痛点
2. 好终值掩盖问题，但不解决问题
```
*优势：完整旅程分析揭示被掩盖的问题*

**原因**：Kahneman等(1993)结肠镜检查实验：短痛苦+长痛苦(好结尾)比短痛苦记忆更好，尽管总痛苦更多。

## 陷阱 2：持续时间忽视

**问题**：体验持续时间不影响记忆评价。

❌ **Before（错误做法）**
```markdown
## 分析结论
面试前40分钟无聊，但最后10分钟好，整体还行。
建议：保持友好结束。
```
*问题：峰终定律使40分钟无聊被10分钟友好覆盖*

✅ **After（正确做法）**
```markdown
## 分析结论
### 时间加权分析
| 时段 | 时长 | 体验 | 时间占比 | 记忆权重 |
|------|------|------|---------|---------|
| 前40分钟 | 40min | 无聊 | 67% | 低 |
| 最后10分钟 | 10min | 友好 | 17% | 高 |

### 建议
1. 67%时间无聊但记忆权重低
2. 优化前40分钟内容
```
*优势：揭示时间占比和记忆权重的不匹配*

**原因**：Fredrickson & Kahneman(1993)证明：体验持续时间不影响记忆评价，只有峰值和终值重要。

## 陷阱 3：峰值效应

**问题**：极端时刻主导整体记忆。

❌ **Before（错误做法）**
```markdown
## 分析结论
虽然大部分时间一般，但有一个惊喜时刻，整体印象好。
建议：多制造惊喜时刻。
```
*问题：峰终定律使峰值掩盖日常体验*

✅ **After（正确做法）**
```markdown
## 分析结论
### 峰值vs日常
| 维度 | 评估 |
|------|------|
| 峰值体验 | 9/10(惊喜) |
| 日常体验 | 6/10(一般) |
| 记忆评价 | 8/10(被峰值提升) |
| 实际价值 | 6/10(日常为主) |

### 建议
1. 峰值提升记忆但不提升日常价值
2. 提升日常体验比制造峰值更重要
```
*优势：区分记忆评价和实际价值*

**原因**：峰值体验记忆深刻，但用户大部分时间经历的是日常体验。
