# 常见陷阱

## 陷阱 1：基础率忽视

**问题**：用特征匹配替代基础率计算。

❌ **Before（错误做法）**
```markdown
## 分析结论
说话有逻辑，戴眼镜，喜欢读书，肯定是程序员。
建议：按程序员对待。
```
*问题：代表性启发使基础率被忽视*

✅ **After（正确做法）**
```markdown
## 分析结论
### 贝叶斯分析
| 信息 | 值 |
|------|-----|
| 基础率(程序员) | 2% |
| 特征匹配度 | 高 |
| 后验概率 | 仍受基础率限制 |

### 建议
1. 基础率是概率计算的起点
2. 特征匹配提高概率，但受基础率限制
```
*优势：贝叶斯框架结合基础率和特征*

**原因**：Kahneman & Tversky(1973)：人们用代表性（相似度）替代概率计算，忽视基础率。

## 陷阱 2：合取谬误

**问题**：认为详细描述比简单描述更可能。

❌ **Before（错误做法）**
```markdown
## 分析结论
他更像是一个喜欢编程且热爱开源的程序员，而非普通程序员。
建议：重点考察开源贡献。
```
*问题：代表性启发使详细描述听起来更可能*

✅ **After（正确做法）**
```markdown
## 分析结论
### 概率关系
| 事件 | 概率关系 |
|------|---------|
| 程序员 | P(A) |
| 喜欢编程且热爱开源的程序员 | P(A∩B) ≤ P(A) |

### 建议
1. 合取概率不可能大于单一事件概率
2. 详细描述使故事更可信，但概率更低
```
*优势：概率规则替代叙事可信度*

**原因**：Tversky & Kahneman(1983)Linda实验：85%人认为合取更可能，违反概率论。

## 陷阱 3：样本大小忽视

**问题**：从小样本推断总体特征。

❌ **Before（错误做法）**
```markdown
## 分析结论
我认识的3个程序员都内向，所以程序员都内向。
建议：按内向人对待。
```
*问题：代表性启发使小样本被当作代表性样本*

✅ **After（正确做法）**
```markdown
## 分析结论
### 样本分析
| 维度 | 值 |
|------|-----|
| 样本量 | 3 |
| 统计显著性 | 低 |
| 大样本数据 | 程序员内向率60% |

### 建议
1. 3个样本不足以推断总体
2. 需要大样本数据
```
*优势：样本大小影响推断可靠性*

**原因**：Tversky & Kahneman(1971)：人们忽视样本大小，从小样本过度推断。
