# 分析示例

## 示例 1：代表性启发——人才评估

**输入**：这个人说话很有逻辑，戴眼镜，喜欢读书，他应该是个程序员。

```
Step 1: 识别代表性信号 → 产出：发现"说话有逻辑...应该是程序员"表述，基于特征相似度判断概率
Step 2: 评估基础率 → 产出：程序员占总人口2%，即使特征匹配，基础率仍低
Step 3: 分析代表性机制 → 产出：相似度启发：特征像程序员就认为是；基础率忽视：忽略程序员占比低
Step 4: 校正代表性偏差 → 产出：建议结合基础率和特征匹配度评估
```

**输出**：
```
## 分析结论
### 核心发现
1. 特征匹配度高，但程序员基础率仅2%
2. 代表性启发使特征相似度替代概率计算
3. 这些特征在其他职业（教师、研究员）也常见

### 建议（按优先级）
1. [高优先级] 结合基础率：即使特征匹配，概率仍受基础率限制
2. [高优先级] 识别特征的职业区分度
3. [中优先级] 收集更多信息（技能、经验）

### 关键假设
- 基础率数据可靠
- 特征与职业相关性有限

### 风险
- 代表性启发可能导致刻板印象
- 需要区分特征和实际能力
```

## 示例 2：代表性启发——投资判断

**输入**：这家公司是AI赛道，团队来自大厂，模式像当年的阿里巴巴，肯定是下一个独角兽。

```
Step 1: 识别代表性信号 → 产出：发现"像阿里巴巴...肯定是独角兽"表述，基于相似案例推断成功
Step 2: 评估基础率 → 产出：创业公司5年成功率<10%，即使特征匹配，基础率仍低
Step 3: 分析代表性机制 → 产出：叙事连贯性：成功故事听起来合理；类比推理：像成功者就会成功
Step 4: 校正代表性偏差 → 产出：建议用基础率和具体指标评估，而非类比
```

**输出**：
```
## 分析结论
### 核心发现
1. "像阿里巴巴"是代表性启发，但创业基础成功率<10%
2. 成功公司特征可事后归因，但事前预测准确率低
3. 每个公司成功路径独特，简单类比可能误导

### 建议（按优先级）
1. [高优先级] 基于基础率评估：<10%成功率
2. [高优先级] 用具体指标评估：营收增长、用户留存
3. [中优先级] 理解类比推理的局限性

### 关键假设
- 创业成功率数据可靠
- 每个公司情况独特

### 风险
- 代表性启发可能导致过度乐观
- 需要区分叙事和现实
```

## 反例：不该触发此 Skill 的场景

**输入**：这个人有5年Java开发经验，GitHub有100个提交记录，他是程序员。

**原因**：这是基于直接证据（经验、代码提交）的判断，不是代表性启发。代表性启发指基于表面特征相似度推断。
