# 分析示例

## 示例 1：合取谬误——用户画像评估

**输入**：小李30岁，单身，喜欢旅行和阅读，她更可能是一位喜欢瑜伽的程序员，而不是单纯的程序员。

```
Step 1: 识别合取信号 → 产出：发现"更可能是A且B，而非单纯A"表述，违反概率论基本规则
Step 2: 评估概率关系 → 产出：P(程序员且喜欢瑜伽) ≤ P(程序员)，合取概率不可能大于单一事件
Step 3: 分析合取机制 → 产出：代表性启发："喜欢瑜伽的程序员"更符合描述；叙事连贯性：细节使故事更可信
Step 4: 校正合取谬误 → 产出：建议用集合关系理解概率，合集是子集
```

**输出**：
```
## 分析结论
### 核心发现
1. "喜欢瑜伽的程序员"是"程序员"的子集，概率不可能更大
2. 所有喜欢瑜伽的程序员都是程序员，但反之不成立
3. 描述细节使"喜欢瑜伽的程序员"听起来更可信，但这是代表性启发偏差

### 建议（按优先级）
1. [高优先级] 理解集合关系：程序员 = {喜欢瑜伽的程序员} + {不喜欢瑜伽的程序员}
2. [高优先级] 避免被细节描述误导概率判断
3. [中优先级] 用基础率数据评估，而非叙事连贯性

### 关键假设
- 用户画像描述不影响真实概率
- 集合关系适用于此场景

### 风险
- 合取谬误可能导致错误的用户细分决策
- 需要区分叙事可信度和统计概率
```

## 示例 2：合取谬误——风险评估

**输入**：明年发生经济衰退且股市暴跌的概率，比单纯发生经济衰退的概率更高。

```
Step 1: 识别合取信号 → 产出：发现"A且B概率高于A"表述，违反概率论
Step 2: 评估概率关系 → 产出：P(衰退且暴跌) ≤ P(衰退)，合取事件概率必小于等于单一事件
Step 3: 分析合取机制 → 产出：因果叙事：衰退导致暴跌的故事更连贯；可得性：具体场景更容易想象
Step 4: 校正合取谬误 → 产出：建议用概率规则而非叙事评估风险
```

**输出**：
```
## 分析结论
### 核心发现
1. P(衰退且暴跌) ≤ P(衰退)，这是概率论基本规则
2. "衰退导致暴跌"的叙事使合取听起来更可能，但数学上不成立
3. 历史数据：衰退发生率15%，衰退且暴跌发生率8%

### 建议（按优先级）
1. [高优先级] 用历史数据评估概率，而非叙事
2. [高优先级] 理解合取概率必小于等于单一事件概率
3. [中优先级] 分别评估衰退概率和暴跌条件概率

### 关键假设
- 历史数据可预测未来
- 衰退和暴跌非完全相关

### 风险
- 合取谬误可能高估复合风险
- 需要区分因果叙事和统计概率
```

## 反例：不该触发此 Skill 的场景

**输入**：在已知经济衰退的情况下，股市暴跌的概率是60%。

**原因**：这是条件概率P(暴跌|衰退)，不是合取概率。条件概率可以高于边际概率，不违反概率规则。
