# 常见陷阱

## 陷阱 1：认为合取事件比单一事件更可能

**问题**：因细节描述使合取事件听起来更可信。

❌ **Before（错误做法）**
```markdown
## 分析结论
小李更可能是喜欢瑜伽的程序员，而非单纯程序员。
建议：针对喜欢瑜伽的程序员做营销。
```
*问题：合取谬误使违反概率论基本规则*

✅ **After（正确做法）**
```markdown
## 分析结论
### 概率关系
| 事件 | 概率关系 | 说明 |
|------|---------|------|
| 程序员 | P(A) | 包含所有程序员 |
| 喜欢瑜伽的程序员 | P(A∩B) | 是程序员的子集 |
| 关系 | P(A∩B) ≤ P(A) | 数学规则 |

### 建议
1. 所有喜欢瑜伽的程序员都是程序员
2. 针对程序员群体，再细分瑜伽爱好者
```
*优势：遵循概率论规则，正确理解集合关系*

**原因**：Tversky & Kahneman(1983)Linda实验：85%参与者认为"Linda是银行出纳且是女权主义者"比"Linda是银行出纳"更可能，违反概率论。

## 陷阱 2：被叙事连贯性误导

**问题**：因故事更详细、更连贯而认为更可能。

❌ **Before（错误做法）**
```markdown
## 分析结论
经济衰退且股市暴跌的概率更高，因为衰退会导致暴跌。
建议：重点防范复合风险。
```
*问题：合取谬误使因果叙事替代概率评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 叙事vs概率
| 维度 | 评估 |
|------|------|
| 叙事连贯性 | 高(衰退→暴跌) |
| 统计概率 | P(衰退且暴跌) ≤ P(衰退) |
| 历史数据 | 衰退15%，衰退且暴跌8% |

### 建议
1. 叙事连贯≠概率更高
2. 用历史数据评估概率
```
*优势：区分叙事可信度和统计概率*

**原因**：合取谬误源于代表性启发：更详细的描述看起来更"代表性"，但概率上不可能更高。

## 陷阱 3：忽视基础率

**问题**：用具体描述替代基础率数据。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个描述太符合喜欢瑜伽的程序员了，概率肯定高。
建议：基于描述做决策。
```
*问题：合取谬误使描述替代基础率*

✅ **After（正确做法）**
```markdown
## 分析结论
### 基础率vs描述
| 信息类型 | 值 | 权重 |
|---------|-----|------|
| 基础率(程序员占比) | 5% | 高 |
| 基础率(喜欢瑜伽的程序员) | 1% | 高 |
| 描述匹配度 | 高 | 低 |

### 建议
1. 基础率是概率评估的基础
2. 描述匹配度不应替代基础率
```
*优势：用基础率数据校准描述匹配度*

**原因**：基础率忽视是合取谬误的核心机制。具体描述使基础率被忽略。
