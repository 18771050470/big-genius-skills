# 分析示例

## 示例 1：技术项目乐观偏差评估

**输入**：这个AI项目我们计划6个月上线，技术团队很有信心，应该能按时完成。

```
Step 1: 识别乐观信号 → 产出：发现"应该能按时完成"表述，未考虑历史延期数据
Step 2: 评估乐观影响 → 产出：技术团队信心高，但历史项目平均延期40%
Step 3: 分析乐观机制 → 产出：规划谬误：低估完成时间；控制错觉：高估自己对结果的控制
Step 4: 校正乐观偏差 → 产出：建议参考类预测，用历史数据调整估计
```

**输出**：
```
## 分析结论
### 核心发现
1. 技术团队信心高是乐观信号，但历史数据显示平均延期40%
2. 6个月计划调整后应为8.4个月（6×1.4）
3. 乐观偏差使团队低估集成测试和bug修复时间

### 建议（按优先级）
1. [高优先级] 基于历史数据调整计划：8-9个月
2. [高优先级] 设置月度里程碑，每月评估进度偏差
3. [中优先级] 识别最可能延期的环节（集成测试）

### 关键假设
- 历史项目与当前项目可比
- 团队规模和技术栈相似

### 风险
- 乐观计划可能导致资源紧张
- 延期可能影响业务节奏
```

## 示例 2：健康风险乐观偏差

**输入**：我每年体检都正常，应该不会得什么大病，不用买重疾险。

```
Step 1: 识别乐观信号 → 产出：发现"应该不会得什么大病"表述，低估个人风险
Step 2: 评估乐观影响 → 产出：体检正常≠无风险，重疾发生率与年龄相关
Step 3: 分析乐观机制 → 产出：独特优越感：认为自己比统计更幸运；当下偏好：当前健康状态被外推未来
Step 4: 校正乐观偏差 → 产出：建议用年龄别发病率数据评估风险
```

**输出**：
```
## 分析结论
### 核心发现
1. "体检正常"是当前状态，不预测未来风险
2. 35-45岁重疾年发生率约0.3%，10年累积风险约3%
3. 乐观偏差使低估个人风险，认为"我不会那么倒霉"

### 建议（按优先级）
1. [高优先级] 用年龄别发病率数据评估风险，而非当前健康状态
2. [高优先级] 重疾险保费随年龄增长，越早买越便宜
3. [中优先级] 评估家庭病史，识别额外风险因素

### 关键假设
- 发病率数据基于大样本统计
- 个人风险接近平均水平

### 风险
- 乐观偏差可能导致风险保障不足
- 年龄增长后保费显著增加
```

## 反例：不该触发此 Skill 的场景

**输入**：根据过去3年数据，这个项目成功率85%，我们有信心完成。

**原因**：这是基于历史数据的信心（85%成功率），不是无依据的乐观。乐观偏差指信心超过数据支持。
