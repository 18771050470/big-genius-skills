# 常见陷阱

## 陷阱 1：个体故事替代统计数据

**问题**：因个体故事生动而赋予过高权重。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个用户反馈很详细，应该优先解决。
建议：立即修复此问题。
```
*问题：可识别效应使个体反馈替代数据评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 反馈权重分析
| 反馈类型 | 数量 | 占比 | 优先级 |
|---------|------|------|--------|
| 个体详细反馈 | 1 | 0.5% | 低 |
| 登录失败 | 150 | 15% | 高 |
| 加载慢 | 120 | 12% | 高 |

### 建议
1. 个体反馈生动但不普遍
2. 基于数据普遍性排优先级
```
*优势：数据框架评估反馈权重*

**原因**：Small等(2007)实验：具体个体故事比统计数据触发更强捐赠意愿，但实际需求可能相反。

## 陷阱 2：情感响应替代理性评估

**问题**：因情感强弱决定资源分配。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个小女孩的故事太感人了，我们应该捐给她。
建议：大额捐赠。
```
*问题：可识别效应使情感响应替代效率评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 捐赠效率对比
| 选项 | 情感强度 | 每元拯救生命 | 效率 |
|------|---------|------------|------|
| 个体故事 | 高 | 低 | 低 |
| 疟疾蚊帐 | 低 | 高 | 高 |

### 建议
1. 情感触发行动，数据指导方向
2. 选择效率更高的捐赠目标
```
*优势：效率评估替代情感决策*

**原因**：情感启发使生动个体触发更强共情，但可能不是最有效率的帮助目标。

## 陷阱 3：生动性偏差

**问题**：详细信息被误认为更重要。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个反馈写得很详细，问题一定很严重。
建议：优先处理。
```
*问题：可识别效应使详细程度替代问题严重性*

✅ **After（正确做法）**
```markdown
## 分析结论
### 详细程度vs严重性
| 维度 | 评估 |
|------|------|
| 反馈详细程度 | 高(详细描述) |
| 影响用户数 | 低(0.5%) |
| 问题严重性 | 低(非核心功能) |

### 建议
1. 详细≠重要
2. 基于影响范围和严重性评估
```
*优势：区分详细程度和问题严重性*

**原因**：生动性使信息更容易被回忆和重视，但生动性不等于重要性。
