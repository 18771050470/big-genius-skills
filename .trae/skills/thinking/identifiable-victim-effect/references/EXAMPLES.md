# 分析示例

## 示例 1：可识别受害者效应——慈善捐赠决策

**输入**：看到这个生病小女孩的照片和故事，我很想捐款，但看到统计数据说每年有百万人死于贫困，反而没什么感觉。

```
Step 1: 识别可识别信号 → 产出：发现"小女孩的故事想捐款...统计数据没感觉"表述，个体故事比统计数据触发更强情感
Step 2: 评估情感响应 → 产出：个体故事触发强烈共情，统计数据因抽象而情感响应弱
Step 3: 分析可识别机制 → 产出：情感启发：具体个体更容易触发共情；心理距离：统计数据心理距离远
Step 4: 校正可识别效应 → 产出：建议结合情感和理性，用数据指导捐赠决策
```

**输出**：
```
## 分析结论
### 核心发现
1. 个体故事触发强烈情感响应，但可能不是最有效率的捐赠目标
2. 统计数据反映更大规模问题，但情感响应弱
3. Small等(2007)实验：个体故事捐赠额是统计数据的2倍，但实际需求可能相反

### 建议（按优先级）
1. [高优先级] 用数据评估捐赠效率：每元能拯救的生命数
2. [高优先级] 理解情感响应偏差，不因故事强弱决定捐赠
3. [中优先级] 结合情感（触发行动）和理性（指导方向）

### 关键假设
- 数据可评估慈善效率
- 情感响应不等于实际需求

### 风险
- 可识别效应可能导致资源分配不均
- 需要平衡情感和理性
```

## 示例 2：可识别受害者效应——产品用户反馈

**输入**：这个用户发邮件抱怨我们的功能难用，说得很详细，我们应该优先解决这个问题。

```
Step 1: 识别可识别信号 → 产出：发现"这个用户...优先解决"表述，个体反馈被赋予过高权重
Step 2: 评估反馈权重 → 产出：1个详细反馈 vs 1000个用户中只有5个提到此问题，占比0.5%
Step 3: 分析可识别机制 → 产出：生动性：详细反馈更容易想象；情感触发：个体故事触发共情
Step 4: 校正可识别效应 → 产出：建议用数据评估问题普遍性，而非个体反馈生动性
```

**输出**：
```
## 分析结论
### 核心发现
1. 个体反馈详细生动，但只代表0.5%用户
2. 数据反馈显示Top3问题是：登录失败(15%)、加载慢(12%)、支付失败(8%)
3. 个体反馈的生动性不应替代数据普遍性

### 建议（按优先级）
1. [高优先级] 基于数据普遍性排优先级：登录失败>加载慢>支付失败
2. [高优先级] 记录个体反馈，但纳入数据框架评估
3. [中优先级] 回复个体用户，说明改进计划

### 关键假设
- 数据反馈样本具有代表性
- 个体反馈可能是边缘案例

### 风险
- 可识别效应可能导致资源错配
- 需要平衡个体关怀和数据驱动
```

## 反例：不该触发此 Skill 的场景

**输入**：这个用户反馈的问题影响了30%的用户，需要优先修复。

**原因**：这是基于数据普遍性(30%影响率)的决策，不是可识别效应。可识别效应指因个体故事生动而赋予过高权重。
