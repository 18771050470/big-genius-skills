# Critical Path Thinking - 示例

## 示例1：软件版本发布计划

**输入**：需要在6周内发布新版本，包含5个功能模块，用关键路径思维分析。

### Step 1: 列出所有任务
| 任务 | 前置任务 | 预估时间 | 资源 |
|------|---------|---------|------|
| A: 需求评审 | 无 | 3天 | 产品+开发 |
| B: 模块1开发 | A | 10天 | 开发组1 |
| C: 模块2开发 | A | 8天 | 开发组2 |
| D: 模块3开发 | A | 12天 | 开发组3 |
| E: 模块1+2集成 | B,C | 5天 | 开发组1+2 |
| F: 模块3集成 | D | 3天 | 开发组3 |
| G: 全系统集成测试 | E,F | 7天 | QA |
| H: Bug修复 | G | 5天 | 开发 |
| I: 上线准备 | H | 2天 | 运维 |

### Step 2: 构建依赖网络
- A→B→E→G→H→I
- A→C→E→G→H→I
- A→D→F→G→H→I
- 并行任务：B和C和D可以并行，E和F可以并行

### Step 3: 计算关键路径
- 路径1: A(3)+B(10)+E(5)+G(7)+H(5)+I(2) = 32天
- 路径2: A(3)+C(8)+E(5)+G(7)+H(5)+I(2) = 30天
- 路径3: A(3)+D(12)+F(3)+G(7)+H(5)+I(2) = 32天
- 关键路径：路径1和路径3（都是32天）
- 浮动时间：C有2天浮动（32-30=2）

### Step 4: 分析优化机会
- 压缩关键路径上的任务：
  - D(模块3开发)从12天压缩到10天：需要增加1名开发，成本+5万
  - G(集成测试)从7天压缩到5天：需要增加自动化测试，成本+3万
- 压缩后关键路径：32-2-2=28天
- 注意：压缩后C的浮动时间变为0，C成为新的关键路径的一部分

### Step 5: 结论
最短完成时间32天（约6.4周），超过6周目标。优化方案：压缩D和G各2天，总时间28天（约5.6周），成本增加8万。警告：C的浮动时间变为0，任何延迟都会影响整体交付。

## 示例2：营销活动上线计划

**输入**：需要在4周内上线双11营销活动，用关键路径思维分析。

### Step 1: 列出所有任务
| 任务 | 前置任务 | 预估时间 | 资源 |
|------|---------|---------|------|
| A: 活动策划 | 无 | 3天 | 营销 |
| B: 页面设计 | A | 5天 | 设计 |
| C: 页面开发 | B | 7天 | 前端 |
| D: 活动规则法务审核 | A | 4天 | 法务 |
| E: 奖品采购 | D | 5天 | 采购 |
| F: 联调测试 | C,E | 3天 | 开发+QA |
| G: 上线 | F | 1天 | 运维 |

### Step 2: 构建依赖网络
- A→B→C→F→G
- A→D→E→F→G
- 并行任务：B和D可以并行，C和E可以并行

### Step 3: 计算关键路径
- 路径1: A(3)+B(5)+C(7)+F(3)+G(1) = 19天
- 路径2: A(3)+D(4)+E(5)+F(3)+G(1) = 16天
- 关键路径：路径1（19天）
- 浮动时间：D+E有3天浮动（19-16=3）

### Step 4: 分析优化机会
- 压缩关键路径上的任务：
  - C(页面开发)从7天压缩到5天：使用组件库，无需额外成本
  - B(页面设计)从5天压缩到3天：使用模板，无需额外成本
- 压缩后关键路径：19-2-2=15天
- 注意：压缩后D+E的浮动时间变为0+3=3天，仍然有缓冲

### Step 5: 结论
最短完成时间19天（约3.8周），在4周目标内。优化方案：使用组件库和模板压缩B和C，总时间15天（约3周），无额外成本。D+E有3天浮动，可以容忍延迟。
