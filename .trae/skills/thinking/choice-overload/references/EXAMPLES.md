# 分析示例

## 示例 1：选择过载——产品功能设计

**输入**：我们的设置页面应该提供所有可能的选项，让用户自由配置，越多越好。

```
Step 1: 识别过载信号 → 产出：发现"所有可能选项"、"越多越好"表述，选项数量超过认知负荷
Step 2: 评估选项数量 → 产出：设置选项超过20个，用户决策疲劳，可能导致放弃配置
Step 3: 分析过载机制 → 产出：认知负荷：选项越多比较越困难；后悔预期：担心选错导致决策瘫痪
Step 4: 校正选择过载 → 产出：建议分组选项，提供推荐配置，减少初始选项
```

**输出**：
```
## 分析结论
### 核心发现
1. 20+设置选项超过用户认知负荷，Iyengar & Lepper(2000)实验显示6选项比24选项转化率高10倍
2. 用户更可能需要"一键推荐配置"而非完全自由配置
3. 选项过多导致决策疲劳和放弃率增加

### 建议（按优先级）
1. [高优先级] 提供3个预设配置：简单/标准/高级
2. [高优先级] 将高级选项折叠到"高级设置"中
3. [中优先级] 添加"推荐配置"按钮，降低决策成本

### 关键假设
- 80%用户只需要基础配置
- 预设配置覆盖常见场景

### 风险
- 选项过多导致用户放弃配置
- 需要平衡灵活性和易用性
```

## 示例 2：选择过载——餐厅菜单设计

**输入**：菜单上应该有100道菜，让顾客有充分选择，满足不同口味。

```
Step 1: 识别过载信号 → 产出：发现"100道菜"、"充分选择"表述，选项数量导致决策困难
Step 2: 评估选项数量 → 产出：100道菜远超最佳选择数量(5-9个)，顾客可能感到焦虑
Step 3: 分析过载机制 → 产出：比较疲劳：100道菜需要大量比较；满意度下降：选项越多越难满意
Step 4: 校正选择过载 → 产出：建议精简菜单，分类推荐，设置特色菜
```

**输出**：
```
## 分析结论
### 核心发现
1. 100道菜导致选择过载，Miller(1956)证明人类工作记忆容量7±2个组块
2. 选项过多使顾客满意度下降，而非上升
3. 餐厅数据：精简菜单(30道)比丰富菜单(100道)营业额高15%

### 建议（按优先级）
1. [高优先级] 精简到30-40道菜，按类别分组
2. [高优先级] 设置"主厨推荐"(5-7道)，引导选择
3. [中优先级] 季节性轮换菜单，保持新鲜感

### 关键假设
- 80%顾客只点常见菜
- 推荐菜可引导选择

### 风险
- 选项过多降低顾客满意度
- 需要平衡选择和决策负担
```

## 反例：不该触发此 Skill 的场景

**输入**：专业设计软件需要提供所有工具选项，因为专业用户需要精确控制。

**原因**：专业用户有明确需求和领域知识，需要完整工具集。选择过载主要针对无明确偏好的普通用户。
