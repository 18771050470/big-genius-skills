# Bounded Rationality - 示例

## 示例1：技术选型决策

**输入**：创业公司需要选择数据库，有100+选项但只有2周时间决定。

### Step 1: 评估决策约束
- 信息约束：无法测试所有100+数据库
- 时间约束：2周决策截止
- 认知约束：团队只熟悉3-5种数据库
- 资源约束：无预算请外部顾问

### Step 2: 设定满意标准
- 支持ACID事务（必须）
- 社区活跃、文档完善（必须）
- 团队有使用经验或学习成本低（必须）
- 能支撑100万QPS（期望，非必须）
- 依据：当前业务规模+6个月增长预期

### Step 3: 顺序搜索可行方案
- 方案1：PostgreSQL — 满足所有必须条件，学习成本低 → 接受
- 停止搜索

### Step 4: 评估搜索成本与收益
- 继续搜索的边际收益：可能找到更好的数据库
- 边际成本：额外2周延迟上线，机会成本巨大
- 决策：接受PostgreSQL

### Step 5: 结论
选择PostgreSQL。满意标准合理，方案满足所有必须条件。2周后复盘：如果业务增长超预期，重新评估。

## 示例2： hiring决策

**输入**：需要在1个月内招聘一名后端工程师，收到10份简历。

### Step 1: 评估决策约束
- 信息约束：面试只能了解候选人部分能力
- 时间约束：1个月内必须入职
- 认知约束：面试官时间有限
- 资源约束：无猎头预算

### Step 2: 设定满意标准
- 3年以上后端经验（必须）
- 能通过技术面试（必须）
- 薪资在预算范围内（必须）
- 有团队协作经验（期望）

### Step 3: 顺序搜索可行方案
- 面试前3位候选人，第2位满足所有必须条件 → 接受
- 停止搜索

### Step 4: 评估搜索成本与收益
- 继续搜索的边际收益：可能找到更优秀的候选人
- 边际成本：职位空缺继续产生业务损失，其他候选人可能被抢走
- 决策：发出offer

### Step 5: 结论
录用第2位候选人。满意标准合理，1个月后评估入职表现，如果不满意再启动新搜索。
