# 常见陷阱

## 陷阱 1：追求确定性而过度收集信息

**问题**：用更多信息缓解不确定焦虑，但信息边际价值递减。

❌ **Before（错误做法）**
```markdown
## 分析结论
还需要更多数据，再调研2周。
建议：继续调研。
```
*问题：信息偏差使追求确定性，忽视信息边际价值递减*

✅ **After（正确做法）**
```markdown
## 分析结论
### 信息边际价值分析
| 阶段 | 信息量 | 决策信心 | 边际价值 |
|------|--------|---------|---------|
| 当前 | 200问卷+50访谈 | 85% | - |
| +2周调研 | +50访谈 | 88% | 低(+3%) |
| 成本 | - | - | 人力3万+机会成本10万 |

### 建议
1. 边际价值(3%)远低于成本(13万)
2. 基于现有数据决策，用MVP验证剩余不确定性
```
*优势：量化信息边际价值，揭示继续调研不划算*

**原因**：Baron等(1988)证明：人们倾向于收集即使对决策无影响的信息。更多信息不必然导致更好决策。

## 陷阱 2：将信息量等同于决策质量

**问题**：认为信息越多决策越好，忽视信息质量和决策框架。

❌ **Before（错误做法）**
```markdown
## 分析结论
我们收集了1000份数据，决策质量肯定高。
建议：基于这些数据决策。
```
*问题：信息偏差使信息量替代信息质量*

✅ **After（正确做法）**
```markdown
## 分析结论
### 信息质量评估
| 维度 | 评估 |
|------|------|
| 样本代表性 | 中(偏向年轻用户) |
| 问题设计 | 低(引导性问题) |
| 数据时效性 | 低(3个月前收集) |

### 建议
1. 1000份数据量充足，但质量有问题
2. 重新收集100份高质量数据优于1000份低质量数据
```
*优势：评估信息质量，而非仅关注数量*

**原因**：信息偏差使关注信息数量而非质量。低质量大量信息可能导致错误决策。

## 陷阱 3：用信息收集替代决策

**问题**：以"需要更多信息"为由延迟决策。

❌ **Before（错误做法）**
```markdown
## 分析结论
信息还不够，等有更多数据再决定。
建议：继续收集信息。
```
*问题：信息偏差使信息收集成为决策延迟的合理化理由*

✅ **After（正确做法）**
```markdown
## 分析结论
### 决策时机分析
| 因素 | 评估 |
|------|------|
| 信息充足度 | 85%(足够决策) |
| 延迟成本 | 高(市场窗口期2周) |
| 不确定性来源 | 执行层面(非信息层面) |

### 建议
1. 信息已足够，不确定性在执行中验证
2. 设定决策截止时间：本周五
3. 用MVP验证剩余不确定性
```
*优势：识别决策时机，区分信息不足和执行不确定*

**原因**：信息偏差常被用作决策延迟的合理化。不确定性焦虑使偏好"再等等"。
