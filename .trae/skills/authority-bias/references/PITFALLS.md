# 常见陷阱

## 陷阱 1：用权威身份替代论点评估

**问题**：因推荐人是权威而接受方案，不评估方案本身质量。

❌ **Before（错误做法）**
```markdown
## 分析结论
Google首席工程师推荐的方案，肯定没问题。
建议：直接采用。
```
*问题：权威偏差使论点被不加批判接受，忽视适用性分析*

✅ **After（正确做法）**
```markdown
## 分析结论
### 方案独立评估
| 评估维度 | 评分 | 说明 |
|---------|------|------|
| 功能匹配 | 7/10 | 覆盖核心需求，但过度设计 |
| 场景适配 | 5/10 | Google场景有效≠我们场景有效 |
| 学习成本 | 4/10 | 团队需3个月学习期 |
| 社区支持 | 3/10 | 社区小，问题难解决 |

### 建议
1. 方案本身评分5/10，不推荐直接采用
2. 对比3个备选方案，做POC验证
```
*优势：独立评估方案质量，不受推荐人身份影响*

**原因**：Milgram(1963)服从实验证明65%参与者因权威指令而伤害他人。权威身份使批判性思维关闭，论点被默认接受。

## 陷阱 2：责任外包给权威

**问题**：依赖权威免除自身验证责任。

❌ **Before（错误做法）**
```markdown
## 分析结论
四大出的合规方案，肯定符合法规。
建议：直接执行，不需要再验证。
```
*问题：权威偏差使将合规责任外包给机构，忽视自身验证义务*

✅ **After（正确做法）**
```markdown
## 分析结论
### 合规验证结果
| 法规章节 | 方案覆盖 | 验证结果 |
|---------|---------|---------|
| 第一章 总则 | 是 | 符合 |
| 第二章 数据收集 | 是 | 符合 |
| 第三章 数据跨境 | 部分 | 遗漏最新修订条款 |
| 第四章 行业特殊要求 | 否 | 通用模板未覆盖 |

### 建议
1. 方案覆盖85%，需补充15%未覆盖部分
2. 合规责任最终在我们，不在机构
```
*优势：逐条验证揭示覆盖缺口，明确自身责任*

**原因**：权威偏差使信任转移到方案质量，产生"权威负责"错觉。但合规责任最终在使用方。

## 陷阱 3：忽视权威的领域匹配

**问题**：将某领域权威视为所有领域权威。

❌ **Before（错误做法）**
```markdown
## 分析结论
这位诺贝尔物理学奖得主说这个经济政策好，肯定对。
建议：采纳政策建议。
```
*问题：物理学权威≠经济学权威，领域不匹配*

✅ **After（正确做法）**
```markdown
## 分析结论
### 领域匹配分析
| 维度 | 评估 |
|------|------|
| 权威领域 | 物理学（诺贝尔奖） |
| 建议领域 | 经济政策 |
| 匹配度 | 低（跨领域） |
| 经济学界共识 | 该政策存在争议 |

### 建议
1. 物理学权威在经济政策上无特殊权威性
2. 参考经济学界共识和实证研究
```
*优势：识别领域匹配度，防止跨领域权威误用*

**原因**：权威偏差使权威身份被泛化，忽视领域边界。权威只在特定领域内有效。
