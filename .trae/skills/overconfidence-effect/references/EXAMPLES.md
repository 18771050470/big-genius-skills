# 分析示例

## 示例 1：过度自信效应——技术能力评估

**输入**：这个新技术我学2周就能掌握，之前学类似的技术我也很快上手。

```
Step 1: 识别过度自信信号 → 产出：发现"2周就能掌握"表述，能力估计超过实际水平
Step 2: 评估自信程度 → 产出：自信度90%，但历史数据显示类似技术平均需要6周掌握
Step 3: 分析过度自信机制 → 产出：优于平均效应：认为自己比一般人强；确认偏误：回忆成功案例忽视失败
Step 4: 校正过度自信 → 产出：建议用历史数据和基准率校准自信度
```

**输出**：
```
## 分析结论
### 核心发现
1. 自信度90%，但历史数据显示平均需要6周，非2周
2. 过度自信效应在技术领域普遍存在，80-90%开发者认为优于平均
3. "很快上手"可能只覆盖基础，精通需要更长时间

### 建议（按优先级）
1. [高优先级] 用历史数据校准：6周掌握
2. [高优先级] 分阶段学习：2周基础+4周精通
3. [中优先级] 设置学习里程碑，每月评估进度

### 关键假设
- 历史数据可参考
- 技术复杂度相似

### 风险
- 过度自信可能导致时间估计不足
- 需要现实预期
```

## 示例 2：过度自信效应——创业成功率

**输入**：我们创业成功率有80%，团队经验丰富，市场机会好。

```
Step 1: 识别过度自信信号 → 产出：发现"80%成功率"表述，远超行业基准
Step 2: 评估自信程度 → 产出：自信80%，但行业基准：首年失败率20%，5年失败率50%
Step 3: 分析过度自信机制 → 产出：控制错觉：高估自己对结果的控制；规划谬误：低估困难
Step 4: 校正过度自信 → 产出：建议用行业基准率校准自信度
```

**输出**：
```
## 分析结论
### 核心发现
1. 自信80%成功率，行业基准5年成功率50%
2. 过度自信使高估控制力，低估市场不确定性
3. 经验丰富是优势，但不保证成功

### 建议（按优先级）
1. [高优先级] 基于行业基准调整预期：50%成功率
2. [高优先级] 识别不可控风险因素
3. [中优先级] 设置风险缓解计划

### 关键假设
- 行业基准适用于当前情况
- 经验可降低但不可消除风险

### 风险
- 过度自信可能导致准备不足
- 需要平衡自信和现实
```

## 反例：不该触发此 Skill 的场景

**输入**：根据过去10次考试数据，我平均分90%，这次估计90分左右。

**原因**：这是基于历史数据的校准估计(10次考试平均分90%)，不是过度自信。过度自信指自信度超过数据支持。
