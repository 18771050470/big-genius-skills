# 场景映射表

> 本文件定义场景类型、关键词特征与技能链的映射关系。
> 新增场景或调整关键词时直接修改本文件。
> 路由规则见 [routing-rules.md](routing-rules.md)

---

## 场景映射

| 场景类型 | 关键词特征 | 组合原则 | 说明 |
|---------|-----------|---------|------|
| 新功能开发 | 创建、实现、开发、添加、做一个 | 思维层 → 角色层 → 工具层 | 先分析再执行 |
| 问题修复 | 修复、bug、报错、异常、错误 | 思维层 → 角色层 | 先逆向分析再调试 |
| 架构决策 | 选型、方案、对比、设计、架构 | 思维层 → 角色层 | 多维度评估 |
| 代码审查 | 审查、review、检查、优化代码 | 思维层 → 角色层 | 先建立标准再检查 |
| 性能优化 | 优化、慢、卡顿、性能、提升 | 思维层 → 角色层 → 工具层 | 分析→优化→验证 |
| 文档处理 | 文档、报告、表格、演示、写作 | 工具层 | 直接路由到工具层 |
| 简单任务 | 改一下、小改动、微调、重命名 | 角色层 | 跳过思维层，直接执行 |
| 学习研究 | 学习、了解、研究、解释、原理 | 思维层 → 角色层 | 先拆解再探索 |

> 具体技能由 Commander 根据上下文动态选择，不预设固定技能链。
> 参考：layer-skills.md 查看各层级可用技能。

## 扩展指南

### 新增场景

1. 在表格中添加新行
2. 定义清晰的关键词特征
3. 指定技能链（思维模型 → 角色 → 工具）
4. 填写说明列

### 关键词匹配优先级

1. 精确匹配场景标签（如 `/start bug-fix`）
2. 关键词匹配（按表格顺序，先匹配优先）
3. 默认降级到"学习研究"场景

## 更新说明

- 新增场景：添加表格行
- 调整关键词：修改关键词特征列
- 变更技能链：修改技能链列
