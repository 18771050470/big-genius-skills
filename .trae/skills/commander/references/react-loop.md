# ReAct 循环控制

> 本文件定义 ReAct 循环的机制、各阶段职责与迭代策略。
> 调整循环逻辑时直接修改本文件。

---

## 循环定义

```
┌─────────────────────────────────────────────────────────────┐
│                    ReAct 循环                                │
│                                                              │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │ 思考     │───▶│ 行动     │───▶│ 观察     │             │
│   │ Reason   │    │ Act      │    │ Observe  │             │
│   └──────────┘    └──────────┘    └──────────┘             │
│        ▲                                    │               │
│        └────────────────────────────────────┘               │
│                    循环直到完成                              │
└─────────────────────────────────────────────────────────────┘
```

## 各阶段职责

### 思考阶段（Reason）

- 分析用户输入的意图、领域、任务类型
- 输出：任务特征向量 `{intent, domain, task_type, complexity, risk}`

### 行动阶段（Act）

- 根据任务特征选择技能链
- 依次执行技能，传递上下文
- 输出：技能执行结果

### 观察阶段（Observe）

- 评估执行结果是否满足需求
- 识别遗留问题或优化空间
- 输出：需求满足度评分 + 改进建议

### 循环阶段（Repeat）

- 如果满足度 < 100%，补充技能
- 重新进入行动阶段
- 最多循环 3 次，避免无限循环

## 迭代策略

| 循环次数 | 条件 | 动作 |
|---------|------|------|
| 第1次 | 初始执行 | 按路由规则执行技能链 |
| 第2次 | 满足度 60%-99% | 补充缺失技能，重新执行 |
| 第3次 | 满足度 30%-59% | 调整技能链，重新执行 |
| 终止 | 满足度 < 30% 或已达3次 | 输出当前最佳结果，说明原因 |

## 更新说明

- 调整阶段职责：直接修改对应阶段内容
- 变更迭代策略：更新迭代策略表格
- 修改循环次数限制：更新循环阶段说明
