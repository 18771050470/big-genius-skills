# 路由规则

> 本文件是意图识别的核心承担者，定义完整的决策流程、复杂度评估和路由规则。
> SYSTEM_PROMPT 只定极简模式阈值（<30字符），AGENT_CORE_RULES 只定加载纪律。
> 所有具体识别逻辑集中在此文件。
> 场景→技能链映射由 [scenario-map.md](scenario-map.md) 提供。

---

## 意图识别规则

### 快速路径触发（极简模式）

满足**任一**条件即跳过分析，直接执行：

| 条件 | 示例 |
|------|------|
| 用户明确指定 Skill | "用 qa-engineer 检查这段代码" |
| 输入 < 30 字符 | "优化一下"、"改改这个" |
| 任务明确且步骤 ≤ 3 | "帮我写个函数"、"改个变量名" |
| 纯执行型任务 | "运行测试"、"格式化代码" |

**快速路径执行流程**：
```
用户输入 → 提取关键词 → 匹配 scenario-map.md → 直接调用 Skill → 输出结果
```

### 标准路径触发

满足**任一**条件进入完整分析流程：

| 条件 | 示例 |
|------|------|
| 任务复杂（步骤 > 3） | "设计一个完整的用户系统" |
| 用户请求不明确 | "帮我看看这个项目" |
| 涉及多个 Skill 协调 | "分析并优化性能" |
| 涉及架构变更 | "重构整个模块" |

**标准路径执行流程**：
```
用户输入 → 意图识别 → 复杂度评估 → 层级路由 → 技能链组装 → 执行与监控 → 结果整合
```

## 路由决策流程

```
接收用户输入
    │
    ├── 步骤1：意图识别
    │   ├── 检查快速路径触发条件
    │   ├── 提取关键词（匹配 scenario-map.md）
    │   ├── 识别场景标签
    │   └── 评估复杂度与风险
    │
    ├── 步骤2：层级路由
    │   ├── 是否需要思维框架？ → 路由到 L1（思维层）
    │   ├── 是否需要具体执行？ → 路由到 L2（角色层）
    │   └── 是否需要工具/文档？ → 路由到 L3（工具层）
    │
    ├── 步骤3：技能链组装
    │   ├── 确定执行顺序
    │   ├── 检查依赖关系
    │   └── 验证技能可用性
    │
    ├── 步骤4：执行与观察
    │   ├── 依次执行技能链
    │   ├── 收集中间结果
    │   └── 评估需求满足度
    │
    └── 步骤5：迭代或完成
        ├── 满足需求 → 汇总输出
        └── 不满足   → 补充技能 → 返回步骤3
```

## 复杂度评估

| 复杂度 | 判断标准 | 路由策略 |
|--------|---------|---------|
| 低 | 单文件修改、明确指令、无风险 | 直接路由到 L2 |
| 中 | 多文件协调、需要设计、有一定风险 | L1（简化）→ L2 |
| 高 | 架构变更、多模块、高风险 | L1（完整）→ L2 → L3/L4 |

## 路由决策树

```
if 场景 == "文档处理":
    路由到 L3（工具层 document 子类）

if 场景 == "简单任务":
    路由到 L2（角色层），跳过 L1

if 风险 == "高":
    前置 L1 风险分析

if 复杂度 == "高":
    L1（完整）→ L2 → L3

if 复杂度 == "中":
    L1（简化）→ L2

if 复杂度 == "低":
    L2
```

## Skill 调用指导

### 调用方式

1. **用户显式指定**：用户直接说"用 skill-xxx 处理"
2. **关键词匹配**：用户描述中的关键词与 Skill description 匹配
3. **Commander 判断**：根据任务类型和复杂度，主动选择合适的 Skill

> Agent 不负责自动扫描目录。Skill 列表由运行环境提供，或用户显式告知。

### 场景→技能链映射说明

Commander 根据场景动态组装技能链，以下为组合原则说明：

| 场景类型 | 组合原则 |
|---------|---------|
| 分析型任务 | 思维层(L1) → 角色层(L2) |
| 执行型任务 | 角色层(L2) 或 工具层(L3/L4) |
| 复杂任务 | 思维层(L1) → 角色层(L2) → 工具层(L3/L4) |
| 简单任务 | 直接路由到对应角色层 |
| 文档处理 | 工具层 document 子类 |

> 具体技能链由 Commander 根据上下文动态决定，不预设固定组合。
> 参考：layer-skills.md 查看各层级可用技能
