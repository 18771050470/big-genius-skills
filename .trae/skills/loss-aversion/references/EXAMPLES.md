# EXAMPLES.md

## 示例1：个人投资决策——是否卖出亏损股票

### 场景描述
用户持有股票A，当前价格**100元**，买入成本**120元**（浮亏20元）。现需决定：
- **选项X（卖出）**：确认亏损20元（损失锁定）
- **选项Y（持有）**：50%概率再涨回120元（收益20元），50%概率跌至80元（再亏20元，总亏40元）

### 分析过程

#### 第1步：识别决策境
决策框架为“确定损失 vs 不确定的更差/更好局面”。用户自然偏向**持有**（因为卖出立刻承受痛苦）。

#### 第2步：量化损失与收益
| 选项 | 损失 | 收益 | 概率说明 |
|------|------|------|----------|
| X（卖出） | 确定损失20元 | 0 | 100%发生 |
| Y（持有） | 50%概率损失20元（总亏40） | 50%概率收益20元（回本） | 合计期望损失=0 |

**期望值计算**：X期望=-20元，Y期望=0.5×(-40) + 0.5×0 = -20元。**两者完全相等**。

#### 第3步：评估情绪不对称
- X：确定损失20元 → 痛苦感受 ≈ 2×20 = 40单位痛苦
- Y：50%概率损失20元（痛苦2×20×0.5=20单位） + 50%概率收益20元（快乐1×20×0.5=10单位） → 净感受 = -20+10 = -10单位痛苦

比较：X的痛苦（40） > Y的痛苦（10）。**情绪上强烈偏向Y**。

#### 第4步：检查现状偏见
现状是“持有”，且用户并未主动选择该股票的成本（已发生）。提问：“如果你现在没有这只股票，且资金已变现，你会选择以当前100元买入持有吗？” —— 大多数用户会回答“不会”，说明现状被惯性高估。

#### 第5步：重新框架视角
- **框架A（原始）**：“卖出会亏损20元吗？但持有可能回本。”
- **框架B（反转）**：“如果你此刻有100元现金，你会买入这只股票吗？（买入可能盈利20或亏损20）” —— 在框架B下，选择买入的倾向会大幅降低，因为损失厌恶使“不确定的未来”看上去更糟。

#### 第6步：理性校正
期望值相同，但考虑持有需承担额外时间成本与机会成本。理性看，如果资金有时间成本，卖出并投资其他资产可能更优。将损失权重从2下调至1后，X和Y的心理差异消失。

#### 第7步：权衡与建议
- 损失厌恶让用户倾向于“不卖”，但理性分析表明卖出与持有期望值相同。
- 建议：**若资金有时间成本且有更好标的，卖出；若股票基本面未恶化且用户风险承受力高，持有。** 明确指出损失厌恶是此刻“不想卖”的主要情绪来源，建议忽略“锁亏”的痛苦。

### 最终输出
```
决策建议：卖出
关键依据：损失厌恶导致用户高估了“确认亏损”的痛苦，而低估了“继续亏损”的可能。  
理性比较：期望值相同，但持有需承担额外不确定性，且现状偏见被纠正后建议卖出。  
损失厌恶影响程度：强——情绪上强烈偏向持有，校正后才做出卖出决定。
```

## 示例2：产品功能取舍——是否上线新功能

### 场景描述
产品经理评估是否上线一个“简化注册流程”的新功能。数据预测：
- **收益**：注册转化率提升**10%** → 每月多获**5000**新用户
- **损失**：新流程可能增加技术复杂度，导致页面加载时间增加，预计流失率上升**2%** → 每月损失**1000**存量用户

### 分析过程

#### 第1步：识别决策境
决策框架：用“1000存量用户损失”换“5000新增用户收益”。团队普遍更担心那1000人的流失，而对5000人的增长没那么有感觉。

#### 第2步：量化损失与收益
| 选项 | 损失 | 收益 |
|------|------|------|
| 上线新功能 | 1000用户流失（确定损失） | 5000用户新增（确定收益） |
| 维持现状 | 0用户损失 | 0用户新增 |

#### 第3步：评估情绪不对称
- 损失：1000流失的痛苦 ≈ 2×1000 = 2000单位痛苦
- 收益：5000新增的快乐 ≈ 1×5000 = 5000单位快乐
- 净感受：5000 - 2000 = 3000积极（但仍不及原始数字比例明显）

然而，团队反馈是“流失用户是付费用户，新增用户可能免费”，因此实际痛苦权重可能更高。

#### 第4步：检查现状偏见
现状是“没有新功能”，团队已经习惯。提问：“如果新功能已上线，现在考虑是否下线，你会选择下线吗？” —— 多数人不会（因为下线也会损失用户），说明现状偏见程度中等。

#### 第5步：重新框架视角
- **框架A（强调损失）**：“新功能会导致1000用户失望离开。”
- **框架B（强调收益）**：“不上线新功能意味着每月错失5000获客机会。” —— 在框架B下，不行动的“损失”被凸显，团队更倾向上线。

#### 第6步：理性校正
净绝对值：+4000用户（5000-1000），收益显著大于损失。将损失权重从2降为1，则情绪净效果为5000-1000=4000，与理性值一致。

#### 第7步：权衡与建议
- 上线决策从情绪到理性都是正面的，核心阻碍就是损失厌恶放大了“流失1000”的负面感受。
- 建议：**上线，但需要监控加载时间，确保流失率控制在2%以内。** 明确告知团队：对流失的过度担忧是损失厌恶在作祟，理性上这是正确的决策。

### 最终输出
```
决策建议：上线新功能
关键依据：净收益（+4000用户）远超损失厌恶带来的情绪负担。  
理性比较：上线净得4000用户，远优于0。  
损失厌恶影响程度：中——团队起初反对上线，但经框架反转和理性计算后转为支持。
```

## 反例（不该触发本 Skill）

### 场景描述
用户问：“今天中午我应该吃面条还是米饭？”

### 分析
- 该决策不涉及**损失 vs 收益**的不对称风险。
- 没有“失去已有资源”或“放弃潜在收益”的权衡。
- 选择面条不代表损失米饭，只是偏好差异。
- 不需要损失厌恶分析：直觉选择即可，或使用营养学、心情偏好等模型。

### 正确做法
不调用本Skill，改用其他思维模型（如“多维偏好排序”或“随机决策”）或跳过。
