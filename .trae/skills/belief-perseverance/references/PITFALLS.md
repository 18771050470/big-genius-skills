# 常见陷阱

## 陷阱 1：折扣负面证据

**问题**：用合理化解释discount负面证据，维持原有信念。

❌ **Before（错误做法）**
```markdown
## 分析结论
用户说功能不好用是因为他们没理解价值。
建议：继续优化，加强用户教育。
```
*问题：信念坚持使负面证据被合理化，而非被接受*

✅ **After（正确做法）**
```markdown
## 分析结论
### 证据权重评估
| 证据类型 | 方向 | 权重 | 说明 |
|---------|------|------|------|
| 用户调研 | 负面 | 高 | 样本200人，代表性强 |
| 使用数据 | 负面 | 高 | 使用率<1% |
| "用户没理解" | 中性 | 低 | 无实证支持的假设 |

### 建议
1. 负面证据权重远高于中性假设
2. 设定明确标准：使用率<5%则砍掉
```
*优势：客观评估证据权重，防止合理化discount负面证据*

**原因**：Anderson等(1980)证明：即使初始证据被证明是假的，人们仍坚持基于它形成的信念。信念一旦形成就具有独立性。

## 陷阱 2：沉没成本加剧信念坚持

**问题**：因已投入资源而不愿放弃原有信念。

❌ **Before（错误做法）**
```markdown
## 分析结论
这个功能我们已经做了3人月，不能就这么放弃。
建议：继续投入直到成功。
```
*问题：信念坚持+沉没成本使继续投入，而非理性评估*

✅ **After（正确做法）**
```markdown
## 分析结论
### 决策分析
| 方案 | 额外投入 | 预期收益 | 成功率 |
|------|---------|---------|--------|
| 继续优化 | 2人月 | 使用率可能到3% | 低 |
| 重新设计 | 4人月 | 使用率可能到10% | 中 |
| 砍掉 | 0 | 释放人力做高价值功能 | - |

### 建议
1. 已投入3人月是沉没成本，不应影响决策
2. 砍掉功能释放人力做高价值功能ROI更高
```
*优势：分离沉没成本和未来决策，理性评估选项*

**原因**：信念坚持常与沉没成本叠加。已投入资源使放弃心理成本更高。

## 陷阱 3：设计者偏见

**问题**：自己设计的方案不愿承认缺陷。

❌ **Before（错误做法）**
```markdown
## 分析结论
压测结果有问题，我的架构设计不可能扛不住。
建议：重新做压测。
```
*问题：设计者偏见使拒绝接受负面测试结果*

✅ **After（正确做法）**
```markdown
## 分析结论
### 偏见识别
1. 方案设计师可能有设计者偏见
2. 建议第三方团队复现压测
3. 对比压测场景与实际流量差异

### 建议
1. 消除利益冲突：第三方复现
2. 用数据说话，而非设计者直觉
```
*优势：识别设计者偏见，引入独立验证*

**原因**：Ross等(1975)证明：人们对自己形成的信念有强烈坚持，即使证据被推翻。设计者对自己方案有情感依恋。
