# 分析示例

## 示例 1：公正世界假设——员工绩效评估

**输入**：小王被裁员了，可怜之人必有可恨之处，肯定是他工作不努力。

```
Step 1: 识别公正世界信号 → 产出：发现"可怜之人必有可恨之处"表述，将不幸归因于受害者自身
Step 2: 评估归因模式 → 产出：忽视系统性原因（公司战略调整、部门裁撤、经济环境）
Step 3: 分析公正世界机制 → 产出：控制需求：相信世界公正提供安全感；恐惧管理：责备受害者使自己感觉安全
Step 4: 校正公正世界假设 → 产出：建议多因素归因，考虑系统、随机、个人因素
```

**输出**：
```
## 分析结论
### 核心发现
1. "可怜之人必有可恨之处"是公正世界假设的典型表述
2. 裁员真实原因：公司战略调整，整个部门裁撤30%，非个人绩效问题
3. 小王绩效评估：过去4个季度都是B+，处于中等偏上

### 建议（按优先级）
1. [高优先级] 多因素归因：战略调整(主因)+部门预算(次因)+个人绩效(非主因)
2. [高优先级] 理解裁员是系统性决策，非个人失败
3. [中优先级] 为被裁员工提供转职支持

### 关键假设
- 公司战略调整是裁员主因
- 小王绩效处于中等偏上

### 风险
- 公正世界假设可能导致对受害者的二次伤害
- 忽视系统性原因可能错过重要信号
```

## 示例 2：公正世界假设——创业失败归因

**输入**：他创业失败了，肯定是能力不行，不然怎么会失败呢？

```
Step 1: 识别公正世界信号 → 产出：发现"肯定是能力不行"表述，将失败归因于个人缺陷
Step 2: 评估归因模式 → 产出：忽视随机因素（市场时机、运气）和系统因素（融资环境、竞争格局）
Step 3: 分析公正世界机制 → 产出：责备受害者维持"世界公正"信念；控制错觉：认为失败可完全避免
Step 4: 校正公正世界假设 → 产出：建议分析创业失败的多因素原因
```

**输出**：
```
## 分析结论
### 核心发现
1. 创业失败归因于"能力不行"是公正世界假设
2. 创业成功率统计：首年失败率20%，5年失败率50%，10年失败率70%
3. 失败常见原因：市场时机(35%)、资金链断裂(25%)、竞争(20%)、团队(15%)、其他(5%)

### 建议（按优先级）
1. [高优先级] 理解创业失败是常态，非个人缺陷证明
2. [高优先级] 分析具体失败原因，而非简单归因于能力
3. [中优先级] 从失败中学习，而非责备

### 关键假设
- 创业成功率统计数据可靠
- 失败原因可多因素分析

### 风险
- 公正世界假设可能使不敢创业（害怕失败=能力不行）
- 需要区分可学习的失败和不可控的失败
```

## 反例：不该触发此 Skill 的场景

**输入**：他因连续3个月业绩垫底被辞退，这是明确的绩效原因。

**原因**：这是基于明确绩效数据的合理归因，有客观证据支持，不是公正世界假设。
