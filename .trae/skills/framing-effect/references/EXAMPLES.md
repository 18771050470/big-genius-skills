# 分析示例

## 示例 1：框架效应——损失vs收益框架

**输入**：这个手术有90%成功率，应该做。vs 这个手术有10%失败率，要慎重。

```
Step 1: 识别框架信号 → 产出：发现同一数据两种表述：90%成功(收益框架) vs 10%失败(损失框架)
Step 2: 评估框架影响 → 产出：收益框架下84%选择手术，损失框架下仅50%选择
Step 3: 分析框架机制 → 产出：损失厌恶：损失框架触发风险规避；情感反应：失败比成功引发更强情感
Step 4: 校正框架效应 → 产出：建议同时呈现两种框架，基于绝对数据决策
```

**输出**：
```
## 分析结论
### 核心发现
1. 90%成功率=10%失败率，数据相同但框架不同
2. 收益框架下选择率84%，损失框架下50%，框架效应显著
3. 决策应基于绝对数据(90%成功/10%失败)，而非表述方式

### 建议（按优先级）
1. [高优先级] 同时呈现收益和损失框架，减少单一框架影响
2. [高优先级] 基于绝对数据决策：90%成功率是否可接受
3. [中优先级] 对比其他治疗方案的成功率

### 关键假设
- 90%数据来自可靠统计
- 患者风险偏好可接受10%失败率

### 风险
- 单一框架可能导致决策偏差
- 需要区分框架效应和真实风险差异
```

## 示例 2：框架效应——产品定价

**输入**：这个功能单独买500元，但包含在套餐里只要200元。vs 这个套餐比单独买节省了300元。

```
Step 1: 识别框架信号 → 产出：发现两种表述：套餐价200元(价格框架) vs 节省300元(节省框架)
Step 2: 评估框架影响 → 产出：节省框架使感知价值更高，触发获得感
Step 3: 分析框架机制 → 产出：获得框架：节省被视为获得；参照点：单独买500元成为参照点
Step 4: 校正框架效应 → 产出：建议基于实际需求评估，而非节省感知
```

**输出**：
```
## 分析结论
### 核心发现
1. "节省300元"是获得框架，使套餐显得更划算
2. 关键问题：是否需要套餐中的其他功能？
3. 如不需要其他功能，单独买500元的功能实际成本仍是500元

### 建议（按优先级）
1. [高优先级] 评估是否需要套餐中的其他功能
2. [高优先级] 如只需要单一功能，单独购买更合适
3. [中优先级] 计算套餐中各功能实际使用价值

### 关键假设
- 套餐中其他功能有独立价值
- 不需要为"节省"而购买不需要的东西

### 风险
- 节省框架可能诱导过度购买
- 需要区分"划算"和"需要"
```

## 反例：不该触发此 Skill 的场景

**输入**：方案A成本100元收益150元，方案B成本100元收益120元。

**原因**：这是客观数据对比，没有同一数据的不同表述框架，不涉及框架效应。
