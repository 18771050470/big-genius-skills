# PITFALLS.md

## 陷阱1：过度拟合——把现实硬塞进“看起来很美”的模型

- **问题描述**：在步骤2-3中，为了使用自己熟悉的或看起来精致的模型，忽略模型假设与实际数据的明显不符，强行匹配。例如，对一个呈现周期性波动的数据，硬套线性回归模型，结果得出“稳步增长”的荒谬结论。

- **❌ 反例**：
  一家电商公司连续12个月的销售额数据 [200, 350, 280, 420, 390, 520, 470, 610, 560, 700, 640, 810]。一名分析师喜欢指数模型，带入数据后得出指数函数 y = 150 * exp(0.13x)，R²高达0.94，预测下月销售额为940万元。但实际上，数据有明显的季节性（Q4高峰、Q1回调），指数模型忽视了周期性。真实下月销售额仅为740万元（因Q1回调），预测误差超过20%。

- **✅ 正例**：
  相同数据，分析师先进行步骤2——列出多个候选模型：指数增长、线性增长、带有季节成分的分解模型。在步骤3中，他发现残差在每年末呈正偏，年初负偏，于是选择“乘法季节性+线性趋势”模型（如Holt-Winters）。预测下月销售额为755万元，与实际偏差仅2%。

- **原因分析**：指数模型虽然给出了高拟合度（R²），但那是由于趋势项掩盖了季节成分；分析师被“高R²”欺骗，未检查残差模式。模型思维的早期阶段（步骤2-3）必须进行模型假设与数据特征的严谨对比，不能只看整体拟合优度。任何模型在某种程度上都是“错的”，但如果我们武断选择“好看的”模型，就放弃了“有用的”可能性。

## 陷阱2：忽略模型的局限性，把模型输出当作绝对真理

- **问题描述**：在步骤5的验证和步骤6的决策中，忘记提醒自己模型在“哪些方面是错的”，直接依据模型结论采取重大行动，而不与经验、其他数据或定性判断进行交叉验证。

- **❌ 反例**：
  一家金融机构使用信用评分模型（逻辑回归）来批准贷款。模型显示某客户违约概率为3%，低于5%的阈值，于是自动批准贷款。但模型未考虑该客户近期失业且依赖一次性收入（模型训练数据缺乏经济衰退期的样本）。最终客户违约，公司损失数十万。事后分析发现，模型的假设是“经济环境稳定”，当环境改变时，模型的预测能力下降。

- **✅ 正例**：
  同一家机构在模型输出基础上增加“人为复核”机制：当违约概率在3%~5%之间时，要求信贷员人工审查。审查员发现该客户近期收入不稳定，于是下调评级，拒绝贷款或要求担保。同时，模型团队将“失业率变化”作为一个新特征纳入模型，并每月重新评估假设是否仍然成立。

- **原因分析**：所有模型都是基于特定时间段、特定人群、特定假设的简化。模型输出是“在假设成立条件下的预测”，而非“现实必然”。将模型当作绝对真理，本质上是忘了乔治·博克斯的警示。在步骤5中，敏感性分析和脆弱点清单如果只是流于形式，没有落实到决策层，就会付出代价。正确的做法是：无论模型结果多漂亮，都必须追问“这个模型可能在哪些方面失效？”，并用更宽的信息源进行校准。
