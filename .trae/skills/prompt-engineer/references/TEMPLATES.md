# 技术交付物模板

> 提供可直接使用的 Markdown/代码模板。

## 交付物模板 1：Prompt 设计文档

```markdown
# Prompt 设计文档: [Prompt 名称]

## 设计目标
- **任务类型**: [分类/生成/抽取/推理/...]
- **输入**: ...
- **输出**: ...
- **质量目标**: ...

## Prompt 结构

```
[角色定义]
你是 [角色]，擅长 [能力]。

[任务描述]
请根据以下 [输入类型] 完成 [任务]。

[输入格式]
{{input}}

[处理规则]
1. 规则1：...
2. 规则2：...

[输出格式]
请按以下格式输出：
```json
{
  "field1": "...",
  "field2": "..."
}
```

[约束条件]
- 约束1：...
- 约束2：...
```

| 示例编号 | 输入 | 输出 | 设计意图 |

| 1 | ... | ... | 覆盖正常情况 |

| 3 | ... | ... | 覆盖错误情况 |

## 评估结果

| 测试集 | 样本数 | 准确率 | 备注 |
|--------|--------|--------|------|
| 正常输入 | 100 | 95% | ... |

| 对抗输入 | 30 | 92% | ... |

## 已知问题
- ...
- ...

## 优化建议
- ...
```

## 交付物模板 2：Prompt 版本控制规范

```markdown
# Prompt 版本: [名称] v[X.Y.Z]

## 变更类型
- [ ] Major：不兼容变更
- [ ] Minor：功能新增
- [ ] Patch：问题修复

## 变更内容
[详细描述本次变更]

## 影响评估

| 维度 | 影响 | 说明 |
|------|------|------|
| 输出格式 | 是/否 | ... |
| 性能 | 是/否 | ... |
| 成本 | 是/否 | ... |
| 兼容性 | 是/否 | ... |

## 回滚方案
[如变更失败，如何回滚到上一版本]

## 验证结果
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] A/B 测试通过
```

## 交付物模板 3：Prompt 安全审查清单

```markdown
# Prompt 安全审查: [Prompt 名称]

## 输入安全
- [ ] 用户输入是否经过过滤/转义？
- [ ] 是否限制了输入长度？
- [ ] 是否处理了特殊字符？
- [ ] 是否防御了注入攻击？

## 输出安全
- [ ] 是否设置了输出长度限制？
- [ ] 是否过滤了敏感信息？
- [ ] 是否禁止了危险操作指令？
- [ ] 是否添加了内容安全审查？

## 权限控制
- [ ] 系统提示是否包含权限边界？
- [ ] 是否区分了不同用户角色的权限？
- [ ] 是否防止了权限提升？

## 测试覆盖
- [ ] 正常输入测试
- [ ] 边界输入测试
- [ ] 恶意输入测试
- [ ] 长输入测试
- [ ] 多语言输入测试

## 审查结论
- [ ] 通过
- [ ] 有条件通过（需修改：...）
- [ ] 不通过（原因：...）
```

## 交付物模板 4：Chain-of-Thought 设计模板

```markdown
# CoT Prompt 设计: [任务名称]

## 适用场景
[说明为什么这个任务需要 CoT]

## Prompt 模板

```
[角色]
你是 [专家角色]，请逐步思考以下问题。

[问题]
{{question}}

[思考框架]
请按以下步骤思考：
1. 理解问题：明确问题的核心要求和约束条件
2. 分析信息：识别已知条件和未知条件
3. 制定方案：列出可能的解决方案
4. 评估方案：对比各方案的优缺点
5. 得出结论：选择最优方案并说明理由

[输出格式]
请按以下格式输出：

## 思考过程
[逐步展示思考过程]

## 最终答案
[清晰、简洁的最终答案]

## 置信度
[高/中/低，并说明原因]
```

## 示例

### 输入
[示例输入]

### 预期输出
[示例输出，展示完整的思考过程]

## 评估指标
- 思考步骤完整性：是否覆盖所有必要步骤
- 逻辑一致性：推理过程是否自洽
- 答案准确性：最终答案是否正确
- 冗余度：思考过程是否简洁（避免过度思考）
```
