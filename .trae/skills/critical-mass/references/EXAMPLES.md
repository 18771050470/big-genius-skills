# 分析示例

## 示例 1：临界点——社交产品用户增长

**输入**：这个社交APP上线6个月，用户从0增长到5万，但增长很慢，什么时候能爆发？

```
Step 1: 识别临界信号 → 产出：发现"增长很慢...什么时候爆发"表述，寻找量变积累信号；当前5万用户，日活30%，留存率45%（较高）
Step 2: 评估临界条件 → 产出：社交网络临界阈值通常需要10-15%目标人口采用；假设目标市场500万，需50-75万用户；当前5万仅1%
Step 3: 分析触发机制 → 产出：正反馈回路尚未形成（用户少→内容少→吸引力低）；关键触发：聚焦细分社区达到密度阈值
Step 4: 制定临界策略 → 产出：建议聚焦单一城市/兴趣群体，先在小范围达到临界密度，再向外扩散
```

**输出**：
```
## 分析结论
### 核心发现
1. 当前5万用户（1%目标市场），距离临界点（10-15%）还有10倍差距
2. 留存率45%较高，说明产品有价值，但网络效应未形成
3. 线性增长无法到达临界点，需要策略性聚焦

### 建议（按优先级）
1. [高优先级] 聚焦单一细分群体（如某城市大学生），达到30%渗透率触发局部临界点
2. [高优先级] 增强网络效应功能：好友推荐、内容分享、互动激励
3. [中优先级] 邀请关键用户（KOL、社区活跃者）加速网络密度

### 关键假设
- 目标市场500万可触达
- 留存率45%可持续
- 聚焦策略可执行

### 风险
- 聚焦可能限制短期增长数据
- 需要6-12个月才能看到临界效应
- 竞品可能抢先达到临界点

### 下一步
建议调用 [system-thinking] 分析用户增长反馈回路
```

## 示例 2：临界点——技术采用决策

**输入**：我们自研的AI工具内部试用率只有5%，什么时候全公司都会用起来？

```
Step 1: 识别临界信号 → 产出：发现"5%...什么时候全公司"表述，评估技术采用临界点
Step 2: 评估临界条件 → 产出：Rogers创新扩散理论：创新者2.5%+早期采用者13.5%=16%为临界阈值；当前5%处于创新者阶段
Step 3: 分析触发机制 → 产出：需要从创新者跨越到早期采用者；关键触发：展示成功案例、降低使用门槛
Step 4: 制定临界策略 → 产出：建议聚焦早期采用者特征人群，打造标杆案例
```

**输出**：
```
## 分析结论
### 核心发现
1. 当前5%采用率处于创新者阶段，距离临界点16%还有11个百分点
2. Rogers曲线显示：从创新者到早期采用者是最难的跨越（"死亡之谷"）
3. 需要标杆案例和降低门槛来触发跨越

### 建议（按优先级）
1. [高优先级] 识别并赋能早期采用者（技术敏感、愿意尝试、有影响力）
2. [高优先级] 打造3-5个成功案例，展示ROI和效率提升
3. [中优先级] 简化上手流程，降低学习成本

### 关键假设
- 16%临界阈值适用于企业内部
- 早期采用者可识别
- 标杆案例可复制

### 风险
- 跨越"死亡之谷"失败率约60%
- 需要6-18个月
- 组织文化可能阻碍采用

### 下一步
建议调用 [compound-thinking] 评估持续投入的复利效应
```

## 反例：不该触发此 Skill 的场景

**输入**：这个功能上线后用户量每月增长10%，预计年底达到100万。

**原因**：这是线性增长预测，不涉及临界阈值效应。临界点思维适用于网络效应、正反馈回路导致的非线性爆发场景。
