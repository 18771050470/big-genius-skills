# 分析示例

## 示例 1：自利偏差——项目成功归因

**输入**：这个项目成功上线，主要是我领导有方，团队执行力强。

```
Step 1: 识别自利信号 → 产出：发现"我领导有方...成功"表述，将成功归因于内部因素
Step 2: 评估归因模式 → 产出：成功归因于自己（能力、努力），失败归因于外部（环境、运气）
Step 3: 分析自利机制 → 产出：自我增强：维护积极自我形象；印象管理：向他人展示能力
Step 4: 校正自利偏差 → 产出：建议多因素归因，识别外部贡献
```

**输出**：
```
## 分析结论
### 核心发现
1. 成功归因于"我领导有方"（内部），忽视外部因素
2. 自利偏差使成功归因于自己，失败归因于外部
3. 项目成功多因素：团队努力(40%)、市场环境(30%)、资源支持(20%)、领导(10%)

### 建议（按优先级）
1. [高优先级] 多因素归因：识别各贡献方
2. [高优先级] 公开认可团队贡献
3. [中优先级] 客观评估自己贡献比例

### 关键假设
- 成功是多因素结果
- 团队贡献应被认可

### 风险
- 自利偏差可能导致团队不满
- 需要平衡自信和谦逊
```

## 示例 2：自利偏差——项目失败归因

**输入**：这个项目延期了，主要是需求变更太频繁，客户也不配合，不是我们的问题。

```
Step 1: 识别自利信号 → 产出：发现"不是我们的问题"表述，将失败归因于外部因素
Step 2: 评估归因模式 → 产出：失败归因于外部（需求变更、客户），忽视内部因素（计划、沟通）
Step 3: 分析自利机制 → 产出：自我保护：避免负面自我评价；责任规避：减少个人责任
Step 4: 校正自利偏差 → 产出：建议内外部因素平衡归因
```

**输出**：
```
## 分析结论
### 核心发现
1. 失败归因于外部（需求变更、客户），忽视内部因素
2. 自利偏差使失败归因外部化，避免自我批评
3. 内部因素：计划不周(30%)、沟通不足(20%)、风险管理(10%)

### 建议（按优先级）
1. [高优先级] 平衡归因：外部50%+内部50%
2. [高优先级] 识别可改进的内部因素
3. [中优先级] 建立复盘文化，而非责任规避

### 关键假设
- 失败通常多因素
- 内部因素可改进

### 风险
- 自利偏差可能错过改进机会
- 需要建设性复盘
```

## 反例：不该触发此 Skill 的场景

**输入**：根据数据，这个项目成功主要因为市场需求爆发，我们刚好赶上。

**原因**：这是基于数据的客观归因（市场需求爆发），不是自利偏差。自利偏差指将成功归因于自己。
