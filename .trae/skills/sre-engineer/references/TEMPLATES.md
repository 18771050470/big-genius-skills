# 技术交付物模板

## 交付物模板 1：SLO 定义文档

```markdown
# SLO 定义：[服务名称]

## 服务概述
- **服务名称**：[名称]
- **服务描述**：[一句话描述]
- **用户旅程**：[核心用户操作]

## SLI/SLO 定义

| SLI | 测量方法 | 目标 (SLO) | 窗口 | 数据来源 |
|-----|---------|-----------|------|---------|
| 可用性 | HTTP 200 比例 | 99.95% | 30 天 | Load Balancer 日志 |
| 延迟 (P99) | 请求响应时间 | < 500ms | 30 天 | APM 工具 |
| 错误率 | 5xx 比例 | < 0.1% | 30 天 | 应用日志 |
| 吞吐量 | 每秒请求数 | > 1000/s | 30 天 | 监控系统 |

## 错误预算
- **预算计算**：(1 - 99.95%) × 30 天总请求数 = [N] 个错误请求
- **当前消耗**：[N]（[百分比]%）
- **状态**：🟢 健康 / 🟡 警告 / 🔴 耗尽

## 预算政策
- 🟢 健康（< 50%）：正常发布
- 🟡 警告（50-100%）：发布需 SRE 审批
- 🔴 耗尽（> 100%）：停止非紧急发布

## 例外流程
- 紧急安全补丁：随时发布，事后分析
- 产品 deadline 冲突：CTO 决策
```

## 交付物模板 2：故障 Runbook

```markdown
# Runbook：[故障场景]

## 现象
[故障的外在表现]

## 影响评估
- **用户影响**：[描述]
- **业务影响**：[描述]
- **分级**：P0 / P1 / P2 / P3

## 诊断步骤

### 步骤 1：[名称]
```bash
# 命令
[具体命令]

# 预期输出
[正常输出]

# 判断
- 如果 [条件] → 进入步骤 2
- 如果 [条件] → 进入步骤 3
```

### 步骤 2：[名称]
```bash
# 命令
[具体命令]
```

## 修复步骤

### 方案 A：[名称]
```bash
# 命令
[具体命令]

# 验证
[验证方法]
```

### 方案 B（如果方案 A 无效）：[名称]
```bash
# 命令
[具体命令]
```

## 回滚
```bash
# 回滚命令
[具体命令]

# 验证回滚成功
[验证方法]
```

## 事后检查
- [ ] 服务恢复正常
- [ ] 监控无告警
- [ ] 用户反馈正常
```

```markdown
# Postmortem：[事件名称]

## 事件信息
- **事件编号**：INC-[编号]
- **时间**：[起止时间]

- **分级**：P0 / P1 / P2 / P3

## 时间线
| 时间 | 事件 | 操作 |
|------|------|------|
| 09:00 | [事件] | [操作] |
| 09:05 | [事件] | [操作] |

## 影响评估
- **受影响用户**：[N]
- **业务损失**：[金额/描述]
- **SLI 影响**：可用性从 [X]% 降至 [Y]%

## 根因分析（5 Whys）
1. **为什么**：[问题] → **因为**：[原因]
2. **为什么**：[原因] → **因为**：[原因]
3. **为什么**：[原因] → **因为**：[原因]
4. **为什么**：[原因] → **因为**：[原因]
5. **为什么**：[原因] → **根因**：[根因]

## 改进措施
| ID | 措施 | 负责人 | 截止日期 | 状态 |
|----|------|--------|---------|------|
| [ID] | [措施] | [姓名] | [日期] | 待开始 |

## 经验教训
- [经验 1]
- [经验 2]
```
