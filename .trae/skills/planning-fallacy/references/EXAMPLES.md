# 分析示例

## 示例 1：规划谬误——软件开发估算

**输入**：这个功能开发需要2周，测试1周，总共3周就能上线。

```
Step 1: 识别规划信号 → 产出：发现"3周就能上线"表述，基于最佳情况估计，忽视历史数据
Step 2: 评估估算依据 → 产出：基于理想情况估计，历史类似功能平均耗时5周
Step 3: 分析规划机制 → 产出：内部视角：关注当前计划；忽视分布：不考虑历史完成时间分布
Step 4: 校正规划谬误 → 产出：建议用参考类预测，基于历史数据调整
```

**输出**：
```
## 分析结论
### 核心发现
1. 估计3周，历史类似功能平均5周，低估40%
2. 规划谬误使基于最佳情况估计，忽视常见问题
3. 未考虑：需求变更(平均+1周)、bug修复(平均+1周)

### 建议（按优先级）
1. [高优先级] 参考类预测：基于历史数据调整到5周
2. [高优先级] 添加15%应急缓冲：5.75周
3. [中优先级] 识别最可能延期的环节

### 关键假设
- 历史项目与当前可比
- 团队规模和技术栈相似

### 风险
- 规划谬误可能导致承诺不可实现
- 需要基于历史数据校准
```

## 示例 2：规划谬误——装修项目

**输入**：装修房子1个月就能搞定，材料2周，施工2周。

```
Step 1: 识别规划信号 → 产出：发现"1个月就能搞定"表述，未考虑常见延误因素
Step 2: 评估估算依据 → 产出：基于理想流程估计，历史装修平均耗时2-3个月
Step 3: 分析规划机制 → 产出：最佳情况思维：假设一切顺利；忽视外部依赖：材料延迟、审批时间
Step 4: 校正规划谬误 → 产出：建议用历史数据和外部依赖评估
```

**输出**：
```
## 分析结论
### 核心发现
1. 估计1个月，历史平均2-3个月，低估50-67%
2. 未考虑：审批(1-2周)、材料延迟(1周)、返工(1-2周)
3. 规划谬误使假设一切顺利

### 建议（按优先级）
1. [高优先级] 基于历史数据：2-3个月
2. [高优先级] 识别外部依赖和延误风险
3. [中优先级] 设置阶段性里程碑

### 关键假设
- 历史装修数据可参考
- 外部依赖不可控

### 风险
- 规划谬误可能导致预算超支
- 需要现实时间线
```

## 反例：不该触发此 Skill 的场景

**输入**：根据过去10次类似项目数据，平均耗时3周，这次也估计3周。

**原因**：这是基于历史数据的估计，不是规划谬误。规划谬误指忽视历史数据的乐观估计。
