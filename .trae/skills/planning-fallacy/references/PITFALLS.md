# 常见陷阱

## 陷阱 1：基于最佳情况估计

**问题**：假设一切顺利，不考虑可能出现的问题。

❌ **Before（错误做法）**
```markdown
## 分析结论
开发2周+测试1周=3周上线。
建议：按3周计划。
```
*问题：规划谬误使基于最佳情况，忽视常见问题*

✅ **After（正确做法）**
```markdown
## 分析结论
### 参考类预测
| 阶段 | 最佳情况 | 历史平均 | 建议 |
|------|---------|---------|------|
| 开发 | 2周 | 3周 | 3周 |
| 测试 | 1周 | 1.5周 | 1.5周 |
| 缓冲 | 0 | 0.5周 | 0.5周 |
| 总计 | 3周 | 5周 | 5周 |

### 建议
1. 历史平均比最佳情况更可靠
2. 添加应急缓冲
```
*优势：历史数据替代最佳情况*

**原因**：Kahneman & Tversky(1979)规划谬误：人们基于内部视角(当前计划)估计，忽视外部视角(历史分布)。

## 陷阱 2：忽视外部依赖

**问题**：只考虑自己可控的部分，忽视外部依赖。

❌ **Before（错误做法）**
```markdown
## 分析结论
我们2周开发完就能上线。
建议：按计划执行。
```
*问题：规划谬误使忽视审批、第三方等外部依赖*

✅ **After（正确做法）**
```markdown
## 分析结论
### 依赖分析
| 依赖 | 耗时 | 可控性 |
|------|------|--------|
| 开发 | 2周 | 高 |
| 测试 | 1周 | 高 |
| 审批 | 1周 | 低 |
| 第三方集成 | 1周 | 中 |
| 总计 | 5周 | - |

### 建议
1. 外部依赖增加2周
2. 不可控依赖需要额外缓冲
```
*优势：识别外部依赖，完整估算*

**原因**：规划谬误使聚焦自己可控部分，忽视外部依赖的延误风险。

## 陷阱 3：未考虑需求变更

**问题**：假设需求不会变化。

❌ **Before（错误做法）**
```markdown
## 分析结论
需求已确定，3周完成。
建议：锁定需求。
```
*问题：规划谬误使假设需求不变*

✅ **After（正确做法）**
```markdown
## 分析结论
### 变更概率分析
| 因素 | 变更概率 | 影响 |
|------|---------|------|
| 历史变更率 | 70% | +1周 |
| 需求明确度 | 中 | 可能变更 |
| 利益相关方 | 多 | 变更风险高 |

### 建议
1. 70%概率需求变更
2. 预留变更缓冲
```
*优势：量化变更风险*

**原因**：规划谬误使假设计划按预期执行，但需求变更是常态。
