## 交付物模板 3：设备管理流程

```markdown
# 设备管理设计：[项目名称]

## 设备生命周期

```
[出厂] → [注册] → [激活] → [在线] → [休眠] → [离线] → [注销]
  │        │        │        │        │        │        │
  │        │        │        │        │        │        │
  ▼        ▼        ▼        ▼        ▼        ▼        ▼
烧录证书  云端登记  首次连接  正常运行  低功耗   网络中断  设备退役
```

## 注册流程

1. 设备出厂烧录唯一证书（X.509）
2. 设备首次连接时携带证书
3. 云端验证证书链和签名
4. 验证通过后分配设备ID和Token
5. 设备后续使用Token连接

## OTA升级流程

1. 云端发布新版本固件
2. 设备收到升级通知
3. 设备下载固件到备用分区
4. 设备验证固件签名和完整性
5. 设备切换到新固件启动
6. 启动成功后上报云端
7. 启动失败则回滚到原固件

## 监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| 在线率 | 在线设备/总设备 | < 95% |
| 消息成功率 | 成功消息/总消息 | < 99% |
| 平均延迟 | 消息端到端延迟 | > 1s |
| 升级成功率 | 成功升级/总升级 | < 98% |
```

## 交付物模板 4：数据管道设计

```markdown
# 数据管道设计：[项目名称]

## 数据采集

| 数据源 | 采样频率 | 数据格式 | 上报策略 |
|--------|---------|---------|---------|
| 温度传感器 | 1分钟 | JSON | 变化上报+定时上报 |
| 湿度传感器 | 1分钟 | JSON | 变化上报+定时上报 |
| 告警事件 | 实时 | JSON | 立即上报 |

## 数据清洗

| 规则 | 说明 |
|------|------|
| 异常值过滤 | 超出物理范围的数据丢弃 |
| 单位转换 | 原始ADC值 → 实际物理量 |
| 格式标准化 | 统一时间格式、数值精度 |

## 数据存储

| 数据类型 | 存储方案 | 保留时间 |
|---------|---------|---------|
| 原始数据 | 时序数据库 | 30天 |
| 聚合数据 | 关系数据库 | 1年 |
| 告警数据 | 关系数据库 | 3年 |
| 日志数据 | 对象存储 | 1年 |

## 实时分析

| 规则 | 条件 | 动作 |
|------|------|------|
| 高温告警 | 温度 > 40°C | 发送告警通知 |
| 离线告警 | 设备离线 > 5分钟 | 发送告警通知 |
| 数据异常 | 连续3次异常值 | 标记设备待检查 |

## 离线分析

| 分析类型 | 频率 | 输出 |
|---------|------|------|
| 趋势分析 | 每天 | 日报 |
| 预测维护 | 每周 | 维护建议 |
| 能耗分析 | 每月 | 月报 |
```
